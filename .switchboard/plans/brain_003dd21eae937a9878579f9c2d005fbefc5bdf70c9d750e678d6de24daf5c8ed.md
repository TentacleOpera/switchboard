# UI Layout Adjustment: Move Folders Button and Add Antigravity Note

## Metadata
**Complexity:** 2
**Tags:** ui, ux, frontend

## Goal
Improve the user experience and interface layout of the Local Docs tab in the planning view by:
1. Placing the "Folders" action button directly where it belongs—at the top of the left docs sidebar (`#tree-pane`)—rather than far away on the right-hand side of the top controls strip.
2. Adding a helpful descriptive note below the "Show Antigravity Brain" toggle in the folders management modal to make it clear which directories are being scanned (App, IDE, and CLI brain folders).

### Background & Problem Analysis
* The **Folders** button manages which local directories are scanned for documentation files shown in the left sidebar tree.
* Currently, the button is located on the far right of the top controls strip (the `controls-strip`), which is visually separated and distant from the sidebar tree it actually controls.
* The "Show Antigravity Brain" toggle in the modal does not explain which folders are included, causing ambiguity for users who might expect only IDE or CLI sessions.

## Proposed Changes

### Switchboard Webview

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)

* Insert the descriptive note `<div class="toggle-note">` below the toggle container for `#antigravity-toggle-modal` in the folders modal.

#### [MODIFY] [planning.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.js)
* Refactor the `#btn-manage-folders` click listener into a standalone `openFoldersModal()` function.
* Update `renderLocalDocs()` to dynamically construct and append the `#btn-manage-folders` button (`.sidebar-folders-btn`) to the `#tree-pane` sidebar container, wiring it to the new `openFoldersModal()` function.
