## Step 1: Understand the Goal
Develop an AWS Lambda Python script to consume Amplitude JSON.GZ logs automatically exported to S3, parse them, and efficiently bulk-insert the raw behavioral data into the MongoDB Atlas cluster for long-term storage and custom data science queries.

## Step 2: Complexity, Edge-Case & Dependency Audit
*   **Complexity:** Medium. Requires Python streaming decompression and PyMongo bulk operation integration.
*   **Edge Cases:** Amplitude exports `.json.gz` files that can grow quite large; reading the entire file into RAM will crash the Lambda container (MemoryError). The script must parse the file line-by-line via streaming. Furthermore, backfills or overlapping exports will cause duplicate records; the script must safely deduplicate these using MongoDB's primary key index.
*   **Dependencies & Conflicts:** Depends on configuring IAM roles properly outside this codebase.

## Step 3: Improve Plan (/improve-plan)
*   **Grumpy Persona:** Executing `insert_one` in a Python `for` loop over 50,000 lines of JSON will take 20 minutes, causing a Lambda timeout and skyrocketing our AWS bill. Additionally, if you don't map `insert_id` to `_id`, MongoDB won't deduplicate the records.
*   **Balanced Persona:** We will use `insert_many` with `ordered=False` to batch process 5,000 records at a time. By wrapping the `get_object` stream in `gzip.GzipFile`, we maintain a minimal memory footprint. We will catch and ignore `BulkWriteError` exceptions (specifically code `E11000`) so overlapping files safely skip duplicate insertions.

## 5. Exhaustive Implementation Spec

### Adversarial Synthesis
#### Grumpy Critique
If you try to load a 500MB gzip file into RAM in a 128MB Lambda function, it will crash immediately before reaching the database logic.
#### Balanced Response
Boto3's `get_object` returns a `StreamingBody`. By feeding this `StreamingBody` directly into `gzip.GzipFile` line-by-line, we keep the memory usage constant regardless of file size, effectively eliminating OOM crashes.

#### ETL Script
##### [CREATE] `functions/etl/amplitude_s3_to_mongo.py`
- **Context:** The Lambda entry point for processing S3 Object Created events.
- **Logic:** 
    1. Retrieve Bucket and Key from the incoming S3 Event trigger.
    2. Stream the object from S3 using Boto3.
    3. Decompress the stream natively using `gzip`.
    4. Build JSON batches (5000 events), explicitly assigning Amplitude's `insert_id` to MongoDB's `_id` to guarantee idempotency.
    5. Bulk insert to MongoDB with `ordered=False` to ignore duplicates.
- **Implementation:**
```python
import os
import json
import gzip
import urllib.parse
import boto3
from pymongo import MongoClient
from pymongo.errors import BulkWriteError

s3_client = boto3.client('s3')
MONGO_URI = os.environ.get('MONGO_URI')
DB_NAME = os.environ.get('MONGO_DB_NAME', 'autism360prod')
COLLECTION_NAME = 'amplitude_events'

# Initialize MongoDB Client outside handler for connection pooling
mongo_client = MongoClient(MONGO_URI)
db = mongo_client[DB_NAME]
collection = db[COLLECTION_NAME]

def lambda_handler(event, context):
    for record in event['Records']:
        bucket = record['s3']['bucket']['name']
        key = urllib.parse.unquote_plus(record['s3']['object']['key'])
        
        print(f"Processing s3://{bucket}/{key}")
        
        try:
            response = s3_client.get_object(Bucket=bucket, Key=key)
            # Use streaming body wrapped in gzip to avoid memory limits
            with gzip.GzipFile(fileobj=response['Body'], mode='rb') as gz:
                batch = []
                batch_size = 5000
                
                for line in gz:
                    try:
                        event_data = json.loads(line.decode('utf-8'))
                        
                        # Map Amplitude insert_id to MongoDB _id to guarantee idempotency
                        if 'insert_id' in event_data:
                            event_data['_id'] = event_data['insert_id']
                            
                        batch.append(event_data)
                        
                        # Insert in chunks to optimize memory and network
                        if len(batch) >= batch_size:
                            _flush_batch(batch)
                            batch = []
                    except json.JSONDecodeError:
                        continue
                
                # Flush any remaining records
                if batch:
                    _flush_batch(batch)
                    
        except Exception as e:
            print(f"Error processing file {key} from bucket {bucket}: {str(e)}")
            raise e

def _flush_batch(batch):
    try:
        # ordered=False allows Mongo to continue inserting even if duplicate _id conflicts occur
        collection.insert_many(batch, ordered=False)
        print(f"Successfully inserted batch of {len(batch)} records.")
    except BulkWriteError as bwe:
        # We expect E11000 duplicate key errors if backfills overlap; safely ignore them
        write_errors = bwe.details.get('writeErrors', [])
        duplicates = sum(1 for err in write_errors if err.get('code') == 11000)
        print(f"Batch completed with {duplicates} duplicates safely ignored.")
```
