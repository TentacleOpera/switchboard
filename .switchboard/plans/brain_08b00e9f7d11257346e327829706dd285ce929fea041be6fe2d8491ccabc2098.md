# Aligning Planning View with Kanban Theme and Improving Preview Gridlines Visibility

## Metadata
**Complexity:** 2
**Tags:** frontend, ui, ux, bugfix

## Goal
The goal of this change is to align the aesthetic theme of the Planning/Artifacts view (`planning.html`) with the Kanban board (`kanban.html`) and to make the gridlines pattern clearly visible in the document preview sections.

### Problem Analysis & Root Cause
1. **Mismatched Webview Themes:** When the "Cyber Panel" theme is enabled, `planning.html` currently uses a dark blue background color (`#040409`) and magenta/cyan radial glows, which differs significantly from the dark grey (`#1a1a1a`) background used by the Kanban board (`kanban.html`). `kanban.html` is the source of truth and must remain untouched.
2. **Obscured Gridlines in Preview Panes:** The document preview panes (`#preview-pane`, `#preview-pane-online`, `#kanban-preview-pane`) use `backdrop-filter: blur(6px)` and a semi-opaque background. Since the gridlines are only applied on the `body` element, the blur effect completely smudges them, making them invisible in the document preview sections.

## Proposed Changes

### Webview Frontend

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)
1. **Remove Blue Tint:**
   - Update `body.cyber-theme-enabled` background color from `#040409` to `#1a1a1a` to match `kanban.html`.
   - Remove the cyan/magenta radial gradients from the `background-image` of `body.cyber-theme-enabled` to eliminate the colored glow.
2. **Make Gridlines Visible in Doc Previews:**
   - Apply the gridline pattern directly to the preview pane selectors (`#preview-pane`, `#preview-pane-online`, `#kanban-preview-pane`) inside the cyber theme CSS block.
   - Increase the gridline line color opacity slightly (e.g. from `0.04` to `0.08`) so they stand out clearly against the dark panel background without being blurred by the backdrop filter.
