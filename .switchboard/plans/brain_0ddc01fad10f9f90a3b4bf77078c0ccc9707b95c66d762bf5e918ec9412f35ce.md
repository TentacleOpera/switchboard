# Kanban Layout Refinement and Duplicate Copy Chat Button Cleanup

## Goal

Clean up and restructure the Kanban board controls strip, sub-bar, columns, and toast messages to improve layout consistency, prevent UI shifting when automation is inactive, and remove the duplicate "Copy Chat" button whose functionality is fully covered by the Created column's chat button.

## Metadata

- **Tags:** [UI, UX, workflow]
- **Complexity:** 4

## User Review Required

> [!IMPORTANT]
> The duplicate "Copy Chat" button in the sub-bar (`#btn-copy-chat-workflow`) will be removed entirely, as its functionality is fully covered by the copy chat button in the `Created` column header. The sub-bar button always sends an empty sessionIds list (general chat only), while the column button supports both selected-plan and no-selection (general chat) modes — making the sub-bar button redundant.

## Complexity Audit

### Routine
- Moving `#btn-cli-triggers` from controls-strip to sub-bar (DOM reorder, same event handler)
- Removing `#btn-copy-chat-workflow` HTML element and its click handler (lines 2165-2167, 6711-6713)
- Repositioning `#status-message` to the end of `.kanban-sub-bar` (DOM reorder)
- Updating tooltip text on the Created column chat button

### Complex / Risky
- Toggling `#autoban-timers-inline` visibility via class-based approach to eliminate the double-gap when automation is off — must not fight the CSS `display: flex` declaration
- Moving the chat button from `column-button-area` to the header `rightSide` — must verify event delegation still resolves `data-action="chatCopyPrompt"` correctly from the header context

## Edge-Case & Dependency Audit

- **Race Conditions:** None — all changes are static DOM restructuring or synchronous JS toggling.
- **Security:** No security implications — purely UI layout changes.
- **Side Effects:** Removing `#btn-copy-chat-workflow` eliminates the `copyChatWorkflow` message type from the webview. The backend handler in `KanbanProvider.ts` (line 5234-5242) will become dead code. Consider whether to also remove the backend handler or leave it as a no-op for backward compatibility.
- **Dependencies & Conflicts:** The `chatCopyPrompt` handler (KanbanProvider.ts line 5211-5232) already handles the empty-sessionIds case, producing a general chat prompt identical to `copyChatWorkflow`. No backend changes are needed for the column button to cover the removed button's behavior.

## Dependencies

- None

## Adversarial Synthesis

Key risks: (1) The `autoban-timers-inline` display toggling must use a CSS class (e.g., `.is-empty`) rather than inline `style.display` to avoid fighting the stylesheet's `display: flex` declaration. (2) Moving the chat button to the column header requires verifying that the `data-action` event delegation still resolves correctly. (3) The backend `copyChatWorkflow` handler becomes dead code. Mitigations: Use class-based visibility toggle; verify event delegation in manual testing; leave backend handler as no-op or remove in a follow-up.

#### 3. Adjust Timers Container Visibility

- **Context:** `#autoban-timers-inline` (line 2151) currently has CSS `display: flex` (line 1359-1364). When automation is off, `updateAutobanIndicators()` (line 6180-6183) clears `innerHTML` to `''`, but the empty flex container still participates in the parent's `gap: 10px`, creating a visible double-gap.
- **Logic:** Hide the container entirely when it has no content, eliminating the extra gap.
- **Implementation:**
  - Add a CSS rule for the hidden state:
    ```css
    .autoban-timers-inline.is-empty {
        display: none;
    }
    ```
  - In `updateAutobanIndicators()` (line 6180-6183), when clearing the container, add the `is-empty` class:
    ```javascript
    if (!autobanConfig || !autobanConfig.enabled) {
        if (container.innerHTML !== '') { container.innerHTML = ''; }
        container.classList.add('is-empty');
        updateAutobanButtonState();
        return;
    }
    ```
  - When populating the container (lines 6208-6211 and 6254-6257), remove the `is-empty` class:
    ```javascript
    container.classList.remove('is-empty');
    ```
  - Also add `is-empty` class on initial load — add `class="autoban-timers-inline is-empty"` to the HTML element at line 2151.
- **Edge Cases:** The `is-empty` class approach avoids fighting the CSS `display: flex` declaration with inline styles. When the class is removed, the CSS `display: flex` takes effect naturally. When the class is present, `display: none` overrides it via specificity (both are class selectors, but `.is-empty` is listed second in the stylesheet, winning by source order).

#### 4. Reposition Toast Status Message to End of Sub-bar

- **Context:** `#status-message` (line 2161) currently sits between `#btn-reset-autoban-timer` and `#btn-collapse-coders`. When it appears (via `showStatusMessage` at line 5656-5667), it can push the collapse-coders and copy-chat buttons to the right.
- **Logic:** Move `#status-message` to the very end of `.kanban-sub-bar`, after all buttons. This way, when the toast appears, it extends to the right without shifting any button positions.
- **Implementation:**
  - Cut line 2161:
    ```html
    <div id="status-message" role="status" aria-live="polite" class="sub-bar-status" style="display:none;"></div>
    ```
  - Insert it as the last child of `.kanban-sub-bar` (after the `#btn-copy-chat-workflow` removal, this will be after `#btn-collapse-coders`).
  - Add `flex-shrink: 0` to the icon buttons in the sub-bar to prevent them from being compressed when the status message appears. Add `min-width: 0; overflow: hidden;` to `#status-message` to ensure it truncates rather than pushes:
    ```css
    #status-message.sub-bar-status {
        min-width: 0;
        overflow: hidden;
    }
    ```
  - **Clarification:** The existing `text-overflow: ellipsis` and `white-space: nowrap` on `.sub-bar-status` (lines 163-167) already handle text truncation. Adding `min-width: 0` allows the flex item to shrink below its content size, and `overflow: hidden` enables the ellipsis to trigger.
- **Edge Cases:** The `animationend` listener at line 8595-8598 sets `style.display = 'none'` after the flash animation completes. This still works regardless of DOM position. The `showStatusMessage` handler sets `style.display = 'inline-block'` — this also still works.
