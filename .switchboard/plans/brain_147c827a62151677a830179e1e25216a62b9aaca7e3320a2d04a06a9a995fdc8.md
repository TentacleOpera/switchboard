# Remove Unused Codicons Boilerplate Injection

## Goal

The Switchboard extension's webview unconditionally attempts to load the VS Code codicons stylesheet (`node_modules/@vscode/codicons/dist/codicon.css`) in the HTML header at startup.

However:
1. The packaged extension excludes the `node_modules` directory entirely (via `.vscodeignore`), causing the webview loader to throw `404 Not Found` errors in the developer console.
2. A search of the codebase reveals that the webviews do not use any codicons or the `.codicon` stylesheet classes.

This is unused boilerplate code. Rather than packaging and copying the assets, the correct fix is to remove the unused codicon injection logic from `src/services/TaskViewerProvider.ts`.

## Edge-Case & Dependency Audit

- **Race Conditions**: None — removal of dead code, no runtime logic affected.
- **Security**: Removing `codiconsUri` from `localResourceRoots` narrows the webview's file-system access, which is a minor security improvement (principle of least privilege).
- **Side Effects**: The `@vscode/codicons` package remains in `package.json` (line 659) as a devDependency. This plan does not remove it since `npm uninstall` is a separate operation and the package may be used by other tooling. A follow-up task could remove it if confirmed unused project-wide.
- **Dependencies & Conflicts**: No other `.ts` files in `src/` reference codicons (verified via grep). No webview HTML files use `.codicon` CSS classes (verified via grep). The CSP `font-src` directive in `_getHtmlForWebview` (line 17307) remains unchanged — it is still needed for other webview fonts (e.g., Inter).

## Proposed Changes

### Extension Runtime Configuration

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)

1. Remove the unused `codiconsUri` setup from `webviewView.webview.options` (lines 7546-7551):
```typescript
        // Add codicons to localResourceRoots for webview-safe URI access
        const codiconsUri = vscode.Uri.joinPath(this._extensionUri, 'node_modules', '@vscode', 'codicons');
        webviewView.webview.options = {
            enableScripts: true,
            localResourceRoots: [this._extensionUri, codiconsUri]
        };
```
Change to:
```typescript
        webviewView.webview.options = {
            enableScripts: true,
            localResourceRoots: [this._extensionUri]
        };
```

2. Remove the unused HTML head injection of `codicon.css` (lines 17317-17320):
```typescript
            // Inject Codicon CSS with webview-safe URI
            const codiconUri = webview.asWebviewUri(vscode.Uri.joinPath(this._extensionUri, 'node_modules', '@vscode', 'codicons', 'dist', 'codicon.css'));
            const codiconLink = `<link href="${codiconUri}" rel="stylesheet" />`;
            content = content.replace('</head>', `${codiconLink}\n</head>`);
```
