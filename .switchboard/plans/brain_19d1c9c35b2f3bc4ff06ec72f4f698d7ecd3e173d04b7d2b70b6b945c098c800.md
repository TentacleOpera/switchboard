# MCP Setup for Gemini/Antigravity — Proper Implementation Plan

## Problem

When a user installs the Switchboard extension and runs setup, the extension correctly creates:
- [.switchboard/MCP/mcp-server.js](file:///x:/documents/GitHub/switchboard/.switchboard/MCP/mcp-server.js) (the runtime server)
- [.gemini/settings.json](file:///x:/documents/GitHub/switchboard/.gemini/settings.json) _inside_ the workspace (picked up by some Gemini CLI modes)

But it does **not** touch the **global Antigravity config** at `~/.gemini/antigravity/mcp_config.json`, which is what Gemini Desktop (Antigravity) and some Gemini CLI invocations actually read. This leaves every user in a broken state until they manually fix it.

## Core Principle: No Hardcoded Paths — Ever

Every path written anywhere in this feature MUST be resolved programmatically:
- **Global config location**: `path.join(os.homedir(), '.gemini', 'antigravity', 'mcp_config.json')`
- **MCP server path in config**: `path.join(workspaceRoot, '.switchboard', 'MCP', 'mcp-server.js')` — where `workspaceRoot` is always the VS Code workspace root at setup time, resolved via [getPreferredWorkspaceRoot()](file:///x:/documents/GitHub/switchboard/src/extension.ts#195-206).

The result will look different on every user's machine. That is correct and expected.

## Proposed Changes

### Component 1 — New helper in [src/extension.ts](file:///x:/documents/GitHub/switchboard/src/extension.ts)

#### [MODIFY] [extension.ts](file:///x:/documents/GitHub/switchboard/src/extension.ts)

Add a new exported function `getGlobalAntigravityMcpConfigPath()`:

```typescript
import * as os from 'os';

function getGlobalAntigravityMcpConfigPath(): string {
    return path.join(os.homedir(), '.gemini', 'antigravity', 'mcp_config.json');
}
```

Add a new `setupGlobalAntigravityMcpConfig(workspaceRoot: string)` function with this logic:

1. Compute `configPath = getGlobalAntigravityMcpConfigPath()`.
2. Compute `serverPath = path.join(workspaceRoot, '.switchboard', 'MCP', 'mcp-server.js')`.
3. **Validate** `serverPath` exists on disk. If not, [ensureWorkspaceMcpServerFiles](file:///x:/documents/GitHub/switchboard/src/extension.ts#120-166) first, then recheck. Abort with error if still missing.
4. **Validate** `serverPath` is within `workspaceRoot` using existing [isPathWithinRoot](file:///x:/documents/GitHub/switchboard/src/extension.ts#179-183). Abort if not.
5. Check if parent directory of `configPath` exists. If not, surface:
   > "Antigravity config directory not found at `~/.gemini/antigravity/`. Is Gemini Desktop installed?"
   Do NOT create the directory.
6. Read existing config (or start with `{}` if file doesn't exist).
7. Merge the `switchboard` key only — preserve all other existing keys.
8. Produce a JSON diff string (formatted `before` vs `after`).
9. Show a VS Code notification: `"Update global Antigravity MCP config?"` with buttons:
   - **`Write to file`** — backup existing file as `mcp_config.json.bak.<timestamp>`, then write.
   - **`Copy to clipboard`** — write the JSON to clipboard, show instruction: "Paste this into `~/.gemini/antigravity/mcp_config.json`".
   - **`Skip`** — do nothing.
10. On write, call `vscode.window.showInformationMessage('✅ Global Antigravity MCP config updated.')`.

**The generated config entry will look like:**
```json
{
  "mcpServers": {
    "switchboard": {
      "command": "node",
      "args": [
        "/Users/alice/projects/myapp/.switchboard/MCP/mcp-server.js",
        "/Users/alice/projects/myapp"
      ],
      "env": {
        "SWITCHBOARD_WORKSPACE_ROOT": "/Users/alice/projects/myapp"
      }
    }
  }
}
```
(Paths are always the current user's workspace, computed at runtime — never hardcoded by the developer.)

### Component 4 — Health check for global config staleness

#### [MODIFY] [extension.ts](file:///x:/documents/GitHub/switchboard/src/extension.ts)

In [checkMcpConnection](file:///x:/documents/GitHub/switchboard/src/extension.ts#2931-3025), add a global config staleness check:

```typescript
// Check global Antigravity config
const globalConfigPath = getGlobalAntigravityMcpConfigPath();
if (fs.existsSync(globalConfigPath)) {
    try {
        const globalConfig = JSON.parse(fs.readFileSync(globalConfigPath, 'utf8'));
        const serverEntry = globalConfig?.mcpServers?.switchboard;
        const configuredPath = serverEntry?.args?.[0];
        if (configuredPath && !fs.existsSync(configuredPath)) {
            status.diagnostic = 'Global Antigravity MCP config points to a missing server file. Re-run Switchboard setup.';
        }
    } catch { /* non-fatal */ }
}
```

This catches the "user moved their repo" case and surfaces a clear prompt to re-run setup.

## Verification Plan

### Automated
- Unit test: `getGlobalAntigravityMcpConfigPath()` returns correct path on Windows (`C:\Users\<user>\.gemini\antigravity\mcp_config.json`) and macOS/Linux (`/home/<user>/.gemini/antigravity/mcp_config.json`).

### Manual
1. Fresh install on a clean machine — run setup, select Gemini, confirm popup appears with correct paths for THAT machine.
2. Move the workspace folder, re-open. Health check should detect stale path and prompt re-setup.
3. Run setup with Antigravity not installed — confirm error message shows, no directory is created.
4. Run setup with custom extra keys in existing [mcp_config.json](file:///x:/documents/GitHub/switchboard/mcp_config.json) — confirm they are preserved after write.

> [!IMPORTANT]
> All `args` paths in the written config **must be the user's own workspace path**, resolved at runtime. Never commit or ship any absolute path from a developer's machine into templates or defaults.
