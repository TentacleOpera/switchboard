# Resolve Anchoring Bias in Coder Agent Plan Improvement Workflows

## Goal

Update the `improve-plan.md` workflow files in both the `patrickwork` and `switchboard` workspaces to clarify that the `## Goal` section is allowed and expected to contain detailed problem and root-cause analysis, avoiding the anchoring constraint that causes agents to strip out critical context.

**Problem & Root-Cause Analysis:**
The `improve-plan.md` workflow defines the `## Goal` required section as "1-2 sentences summarizing the objective." This specific instruction creates an anchoring bias: when coder agents execute `/improve-plan`, they resolve a conflict between this narrow instruction and the general "CONTENT PRESERVATION" constraint by following the more specific "1-2 sentences" guidance, truncating rich problem analysis down to a brief summary. The result is that critical background context, root-cause documentation, and problem framing are lost during plan improvement cycles. Although the `AGENTS.md` "Plan Authoring & Problem Analysis Protocol" already instructs agents to preserve problem analysis in the Goal section, the workflow-level "1-2 sentences" anchor overrides this higher-level guidance because it is more specific and appears inline in the execution steps. The fix eliminates the contradiction by making the Goal line self-consistent with content preservation.
