# Redesign and Refine Switchboard Research Tab

## Metadata
**Complexity:** 3
**Tags:** frontend, ui, ux, refactor

## Goal
The Switchboard Research tab has been redesigned but contains several usability and structural deficiencies:
1. It lacks an introductory explanation of what it does.
2. Step 1 has too many fields (redundant 'Context & Specifics' and 'Research Depth' controls).
3. The prompt template and research persona are hard-coded to technical/software engineering topics.
4. The document import folder selection is not synchronized with the user-defined folders from the local docs tab, does not persist user selection, lacks folder management, and doesn't handle empty folder lists gracefully.

This plan aims to simplify the UI, generalize prompt templates, align document import destination folders with user configuration, and integrate folder management directly into the Research tab.

## Proposed Changes

### Research Tab Frontend & UI

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)
- Add introductory text above Step 1 planning card:
  > Switchboard Research offers an easy way to use Google AI Studio to quickly perform high quality research, and get the results back into your workspace as a fully formatted document.
- Update Step 1 description to: `"Send instructions to IDE-agent to draft a detailed research prompt for you."`
- Remove the `Context & Specifics (Optional)` label and textarea.
- Remove the `Research Depth` label and select dropdown.
- In Step 3:
  - Add a warning element `#research-no-folders-warning` showing `no destination folders set` if no folders are configured.
  - Wrap the `#research-destination-folder` select and add a `MANAGE FOLDERS` button (`#btn-manage-research-folders`) next to it to launch the local docs folder management modal.

#### [MODIFY] [planning.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.js)
- Maintain user's last selected folder preference in webview state (`state.lastResearchFolder`) and persist it via `vscode.setState` when changed.
- On `localFoldersListed` message:
  - Toggle visibility of `#research-no-folders-warning` depending on whether `msg.paths` is empty.
  - Populate `#research-destination-folder` options and pre-select `state.lastResearchFolder` if it exists in the list.
- In `draftWithAnalyst` click handler:
  - Read only the topic input.
  - Omit context parameters.
  - Hardcode `depth: 'deep'` (representing the deep 50+ sources option).
- Bind click handler for `#btn-manage-research-folders` to call `openFoldersModal()`.
- Generalize `generateResearchPrompt()` template:
  - Remove reference to `depthInput` and `contextInput`. Hardcode complexity/depth to `'deep'`.
  - Replace technical jargon default value (`"General software engineering and technology"`) with `"General subject matter research"`.
  - Replace target reader `"software engineer"` with `"domain practitioner"`.
  - Update `DECISION THIS FEEDS` to `"Recommended action plan — end with a recommended default."`.
  - Generalize `SCOPE` and `OUTPUT` prompts to focus on general subject matter research instead of purely technology stacks.

### Agent Skills & Persona Prompting

#### [MODIFY] [draft_research_prompt.md](file:///Users/patrickvuleta/Documents/GitHub/switchboard/.agent/skills/draft_research_prompt.md)
- Update parameters and instructions to note that if `Context` is empty or not provided, the Analyst agent must infer and define the appropriate domain/context from the `Topic` themselves.

## Verification Plan

### Automated Tests
Since this is primarily a Webview UI change, there are no existing Jest or Mocha unit tests for this specific panel. Compilation check will be run using:
```bash
npm run vscode:prepublish
```

### Manual Verification
1. Open the Switchboard panel and switch to the **Research** tab.
2. Verify overall intro text is present above Step 1.
3. Verify Step 1 contains only the "Central Question / Topic" field and that the "Context" and "Research Depth" fields are gone.
4. Verify clicking "COPY PROMPT TEMPLATE" copy prompt template text to clipboard, and check that the text has generalised language (no hardcoded "software engineer" references).
5. Verify clicking "DRAFT WITH ANALYST AGENT" drafts the prompt using depth 'deep' and the agent successfully infers context from the topic.
6. Verify folder selection:
   - If no folders are configured in Local Docs, Step 3 displays "no destination folders set".
   - Click "MANAGE FOLDERS" button, configure folders in the modal, and verify they show up in both the Local Docs tab and the Research tab Destination Folder dropdown.
   - Select a destination folder, import a research document from clipboard, and verify it imports to the selected folder.
   - Reopen the panel or switch tabs, and verify the dropdown selection is remembered.
