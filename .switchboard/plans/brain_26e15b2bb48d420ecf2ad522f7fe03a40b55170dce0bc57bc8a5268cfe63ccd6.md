# Rename Chat Workflow to Switchboard-Chat and Add Webview Copy Button

Consolidate custom/default chat workflows by renaming `chat.md` to `switchboard-chat.md` to prevent compatibility conflicts and accidental overwrites during updates. To improve developer ergonomics, we will add a dedicated "Copy Chat Workflow" button next to the collapse button in the Kanban board's controls strip, allowing users to copy the entire workflow instructions directly to their clipboard with one click.

## Goal

Rename the bundled `chat.md` workflow to `switchboard-chat.md` and add a one-click "Copy Chat Workflow" button to the Kanban controls strip that copies the raw workflow file contents to the clipboard.

- **Tags:** [frontend, workflow, UX]
- **Complexity:** 5

## Complexity Audit

### Routine
- Renaming `chat.md` to `switchboard-chat.md` (file operation, content preserved verbatim)
- Updating the `chatWorkflowPath` string literal in `KanbanProvider.ts` (line 5166)

- Adding a `strip-icon-btn` to the `controls-strip` in `kanban.html` (follows existing pattern)
- Registering a click event listener for the new button (follows existing pattern)

### Complex / Risky
- The new `copyChatWorkflow` handler reads a file from disk and writes to clipboard — must handle the case where the file doesn't exist (e.g., user hasn't run setup yet) with a user-facing error message rather than an unhandled exception.

## Edge-Case & Dependency Audit

- **Race Conditions**: None. Both copy handlers are user-triggered, single-action, fire-and-forget.
- **Security**: No secrets or credentials involved. The workflow file is a plain-text instruction document.
- **Side Effects**: The `chatCopyPrompt` handler (card-level button) copies a *dispatch prompt string* (ready to paste into an agent). The new `copyChatWorkflow` handler (strip button) copies *raw file contents*. These are intentionally different payloads for different use cases — no conflict.
- **Dependencies & Conflicts**: The `performSetup` function copies all `.agent/` files from extension source to workspace on version change. After renaming the source file, new setups will get `switchboard-chat.md`. Existing workspaces will still have `chat.md` until the legacy cleanup list removes it. The `cleanupLegacyAgentFiles` function runs during setup, so the next setup after this change will clean up the old file.

## Dependencies

- None (no cross-session dependencies)

## Adversarial Synthesis

Key risks: (1) The `/chat` trigger prefix in the `chatCopyPrompt` prompt string is hardcoded and must be updated alongside the file path — missing this creates a mismatch where the prompt references the new path but uses the old trigger. (2) Existing workspaces retain `chat.md` until setup runs — adding it to the legacy cleanup list prevents stale duplication. (3) The new `copyChatWorkflow` handler must gracefully handle a missing file. Mitigations: All three are straightforward one-line or small-block fixes with clear verification steps.

- **After the `chatCopyPrompt` case (after line 5188)**: Implement a new `copyChatWorkflow` message handler in the webview listener:
  ```typescript
  case 'copyChatWorkflow': {
      const workspaceRoot = this._resolveWorkspaceRoot(msg.workspaceRoot);
      if (!workspaceRoot) { break; }
      const workflowPath = path.join(workspaceRoot, '.agent', 'workflows', 'switchboard-chat.md');
      try {
          const content = await fs.promises.readFile(workflowPath, 'utf8');
          await vscode.env.clipboard.writeText(content);
          this._panel?.webview.postMessage({ type: 'showStatusMessage', message: 'Copied Switchboard Chat workflow to clipboard.', isError: false });
      } catch {
          vscode.window.showWarningMessage('Switchboard Chat workflow file not found. Run Switchboard Setup to scaffold it.');
      }
      break;
  }
  ```
