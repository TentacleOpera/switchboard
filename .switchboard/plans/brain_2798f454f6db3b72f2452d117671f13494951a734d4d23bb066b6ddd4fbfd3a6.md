# Replace Kanban Card Timestamps with Complexity Ratings

Update the Kanban cards to display both a context-aware complexity rating (Unknown, Low, High) and the existing "time ago" timestamp side-by-side.

## Goal
Improve the triage experience by surfacing the plan's complexity directly on the Kanban card without sacrificing the freshness indicator (timestamp).

## Proposed Changes

### Kanban Backend

#### [MODIFY] [KanbanProvider.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)
- Update `KanbanCard` interface to include `complexity: 'Unknown' | 'Low' | 'High'`.
- Update `_refreshBoard` to be fully async and parse the plan file for each card to determine complexity.
- Implement a helper `_getComplexityFromPlan(planPath: string)` that:
    - Reads the plan file.
    - Searches for the `Complexity Audit` section.
    - Checks for `Band B` tasks (non-"None" content). *Note: Must robustly handle variations like "None", "- None", "N/A", "None.", ignoring case.*
    - Returns the appropriate complexity level.

---

### Kanban Frontend

#### [MODIFY] [kanban.html](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/webview/kanban.html)
- Update `createCardHtml` to render *both* the complexity rating and the `formatTimeAgo(card.lastActivity)` timestamp within the `.card-meta` div.
- Format them with a visual separator, e.g., `Complexity: <span class="complexity-indicator">Low</span> | 1h ago`.
- Adjust styling of `.card-meta` and the new `.complexity-indicator` to color-code the complexity (e.g., Red for High, Green for Low, Gray for Unknown) while keeping the timestamp visually subdued.

## Review Feedback
- **Grumpy Review:** "You want to read and parse the contents of EVERY SINGLE PLAN FILE every time the Kanban board refreshes?! Have you lost your mind? `_refreshBoard` is going to hit the disk for 50 different files, parse arbitrary markdown text looking for 'Band B'—which is incredibly brittle by the way! What if I write '- None.' instead of 'None'? What if I write 'N/A'? Your little string parser is going to flag everything as 'High' complexity! And you're completely deleting the 'time ago' timestamp? People use that to know which plans are stale! This is an expensive, brittle UI regression!"
- **Balanced Synthesis:** "Grumpy is right that parsing raw markdown to extract state is inherently brittle and disk I/O on every refresh could scale poorly if we had hundreds of active plans. However, the number of active plans on a Kanban board is typically small enough that async disk reads during refresh won't cause noticeable latency. To mitigate the parsing brittleness, the `_getComplexityFromPlan` regex must robustly account for common 'empty' states like `None`, `- None`, `N/A`, or `None.` under the Band B section. While removing the timestamp is a trade-off, highlighting complexity provides more actionable value for delegating tasks. We will proceed, but ensure the markdown parser is resilient."
- **User Correction:** The user requested we KEEP the timestamp. There is enough space in the `.card-meta` row to display both (e.g., `Complexity: High | 1h ago`). The plan has been updated to reflect this combined approach, addressing Grumpy's concern about losing staleness visibility.
