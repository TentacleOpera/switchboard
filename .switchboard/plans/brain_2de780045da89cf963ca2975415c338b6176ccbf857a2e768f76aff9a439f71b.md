# Kanban Header Controls Alignment and Spacing Polish

## Goal

The spacing between buttons in the top controls strip of `kanban.html` is inconsistent and too large. Additionally, the height of various elements (select dropdowns, add button, and icon buttons) is mismatched, ranging from 18px to 32px, creating a disorganized layout.

To resolve this, we will:
1. Unify the height of all interactive elements in the top controls strip (`.controls-strip`) to a consistent 24px (standard VS Code toolbar height).
2. Tighten and make the spacing between all elements in the strip uniform.
3. Move the "Start Automation" (`btn-autoban`) button from the bottom row (`.kanban-sub-bar`) to the top row, placing it immediately to the right of the "Collapse Coders" (`btn-collapse-coders`) button.
4. Clean up the bottom row to only display the automation timers, timer controls, and status/toast messages.

## Metadata
**Tags:** frontend, ui, ux, bugfix

## User Review Required

No major architectural risks are associated with this change. It is purely cosmetic/layout refinement.

> [!NOTE]
> The icons are resized from 22px to 14px inside the 24px container height. This ensures they scale proportionally and match the 24px heights of the select boxes and other buttons.

## Edge-Case & Dependency Audit

- **Race Conditions**: N/A — purely CSS/HTML layout changes, no async logic affected.
- **Security**: N/A — no data handling changes.
- **Side Effects**: Moving `#btn-autoban` from `.kanban-sub-bar` to `.controls-strip` changes which CSS rules apply to it. The `.kanban-sub-bar .strip-icon-btn` rules (lines 122–165) will no longer apply; instead, `.controls-strip .strip-icon-btn` rules (lines 318–334) will apply. The JavaScript event listener (line 6572) uses `getElementById('btn-autoban')` so it will continue to work regardless of DOM position.
- **Dependencies & Conflicts**: The `updateAutobanButtonState()` function (line 4632) toggles `.is-active` class on `#btn-autoban`. The `.controls-strip .strip-icon-btn.is-active` styling (inherited from the generic `.strip-icon-btn.is-active` at line 343) should work identically. The `.kanban-sub-bar .strip-icon-btn.is-active` rules (lines 142–152) will no longer apply, but the generic rules provide equivalent visual feedback.

## Dependencies
- None

## Adversarial Synthesis

Key risks: (1) Fixed `height: 24px` on native `<select>` elements may clip content or the dropdown arrow on some platforms — mitigate by using `min-height: 24px` for selects instead of fixed height, (2) `btn-add-plan` "+" character will appear small inside a 24px container without font-size adjustment — mitigate by increasing font-size to ~16px, (3) `kanban-mode-dropdown` padding (currently 3px 6px) will create excessive vertical space with fixed 24px height — mitigate by reducing vertical padding to 2px or adjusting line-height. All risks are low-severity visual issues that can be fine-tuned after initial implementation.
