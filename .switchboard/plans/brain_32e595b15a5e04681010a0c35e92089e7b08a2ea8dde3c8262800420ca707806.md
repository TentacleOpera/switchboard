# Restricting SQL Card Move Instructions to Manual Fallbacks

This change removes SQL-based database update instructions from the standard conversational routing in `AGENTS.md`. Instead, agents are instructed to rely on the automated extension triggers for standard transitions, reserving direct script/SQL execution for a manual fallback/override skill (`kanban_operations`) that will only be used if explicitly requested by the user.

## User Review Required

> [!IMPORTANT]
> The modification restricts `kanban_operations` and database write actions to manual user requests (fallback override). Normal agent execution will no longer run card-moving scripts/SQL queries.
