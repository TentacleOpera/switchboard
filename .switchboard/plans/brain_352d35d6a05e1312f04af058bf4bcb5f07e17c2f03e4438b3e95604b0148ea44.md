# HTML Clipboard Conversion for Rich Text Markdown Import

Enable the "Import from Clipboard" commands to natively read HTML formatting from the system clipboard and convert it into structured Markdown. This prevents rich text copied from Markdown previews, websites, or external editors from importing as a jumbled, unformatted plain-text wall.

## Goal

Provide high-fidelity import of formatted plans by querying the system clipboard for `text/html` (rich-text) formats when the webview's native `navigator.clipboard.read()` fails due to iframe sandbox security restrictions.

### Background & Root Cause
1. **Security Context**: VS Code webviews run in sandboxed iframes. Calling `navigator.clipboard.read()` to extract `text/html` throws a `SecurityError`.
2. **Fallback Failure**: When it fails, the webviews (`kanban.html`, `planning.js`) fall back to sending an import message to the backend with no text payload.
3. **Plain Text Default**: The backend falls back to `vscode.env.clipboard.readText()`. However, copying a Markdown preview puts the formatted content into `text/html`, and its plain-text representation (the fallback `text/plain`) lacks Markdown tags, resulting in a single-line or jumbled unformatted string.
4. **Resolution**: Modify the backend `importPlanFromClipboard` to run platform-specific CLI utilities (`osascript` on macOS, `powershell` on Windows, `xclip`/`xsel` on Linux) to fetch the system's HTML clipboard data, parse it via `jsdom` (already in `package.json` dependencies), and convert it into clean Markdown before creating the plan cards.

### Scope Clarification
The kanban.html webview (lines 4296-4324) already attempts `navigator.clipboard.read()` for `text/html` and converts it client-side via `convertHtmlToMarkdown()` (lines 5104-5204). The backend HTML clipboard read proposed here is a **fallback** that fires only when:
- The webview's `navigator.clipboard.read()` throws a `SecurityError` (sandbox restriction), OR
- The import is triggered from the Planning Panel (`PlanningPanelProvider._handleImportPlansFromClipboard`, line 2852), which delegates directly to the command with no `markdownText` argument and never attempts HTML reading.

Both entry points converge on `importPlanFromClipboard(markdownText?)` in `TaskViewerProvider.ts` (line 15106). When `markdownText` is `undefined`, the backend will now attempt HTML clipboard reading before falling back to plain text.

## Adversarial Synthesis

Key risks: macOS `osascript` hex parsing is fragile and must handle malformed output gracefully; Linux `xclip`/`xsel` absence must silently fall back to plain text; CLI clipboard processes must have timeout guards to prevent hanging the import flow. #### 2. Add `readClipboardHtml(): Promise<string | null>` private helper

**Context:** Platform-specific CLI utilities to read HTML clipboard data. Must use `execFile` (not `exec`) for security. Must include timeout guards.

**Logic:**
- Branch on `process.platform`:
  - **macOS (`darwin`)**: Run `osascript -e 'the clipboard as «class HTML»'` via `execFile`. Parse the output: if it contains `«data HTML` followed by hex bytes and `»`, extract the hex substring, decode hex pairs to bytes, and return the resulting UTF-8 string. If the output doesn't match the expected format (e.g., no HTML on clipboard), return `null`.
  - **Windows (`win32`)**: Run `powershell -NoProfile -Command "Get-Clipboard -TextFormatType Html"` via `execFile`. Extract the fragment between `<!--StartFragment-->` and `<!--EndFragment-->` if present; otherwise return the full output. If the output is empty or the command fails, return `null`.
  - **Linux (`linux`)**: Try `xclip -selection clipboard -t text/html -o` first. If `xclip` is not found (ENOENT), try `xsel --clipboard --output --html`. If neither is available, return `null` (no error shown to user).
- All `execFile` calls use a **5-second timeout** to prevent hanging.
- All calls wrapped in try/catch returning `null` on any failure.

**Implementation:**
```typescript
private async _readClipboardHtml(): Promise<string | null> {
    const execFileAsync = promisify(cp.execFile);
    const TIMEOUT_MS = 5000;

    try {
        switch (process.platform) {
            case 'darwin': {
                const { stdout } = await execFileAsync('osascript', [
                    '-e', 'the clipboard as «class HTML»'
                ], { timeout: TIMEOUT_MS, encoding: 'utf8' });
                // Parse «data HTML3C703E...» format
                const hexMatch = stdout.match(/«data HTML([0-9A-Fa-f]+)»/);
                if (!hexMatch) { return null; }
                const hex = hexMatch[1];
                // Decode hex pairs to UTF-8 string
                const bytes = Buffer.from(hex, 'hex');
                return bytes.toString('utf8');
            }
            case 'win32': {
                const { stdout } = await execFileAsync('powershell', [
                    '-NoProfile', '-Command', 'Get-Clipboard -TextFormatType Html'
                ], { timeout: TIMEOUT_MS, encoding: 'utf8' });
                if (!stdout || !stdout.trim()) { return null; }
                // Extract fragment between StartFragment/EndFragment if present
                const fragMatch = stdout.match(/<!--StartFragment-->([\s\S]*?)<!--EndFragment-->/);
                return fragMatch ? fragMatch[1] : stdout;
            }
            case 'linux': {
                // Try xclip first, then xsel
                try {
                    const { stdout } = await execFileAsync('xclip', [
                        '-selection', 'clipboard', '-t', 'text/html', '-o'
                    ], { timeout: TIMEOUT_MS, encoding: 'utf8' });
                    if (stdout && stdout.trim()) { return stdout; }
                } catch { /* xclip not available or failed */ }
                try {
                    const { stdout } = await execFileAsync('xsel', [
                        '--clipboard', '--output', '--html'
                    ], { timeout: TIMEOUT_MS, encoding: 'utf8' });
                    if (stdout && stdout.trim()) { return stdout; }
                } catch { /* xsel not available or failed */ }
                return null;
            }
            default:
                return null;
        }
    } catch {
        return null;
    }
}
```

**Edge Cases:**
- `osascript` returns empty string or non-HTML content → hex match fails → returns `null` → falls back to plain text
- `powershell` `Get-Clipboard` not available on older Windows → command fails → catch returns `null`
- Linux with neither `xclip` nor `xsel` → both ENOENT → returns `null` silently
- Clipboard daemon hung → 5s timeout fires → catch returns `null`
- HTML contains `<!--StartFragment-->` but no `<!--EndFragment-->` → regex fails → returns full stdout

#### 3. Add `convertHtmlToMarkdown(html: string): string` private helper

**Context:** Port of the webview's `convertHtmlToMarkdown()` (kanban.html lines 5104-5204) using `jsdom` instead of browser `DOMParser`. Handles the same set of HTML tags: `H1-H6`, `P`, `DIV`, `BR`, `STRONG`, `B`, `EM`, `I`, `CODE`, `PRE`, `A`, `LI`, `UL`, `OL`.

**Logic:**
- Parse HTML string via `new JSDOM(html)` and extract `dom.window.document.body`.
- Walk the DOM tree recursively, converting each element to its Markdown equivalent.
- Preserve the same tag-to-Markdown mapping as the webview version:
  - `H1-H6` → `# ... ######`
  - `P`, `DIV` → paragraph with blank lines
  - `BR` → newline
  - `STRONG`, `B` → `**bold**`
  - `EM`, `I` → `*italic*`
  - `CODE` (inside PRE) → raw text; (outside PRE) → `` `inline` ``
  - `PRE` → fenced code block with optional language
  - `A` → `[text](href)`
  - `LI` → indented `- ` or `1. ` based on parent list type
  - `UL`, `OL` → list container with depth tracking
- Collapse 3+ consecutive newlines to 2.
- Trim final result.

**Implementation:**
```typescript
private _convertHtmlToMarkdown(html: string): string {
    const dom = new JSDOM(html);
    const doc = dom.window.document;

    function cleanText(text: string): string {
        return text.replace(/\u00a0/g, ' ');
    }

    function walk(node: Node, listDepth = 0, isPre = false, listType: string | null = null): string {
        if (node.nodeType === dom.window.Node.TEXT_NODE) {
            return cleanText(node.textContent || '');
        }
        if (node.nodeType !== dom.window.Node.ELEMENT_NODE) {
            return '';
        }

        const tagName = (node as Element).tagName.toUpperCase();
        let childrenMarkdown = '';

        let nextListDepth = listDepth;
        let nextListType = listType;
        if (tagName === 'UL' || tagName === 'OL') {
            nextListDepth = listDepth + 1;
            nextListType = tagName;
        }

        for (let i = 0; i < node.childNodes.length; i++) {
            childrenMarkdown += walk(node.childNodes[i], nextListDepth, isPre || tagName === 'PRE', nextListType);
        }

        switch (tagName) {
            case 'H1': return `\n\n# ${childrenMarkdown.trim()}\n\n`;
            case 'H2': return `\n\n## ${childrenMarkdown.trim()}\n\n`;
            case 'H3': return `\n\n### ${childrenMarkdown.trim()}\n\n`;
            case 'H4': return `\n\n#### ${childrenMarkdown.trim()}\n\n`;
            case 'H5': return `\n\n##### ${childrenMarkdown.trim()}\n\n`;
            case 'H6': return `\n\n###### ${childrenMarkdown.trim()}\n\n`;
            case 'P':
            case 'DIV':
                return `\n\n${childrenMarkdown.trim()}\n\n`;
            case 'BR': return '\n';
            case 'STRONG':
            case 'B':
                return `**${childrenMarkdown.trim()}**`;
            case 'EM':
            case 'I':
                return `*${childrenMarkdown.trim()}*`;
            case 'CODE':
                if (isPre) { return childrenMarkdown; }
                return `\`${childrenMarkdown.trim()}\``;
            case 'PRE': {
                let lang = '';
                const classAttr = (node as Element).getAttribute('class') || '';
                const langMatch = classAttr.match(/language-(\w+)/) || classAttr.match(/lang-(\w+)/);
                if (langMatch) {
                    lang = langMatch[1];
                } else {
                    const codeChild = (node as Element).querySelector('code');
                    if (codeChild) {
                        const codeClass = codeChild.getAttribute('class') || '';
                        const codeLangMatch = codeClass.match(/language-(\w+)/) || codeClass.match(/lang-(\w+)/);
                        if (codeLangMatch) { lang = codeLangMatch[1]; }
                    }
                }
                return `\n\n\`\`\`${lang}\n${childrenMarkdown.trim()}\n\`\`\`\n\n`;
            }
            case 'A': {
                const href = (node as Element).getAttribute('href') || '';
                const text = childrenMarkdown.trim() || href;
                return `[${text}](${href})`;
            }
            case 'LI': {
                const indent = '  '.repeat(Math.max(0, listDepth - 1));
                const prefix = listType === 'OL' ? '1. ' : '- ';
                return `\n${indent}${prefix}${childrenMarkdown.trim()}`;
            }
            case 'UL':
            case 'OL':
                return `\n${childrenMarkdown}\n`;
            default:
                return childrenMarkdown;
        }
    }

    let result = walk(doc.body);
    result = result.replace(/\n{3,}/g, '\n\n');
    return result.trim();
}
```

**Edge Cases:**
- Malformed HTML → JSDOM's error-tolerant parser handles it (same as browser DOMParser)
- Empty HTML string → `doc.body` has no children → returns empty string → caller falls back to plain text
- Deeply nested lists → `listDepth` tracking ensures correct indentation
- `<code>` inside `<pre>` → `isPre` flag prevents double-backtick wrapping
