# Fix Clipboard Import Preamble Stub Issue

The Switchboard "Import plans from clipboard" functionality incorrectly treats conversational LLM preamble text (e.g., "Here are the separated plans as requested") as a standalone plan when doing multi-plan imports. Because this preamble lacks standard Switchboard plan headers like `## Goal`, the `_createInitiatedPlan` method injects an empty plan template around it, resulting in the creation of a garbage "Imported Plan X" stub file.

## Goal

Prevent the creation of blank stub plans caused by conversational text appearing before the first valid `### PLAN [X] START` clipboard separator.

## Metadata
**Tags:** backend, bugfix
**Complexity:** 3

## User Review Required

> [!WARNING]
> This change modifies how clipboard imports are parsed. If a user tries to import a clipboard where the very first intended plan does *not* possess a multi-plan separator, but subsequent plans do, the first un-marked section will be ignored. This enforces strict adherence to the separator syntax for multi-plan imports.

- **Security:** No new input surfaces. Clipboard text is already size-gated at 200 KB (line 9109). The preamble is silently dropped, not stored.
- **Side Effects:** A clipboard where the first intended plan genuinely lacks a separator marker will have that first section silently discarded. The existing `> [!WARNING]` in User Review Required documents this trade-off.
- **Dependencies & Conflicts:** No conflicts with other active Kanban plans. At MINIMUM you should count discarded preamble chunks and surface it in the import summary."*

### Balanced Response
The Grumpy critique raises two valid points, both addressed below:
1. **Silent discard with no feedback:** The implementation already logs to console, but we will enhance the import summary message to include a note when preamble text was skipped. This gives the user a visible signal without adding a blocking dialog.
2. **Large preamble performance:** The existing 200KB clipboard cap (line 9109) already bounds total input. After `.split()`, preamble chunks are just skipped via an `if` branch — no allocation or processing cost. Performance is not a concern.

The zero-separator case is irrelevant here: when no separators are detected, the code takes the single-plan import path (line 9118-9136) and never enters `_importMultiplePlansFromClipboard`.

### Clipboard Preamble Guard

#### [MODIFY] `src/services/TaskViewerProvider.ts`
- **Context:** The `_importMultiplePlansFromClipboard` method (line 9142) splits clipboard text by separator markers and accumulates content chunks into plan objects. Currently, content chunks appearing *before* the first marker create a spurious `currentPlan` accumulator (lines 9171-9174), which eventually becomes a garbage stub plan.
- **Logic:**
  1. Invert the conditional: only push content to `currentPlan.lines` when `currentPlan` is already initialized (i.e., a marker has been seen).
  2. When `currentPlan` is `null`, silently skip the chunk and increment a `preambleSkipped` counter.
  3. After the import loop, if `preambleSkipped > 0`, append a note to the success summary message.
- **Implementation:**

Replace the `else` block at lines 9169–9176 with:

```typescript
            } else {
                // Content chunk — only accumulate after the first marker has been seen
                if (currentPlan) {
                    currentPlan.lines.push(part);
                } else {
                    // Preamble before first plan marker — discard silently
                    preambleSkipped++;
                    console.log('[TaskViewerProvider] Ignoring preamble chunk before first plan marker');
                }
            }
```

Additionally, declare the counter at line 9153 (after `let currentPlan`):

```typescript
        let preambleSkipped = 0;
```

And update the success summary (lines 9225–9229) to surface preamble info:

```typescript
        if (importedTitles.length > 0) {
            const preambleNote = preambleSkipped > 0 ? ` (${preambleSkipped} preamble chunk(s) skipped)` : '';
            const summary = importedTitles.length === 1
                ? `Imported plan: ${importedTitles[0]}${preambleNote}`
                : `Imported ${importedTitles.length} plans: ${importedTitles.slice(0, 3).join(', ')}${importedTitles.length > 3 ? '...' : ''}${preambleNote}`;
            vscode.window.showInformationMessage(summary);
        }
```

- **Edge Cases Handled:** Silent data loss is mitigated by the preamble counter being surfaced in the user-visible import summary. The zero-separator case never enters this code path.
