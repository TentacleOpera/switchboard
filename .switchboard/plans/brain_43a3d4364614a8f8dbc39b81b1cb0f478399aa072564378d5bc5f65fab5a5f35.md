# Fix Verification Step Parsing for Spelling Typos and Direct Headers

In the Kanban view UAT ('test') tab, plan verification steps are not parsed under headers with spelling typos (e.g., "Verificaton Plan", "Verificaiton Steps") or when steps are written directly under a "## Verification Plan" header without manual subheadings. This plan addresses both issues.

## User Review Required

> [!NOTE]
> The proposed fix modifies the regex matching for UAT verification plan sections in the Kanban provider to be typo-tolerant and support direct lists. It preserves all existing parsing rules and tests.

## Proposed Changes

### Switchboard Core Services

#### [MODIFY] [KanbanProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)

- Update the Pattern 3 regex in `_parseVerificationSteps` to match `Verific` followed by any letters, followed by either `Plan` or `Steps` (case-insensitive).
- Initialize `inManualStepsSection` to `true` by default in Pattern 3 so that any steps or checklists placed directly under the verification header (before any other markdown subheadings are encountered) are correctly extracted.

```diff
-            const verificationPlanMatch = content.match(/##\s*Verification\s+Plan\s*\n([\s\S]*?)(?=\n##|$)/i);
+            const verificationPlanMatch = content.match(/##\s*Verific[a-z]*\s+(?:Plan|Steps)\s*\n([\s\S]*?)(?=\n##|$)/i);
             if (verificationPlanMatch) {
                 const lines = verificationPlanMatch[1].split('\n');
-                let inManualStepsSection = false;
+                let inManualStepsSection = true;
```
