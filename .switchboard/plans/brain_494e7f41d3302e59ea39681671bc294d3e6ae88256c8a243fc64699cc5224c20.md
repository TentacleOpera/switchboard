# Fixing Webview Text Spacing and Markdown Code Highlight Escaping

## Metadata
**Complexity:** 2
**Tags:** ui, ux, bugfix, refactor, frontend

This plan addresses two issues in the Switchboard planning webview:
1. **Typography & Layout Spacing**: The previewed text is cramped against borders, and margins between paragraphs, lists, and headers are too tight.
2. **Escaped Backticks Bug**: Escaped backticks (e.g. `\`` inside template literals in markdown) are matched by the inline code block parser, resulting in unclosed/mismatched code tags that highlight the remainder of the document.

## User Review Required

> [!NOTE]
> All changes are restricted to the webview frontend code (`planning.html` and `planning.js`). There are no backend API, schema, or command changes.

## Open Questions

None. The issues have clear causes and simple, clean fixes.

## Proposed Changes

### Switchboard Webview

---

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)

Adjust CSS rules to improve layout breathing room and vertical text rhythm:

- Increase margins on headers (`h1`-`h6`) to `24px` top and `12px` bottom.
- Increase paragraph (`p`) margin-bottom to `16px`.
- Increase list (`ul`, `ol`) margin-bottom to `16px`.
- Increase list item (`li`) margin-bottom to `6px`.
- Add `padding: 8px 16px` to `#kanban-preview-pane` to prevent text from touching the edge of the panel borders.

#### [MODIFY] [planning.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.js)

Enhance `renderMarkdown` function to handle escaped backticks properly:

1. Replace `\\`` with `__ESCAPED_BACKTICK__` at the very beginning of the `renderMarkdown` function.
2. Allow normal markdown replacements to run (which will now correctly find matching unescaped backticks).
3. At the end of `renderMarkdown`, restore `__ESCAPED_BACKTICK__` tokens:
   - Within `<code>` or `<pre>` tags, restore to `\\`` (preserving the backslash for code accuracy).
   - Outside code tags, restore to `` ` `` (removing the backslash).
