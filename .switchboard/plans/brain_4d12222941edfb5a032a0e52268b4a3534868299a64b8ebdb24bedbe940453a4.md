# Make Copy Prompt Button Instant with Optimistic UI

## Goal

Make the "Copy Prompt" button on Kanban cards respond instantly (<20ms) by adding optimistic UI feedback and deferring slow backend operations, while fixing the broken `copyPlanLinkResult` message that currently sends no card identifier back to the frontend.

## Metadata

- **Tags:** [frontend, backend, performance, workflow]
- **Complexity:** 5
- **Status:** improved

## User Review Required

> [!NOTE]
> Since `sessionId` is deprecated as a unique key in favor of `plan_file` / `planId`, we will structure our message payload to accept both `planId` and `sessionId` on the backend, and target buttons in the frontend using `[data-plan-id="${msg.planId}"], [data-session="${msg.sessionId}"]`. This guarantees compatibility with both modern file-based plans and older legacy plans.

> [!WARNING]
> The current `copyPlanLinkResult` message (line 12623-12627) sends **neither** `sessionId` nor `planId` back to the webview. This means the result handler at `kanban.html:4724` (`document.querySelector('.card-btn.copy[data-session="${msg.sessionId}"]')`) **always fails to find the button** because `msg.sessionId` is `undefined`. The "Copied!" visual feedback is currently broken for ALL cards — not just new file-based ones. This is a pre-existing bug that this plan fixes.

## Complexity Audit

### Routine
- Add `planId` and `sessionId` to the `copyPlanLinkResult` message payload (1 line change in 2 places — success and error paths)
- Update the `copyPlanLinkResult` handler in `kanban.html` to use `planId`-first selector with `sessionId` fallback
- Add optimistic "Copied!" visual feedback on button click (before the message is even sent)

### Complex / Risky
- **Fire-and-forget column advance**: Moving `_applyManualKanbanColumnChange`, `queueIntegrationSyncForSession`, `_recordDispatchIdentity`, `_updateSessionRunSheet`, and `_scheduleSidebarKanbanRefresh` into an un-awaited `void` block. The current code deliberately awaits these to include an `advanceWarning` in the result message. We must ensure advance failures are still surfaced (via `console.error` + non-blocking `vscode.window.showWarningMessage`) even though the copy result is sent immediately.
- **`planId`-based resolution path**: The `copyPlanLink` message handler at line 8010-8013 currently guards on `data.sessionId` only (`if (data.sessionId)`). For newer file-based plans with empty `sessionId`, the handler skips entirely. We need a fallback that resolves plan context from `planId` when `sessionId` is empty/missing, which requires either extending `_resolvePlanContextForSession` or adding a parallel `_resolvePlanContextForPlanId` method.

## Edge-Case & Dependency Audit

- **Race Conditions**: If the user clicks "Copy Prompt" on the same card twice rapidly, the second click could fire while the first is still processing. Mitigation: the `btn.disabled = true` on click already prevents this. The optimistic UI should also set `disabled` immediately.
- **Security**: No new security surface — clipboard write and message passing are existing patterns.
- **Side Effects**: Fire-and-forget column advance means the card may not have moved columns by the time the "Copied!" feedback fades. This is acceptable — the advance is a background side-effect, not part of the copy action itself.
- **Dependencies & Conflicts**: The `_resolvePlanContextForSession` method (line 11867) depends on `sessionId` to do a DB lookup via `db.getPlanBySessionId(sessionId)`. For `planId`-based resolution, we need `db.getPlanByPlanFile` or equivalent. The `SessionActionLog` (line 73-76) already demonstrates a dual-lookup pattern: `db.getPlanByPlanFile(sessionIdOrPlanFile, workspaceId)` with fallback to `db.getPlanBySessionId`.

## Dependencies

- `sess_copyPlanLinkRefactor — planId resolution path in _handleCopyPlanLink`

## Adversarial Synthesis

Key risks: (1) Fire-and-forget column advance loses the `advanceWarning` feedback that the current code was carefully designed to provide — failures become silent unless we add console logging + non-blocking notifications. (2) The `planId`-based resolution requires a DB lookup path that doesn't currently exist in `_handleCopyPlanLink` — we need to verify `KanbanDatabase.getPlanByPlanFile` exists and works. Mitigations: log all advance failures to console with `console.error`; use `vscode.window.showWarningMessage` (non-blocking) for advance failures; mirror the dual-lookup pattern already used in `SessionActionLog`.

**Step 3: Add `planId`-based resolution fallback (after line 12509)**

If `_resolvePlanContextForSession` fails or `sessionId` is empty, try resolving via `planId`:
```typescript
let planFileAbsolute: string;
let topic: string;
let resolvedWorkspaceRoot: string;
try {
    ({ planFileAbsolute, topic, workspaceRoot: resolvedWorkspaceRoot } = await this._resolvePlanContextForSession(sessionId, workspaceRoot));
} catch {
    if (!planId) throw new Error('No plan file associated with this session.');
    // Fallback: resolve via planId (relative plan file path)
    const resolvedRoot = workspaceRoot ? this._resolveWorkspaceRoot(workspaceRoot) : await this._resolveWorkspaceRootForSession(sessionId);
    if (!resolvedRoot) throw new Error('No workspace folder found.');
    const db = await this._getKanbanDb(resolvedRoot);
    const record = planId && db ? await db.getPlanByPlanFile(planId, await db.getWorkspaceId() || '') : null;
    if (!record) throw new Error('No plan file associated with this session.');
    planFileAbsolute = path.isAbsolute(record.planFile) ? path.resolve(record.planFile) : path.resolve(resolvedRoot, record.planFile);
    topic = record.topic || '';
    resolvedWorkspaceRoot = resolvedRoot;
}
```

void (async () => {
    try {
        let effectiveColumn = column || '';
        let planRecord: KanbanPlanRecord | null = null;
        const db = await this._getKanbanDb(resolvedWorkspaceRoot);
        if (db) {
            planRecord = await db.getPlanBySessionId(sessionId);

                effectiveColumn = planRecord.kanbanColumn;
            }

        effectiveColumn = this._normalizeLegacyKanbanColumn(effectiveColumn || 'CREATED');

        const customAgents = await this.getCustomAgents(resolvedWorkspaceRoot);

        if (effectiveColumn === 'PLAN REVIEWED' && this._kanbanProvider) {
            const complexity = await this._kanbanProvider.getComplexityFromPlan(resolvedWorkspaceRoot, planFileAbsolute);

            : effectiveColumn === 'PLAN REVIEWED'

document.querySelectorAll('.card-btn.copy').forEach(btn => {
    btn.addEventListener('click', () => {
        btn.disabled = true;
        // Optimistic UI: show "Copied!" immediately
        const originalText = btn.textContent;
        const originalBg = btn.style.backgroundColor;
        const originalColor = btn.style.color;
        btn.textContent = 'Copied!';
        btn.style.backgroundColor = 'var(--vscode-testing-iconPassed, #73c991)';
        btn.style.color = 'var(--vscode-editor-background, #1e1e1e)';
        // Store original state for potential reversion on error
        btn.dataset._origText = originalText;
        btn.dataset._origBg = originalBg || '';
        btn.dataset._origColor = originalColor || '';
        const column = btn.dataset.column || btn.closest('.kanban-column')?.dataset?.column;
        postKanbanMessage({ type: 'copyPlanLink', sessionId: btn.dataset.session || '', planId: btn.dataset.planId || '', column, workspaceRoot: btn.dataset.workspaceRoot });
    });
});

**Step 2: Fix `copyPlanLinkResult` handler to use `planId`-first selector (replace lines 4723-4743)**

```javascript
case 'copyPlanLinkResult':
    // Use planId as primary key (modern), sessionId as fallback (legacy)
    const btn = msg.planId
        ? document.querySelector(`.card-btn.copy[data-plan-id="${msg.planId}"]`)
        : document.querySelector(`.card-btn.copy[data-session="${msg.sessionId}"]`);
    if (btn) {
        if (msg.success) {
            // Button already shows "Copied!" from optimistic UI — just schedule revert
            setTimeout(() => {
                btn.textContent = btn.dataset._origText || 'Copy Prompt';
                btn.style.backgroundColor = btn.dataset._origBg || '';
                btn.style.color = btn.dataset._origColor || '';
                btn.disabled = false;
                delete btn.dataset._origText;
                delete btn.dataset._origBg;
                delete btn.dataset._origColor;
            }, 1500);
        } else {
            // Error: revert optimistic UI immediately
            btn.textContent = btn.dataset._origText || 'Copy Prompt';
            btn.style.backgroundColor = btn.dataset._origBg || '';
            btn.style.color = btn.dataset._origColor || '';
            btn.disabled = false;
            delete btn.dataset._origText;
            delete btn.dataset._origBg;
            delete btn.dataset._origColor;
        }

```

- `msg.planId` is empty string (legacy card): falls back to `data-session` selector
