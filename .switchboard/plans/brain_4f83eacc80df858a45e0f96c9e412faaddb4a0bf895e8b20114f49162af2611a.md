# Reposition Sidebar Collapse Button in Planning Panel

Reposition the left sidebar collapse button (`.sidebar-toggle-btn`) in `planning.html` to prevent it from taking up a full row of vertical space in the expanded state, which pushes sidebar content down.

## Goal Description
In the planning panel (`planning.html`), the sidebar toggle row (`.sidebar-toggle-row`) is part of the normal document layout flow. This pushes the primary headers (like "DOCS" or "NOTION") and list contents down, creating unnecessary vertical empty space.

To resolve this, we will absolutely position the toggle button at the top-right of the sidebar when expanded. This floats the button out of the normal layout flow, allowing headers to sit at the top of the sidebar on the same visual line as the button. When collapsed, the button will revert to normal block positioning so that it remains centered within the narrow sidebar column.

## User Review Required
- **Visual Alignment:** The header text ("DOCS", "NOTION") and any reload/status buttons will move up to the first line, with the toggle button `«` appearing on the far right of the same line.

### Remaining Risks

- The `padding-right: 36px` is a fixed value. If the toggle button styling changes (e.g., wider button, different font size), this padding may need adjustment. A more robust solution would use CSS custom properties or calc(), but the current fixed value is sufficient for the existing button dimensions.
- The `.empty-state` text (`text-align: center`) will be slightly left-of-center due to asymmetric padding. This is acceptable since the toggle button visually occupies the right space, but could be refined with `margin-right: -24px` on `.empty-state` if visual precision is desired.
