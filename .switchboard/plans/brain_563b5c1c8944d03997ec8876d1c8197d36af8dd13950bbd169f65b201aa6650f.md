# Fix Planning Webview Cyber Theme Grid Backgrounds

## Goal
The goal of this task is to ensure that the **Design System** and **HTML Previews** tabs inside the Switchboard Planning webview (`planning.html`) correctly render the cyber grid background and glassmorphism styling when the Cyber theme is active, matching the visual style of the other tabs.

### Problem Analysis & Root Cause
1. **Missing CSS Selectors**: The Cyber theme styling overrides (`.cyber-theme-enabled` rules) do not include selectors for the Design System and HTML Previews tabs' preview containers (`#preview-pane-design`, `#preview-pane-html`) and the Design System sidebar tree pane (`#tree-pane-design`).
2. **Solid Inline Backgrounds**: Both `#preview-pane-design` and `#preview-pane-html` have inline `background: var(--panel-bg)` styles. These solid backgrounds block the underlying body grid background from showing through.
3. **Solid Content Wrapper**: The content wrapper `#design-content` lacks a Cyber theme override to make its background transparent, unlike `#local-content`, `#online-content`, and `#html-preview-content`.

- **Dependencies & Conflicts**: Base CSS rules for `#preview-pane-design` and `#preview-pane-html` must be placed in the base (non-Cyber) section of the stylesheet, *before* the `.cyber-theme-enabled` media block, to ensure Cyber theme rules correctly override them via higher specificity. The `#html-preview-frame` iframe (line 2508) retains its own `background: var(--panel-bg)` inline style — this is intentional and out of scope (iframe content needs its own opaque background).

## Proposed Changes

### Webview UI

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)

- Remove `background: var(--panel-bg);` from the inline `style` attribute of:
  - `#preview-pane-design` (line 2470) — preserve `flex: 1; width: 100%; box-sizing: border-box; overflow: auto; height: 100%;`
  - `#preview-pane-html` (line 2496) — preserve `display: flex; flex-direction: column; overflow: hidden; height: 100%;`

  ```css
  }

  }
  ```

  ```diff
          /* Local and online content views */
  ```
