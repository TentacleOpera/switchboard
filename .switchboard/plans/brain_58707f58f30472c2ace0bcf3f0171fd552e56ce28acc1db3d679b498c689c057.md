# Fix: ID Scoping & Rescue Brain Plans

## Goal
Repair a two-headed regression: (1) DB-backed registry entries are keyed with the `antigravity_`-prefixed `sessionId` rather than the raw SHA-256 hash, causing `_isPlanInRegistry` (which looks up by raw planId) to always miss, and (2) `_migrateLegacyToRegistry` skips all `brainSourcePath` runsheets, so brain plans are never seeded into the registry on a clean workspace. The combined effect causes `_reconcileAntigravityPlanMirrors` to quarantine every local mirror into `archive/orphan_plans/` on extension startup.

## Metadata
**Tags:** backend, bugfix
**Complexity:** 6

## User Review Required

> [!IMPORTANT]
> This fix will perform a **Salvage Migration**. It will scan your active session logs and mirrors. If it finds brain plans that are currently unowned, it will automatically register them to this workspace in the Kanban DB so they reappear on your Kanban board.

> [!NOTE]
> The orphan rescue step moves files **back** from `.switchboard/archive/orphan_plans/` and `.switchboard/archive/orphan_sessions/` to the active staging area. This is destructive only in the sense that the archive snapshot for those entries is removed; the plans themselves are restored.

## Complexity Audit

### Routine
- Strip `antigravity_` prefix when keying in-memory `_planRegistry.entries` after DB load (one-liner in `_loadPlanRegistry`).
- Add `brainSourcePath`-keyed migration path inside `_migrateLegacyToRegistry` (parallel to the existing `local` branch).
- Emit `console.log` entries to confirm fix applied on startup.

### Complex / Risky
- **Orphan Salvage inside `_reconcileAntigravityPlanMirrors`**: Reading from and writing to the archive + staging directories concurrently with the reconciler's own loop creates ordering hazards. The salvage pass must complete and update `activeMirrorNames` **before** the reconciler's final staging sweep (lines 6450-6460) runs, or salvaged files will be immediately re-quarantined in the same tick.
- **DB Upsert de-dup vs. existing rows**: `_registerPlan` checks `existing.planId !== entry.planId` before deleting; if the original `antigravity_`-prefixed row still exists in the DB, the new raw-hash-keyed upsert will insert a second row rather than update the first. The old row must be explicitly deleted.
- **`_migrateLegacyToRegistry` early-exit guard** (`if (existing.length > 0) return`): brain plans already quarantined will not be picked up because the DB already has *other* plans, causing the early exit to fire. A two-phase migration (local first, then brain rescue) is needed, or the guard must be scoped to local plans only.

## Edge-Case & Dependency Audit

- **Race Conditions:** `_reconcileAntigravityPlanMirrors` is called by `_syncFilesAndRefreshRunSheets`, which fires on extension startup before `_ownershipInitPromise` necessarily resolves—only gated by `.then()` on `_ensureOwnershipRegistryInitialized`. The salvage sub-step runs inside the same async chain, so it safely follows registry init. No additional mutex is required.
- **Security:** Orphan salvage reads file content from the archive directory. Ensure `brainSourcePath` read from runsheets is validated with `path.resolve` before use (already performed in `_isPlanEligibleForWorkspace` via `this._getStablePath`).
- **Side Effects:** Writing salvaged `brain_*.md` files back to `.switchboard/plans/` will trigger `_planWatcher` / `_brainWatcher` change events. `_recentMirrorWrites` TTL suppression must be applied for each salvaged path to prevent a re-reconcile loop.
- **Dependencies & Conflicts:**
  - `brain_11d10e198...md` (already present on disk) — specifically mentioned as the failing plan; this fix directly targets it.
  - `make_integration_setup_buttons_discoverable.md` — touches `TaskViewerProvider.ts` but via a different handler path (setup button state). No file-level conflict.
  - `brain_642190a87...md` and `brain_a5243677e...md` — also unregistered brain mirrors on disk; they may benefit from (or be disturbed by) the salvage sweep. Acceptable, since they share the same fix path.
  - No pending plans in CREATED/PLAN REVIEWED touch the registry, reconciler, or migration methods.

### Balanced Response

Grumpy's critique is, theatrically exaggerated though it is, *substantially correct* on every point. The implementation steps below address each concern:

1. **Key fix location**: The strip is applied at load-time in `_loadPlanRegistry` (line 5226), changing `entries[p.sessionId]` to `entries[p.sessionId.replace(/^antigravity_/, '')]`. This is the single canonical read path.
2. **Migration guard bypass**: `_migrateLegacyToRegistry` gains a dedicated brain-plan rescue block that runs *after* the early-exit guard. It queries the DB for `brainSourcePath`-typed records, finds mirrors on disk with no corresponding active DB row, and upserts them.
3. **Salvage ordering**: The orphan rescue sub-function is a new private method `_salvageOrphanBrainPlans(workspaceRoot)` called at the *top* of `_reconcileAntigravityPlanMirrors`, before `activeMirrorNames` is built. Salvaged entries are also added to `activeMirrorNames` so the final sweep does not re-quarantine them.
4. **Ghost row purge**: After fixing the in-memory key, `_loadPlanRegistry` also emits a one-time DB cleanup: any row whose `sessionId` matches `antigravity_<hash>` but whose corresponding in-memory entry already exists under the raw hash is deleted.

### 1. Fix: In-Memory Key Normalization on Registry Load

#### [MODIFY] `src/services/TaskViewerProvider.ts`

- **Context:** `_loadPlanRegistry` (line 5218) populates `this._planRegistry.entries` by iterating `db.getAllPlans()`. Each `KanbanPlanRecord` has a `sessionId` of `antigravity_<hash>` for brain plans. The in-memory key is currently set to `p.sessionId` (i.e. `antigravity_<hash>`). `_isPlanInRegistry` does a direct `this._planRegistry.entries[planId]` lookup using the raw SHA-256 hash. The mismatch means every brain plan lookup returns `undefined`.

- **Logic:**
  1. When building `entries` from DB records, strip the `antigravity_` prefix from the key if `p.sourceType === 'brain'` (or if the sessionId starts with `antigravity_`).
  2. After populating the in-memory map, query the DB for any *stale* rows still keyed as `antigravity_<hash>` that are now duplicated by the corrected raw-hash entries, and delete them (one-shot cleanup).

- **Implementation:**

```typescript
// In _loadPlanRegistry, replace the loop at lines 5224–5238:
const entries: Record<string, PlanRegistryEntry> = {};
const stalePrefixedIds: string[] = [];

for (const p of allPlans) {
    // Normalize in-memory key: brain plans are stored in DB under
    // sessionId = `antigravity_<hash>`, but all in-memory registry
    // lookups (_isPlanInRegistry, _getPlanIdForRunSheet) use the raw hash.
    const normalizedKey = p.sessionId.startsWith('antigravity_')
        ? p.sessionId.replace(/^antigravity_/, '')
        : p.sessionId;

    if (normalizedKey !== p.sessionId) {
        // Flag the old prefixed row for cleanup if we are now keying correctly
        stalePrefixedIds.push(p.sessionId);
    }

    entries[normalizedKey] = {
        planId: normalizedKey,
        ownerWorkspaceId: p.workspaceId,
        sourceType: p.sourceType,
        localPlanPath: p.planFile,
        brainSourcePath: p.brainSourcePath || undefined,
        mirrorPath: p.mirrorPath || undefined,
        topic: p.topic,
        createdAt: p.createdAt,
        updatedAt: p.updatedAt,
        status: p.status === 'completed' ? 'archived' : p.status as PlanRegistryEntry['status'],
    };
}

this._planRegistry = { version: 1, entries };

// One-shot cleanup: delete stale prefixed rows that are now superseded
// by the correctly-keyed entry. Skip if db is unavailable.
if (stalePrefixedIds.length > 0 && db) {
    for (const staleId of stalePrefixedIds) {
        const rawHash = staleId.replace(/^antigravity_/, '');
        // Only delete stale row if a correctly-keyed row already exists in DB
        const correctRow = await db.getPlanBySessionId(rawHash).catch(() => null);
        if (correctRow) {
            await db.deletePlan(staleId).catch((e) =>
                console.warn(`[TaskViewerProvider] Failed to purge stale prefixed DB row: ${staleId}`, e)
            );
        }
    }
    console.log(`[TaskViewerProvider] Purged ${stalePrefixedIds.length} stale antigravity_-prefixed registry rows`);
}
```

- **Edge Cases Handled:** Idempotent — if the stale row is already deleted, `deletePlan` is a no-op. If `getPlanBySessionId(rawHash)` throws (DB unavailable), the catch silently skips deletion and the stale row persists harmlessly until next startup.

### 2. Fix: Migration Includes Brain Plans (Bypass Early-Exit Guard)

#### [MODIFY] `src/services/TaskViewerProvider.ts` — `_migrateLegacyToRegistry`

- **Context:** `_migrateLegacyToRegistry` (line 5828) exits immediately if the DB already has *any* plans (line 5834). This means brain plans are never enrolled for a workspace that already has local plans. The fix adds a dedicated brain-rescue block that runs after the guard, scanning the local `plans/` staging directory for `brain_*.md` files with no corresponding active DB record.

- **Logic:**
  1. After the existing early-exit guard, call a new sub-routine `_rescueBrainMirrorsWithoutRegistryEntry(workspaceRoot, db, wsId)`.
  2. Inside: read all `brain_*.md` files in `.switchboard/plans/`. For each, derive the hash from the filename, compute the canonical `sessionId = antigravity_<hash>`, and call `db.getPlanBySessionId(sessionId)`.
  3. If no DB record exists (or status is `orphan`/`deleted`), read the mirror file to infer a topic, then upsert a new `active` record keyed under the raw hash (to match the corrected `_loadPlanRegistry` behavior).
  4. Also scan `.switchboard/sessions/` for `antigravity_<hash>.json` runsheets that reference a `brainSourcePath` not yet in the registry.

- **Implementation:**

```typescript
private async _migrateLegacyToRegistry(workspaceRoot: string): Promise<void> {
    // DB-first: if DB already has plans for this workspace, skip legacy migration
    // BUT still attempt to rescue unregistered brain mirrors (see below).
    const db = await this._getKanbanDb(workspaceRoot);
    const wsId = await this._getOrCreateWorkspaceId(workspaceRoot);

    if (db) {
        const existing = await db.getAllPlans(wsId);
        if (existing.length > 0) {
            // Rescue brain mirrors that slipped past the original migration
            await this._rescueBrainMirrorsWithoutRegistryEntry(workspaceRoot, db, wsId);
            return;
        }
    }

    // Original first-time migration: local plans only
    const registry: PlanRegistry = { version: 1, entries: {} };
    const now = new Date().toISOString();

    const log = this._getSessionLog(workspaceRoot);
    try {
        const sheets = await log.getRunSheets();
        for (const sheet of sheets) {
            if (sheet.brainSourcePath) continue; // brain plans handled by rescue below
            if (!sheet.sessionId || !sheet.planFile) continue;
            const planId = sheet.sessionId;
            if (registry.entries[planId]) continue;
            registry.entries[planId] = {
                planId,
                ownerWorkspaceId: wsId,
                sourceType: 'local',
                localPlanPath: sheet.planFile,
                topic: sheet.topic || '',
                createdAt: sheet.createdAt || now,
                updatedAt: sheet.completedAt || sheet.createdAt || now,
                status: sheet.completed === true ? 'archived' : 'active'
            };
        }
    } catch (e) {
        console.error('[TaskViewerProvider] Failed to migrate local plans:', e);
    }

    this._planRegistry = registry;
    await this._savePlanRegistry(workspaceRoot);
    console.log(`[TaskViewerProvider] Migrated ${Object.keys(registry.entries).length} plans to registry`);

    // Also rescue any brain mirrors present on disk
    if (db) {
        await this._rescueBrainMirrorsWithoutRegistryEntry(workspaceRoot, db, wsId);
    }
}

/**
 * Scans `.switchboard/plans/brain_*.md` for mirror files with no active DB record
 * and auto-registers them as owned by this workspace.
 * Called both from the migration path (first run) and after the early-exit guard
 * (subsequent runs where local plans already exist).
 */
private async _rescueBrainMirrorsWithoutRegistryEntry(
    workspaceRoot: string,
    db: KanbanDatabase,
    wsId: string
): Promise<void> {
    const stagingDir = path.join(workspaceRoot, '.switchboard', 'plans');
    if (!fs.existsSync(stagingDir)) return;

    let rescued = 0;
    const files = await fs.promises.readdir(stagingDir).catch(() => [] as string[]);
    const now = new Date().toISOString();

    for (const file of files) {
        if (!file.startsWith('brain_') || !file.endsWith('.md')) continue;

        const hash = file.replace(/^brain_/, '').replace(/\.md$/, '');
        if (!/^[a-f0-9]{64}$/i.test(hash)) continue; // validate as SHA-256

        const sessionId = `antigravity_${hash}`;
        let existingRow: KanbanPlanRecord | null = null;
        try {
            existingRow = await db.getPlanBySessionId(sessionId);
        } catch { /* continue */ }

        // Skip if already registered and active
        if (existingRow && existingRow.status === 'active') continue;

        const mirrorFullPath = path.join(stagingDir, file);
        let topic = '(rescued brain plan)';
        try {
            const content = await fs.promises.readFile(mirrorFullPath, 'utf8');
            const match = content.match(/^#\s+(.+)$/m);
            if (match?.[1]) topic = match[1].trim().slice(0, 120);
        } catch { /* leave default topic */ }

        const mirrorRelPath = path.join('.switchboard', 'plans', file).replace(/\\/g, '/');

        try {
            await db.upsertPlans([{
                planId: hash,
                sessionId,
                topic,
                planFile: mirrorRelPath,
                kanbanColumn: 'CREATED',
                status: 'active',
                complexity: 'Unknown',
                tags: '',
                dependencies: '',
                workspaceId: wsId,
                createdAt: now,
                updatedAt: now,
                lastAction: '',
                sourceType: 'brain',
                brainSourcePath: '',  // Will be resolved by _resolveBrainSourcePathForMirrorHash on first use
                mirrorPath: file,
                routedTo: '',
                dispatchedAgent: '',
                dispatchedIde: '',
            }]);

            // Mirror the entry into in-memory registry too
            this._planRegistry.entries[hash] = {
                planId: hash,
                ownerWorkspaceId: wsId,
                sourceType: 'brain',
                mirrorPath: file,
                localPlanPath: mirrorRelPath,
                topic,
                createdAt: now,
                updatedAt: now,
                status: 'active',
            };

            rescued++;
            console.log(`[TaskViewerProvider] Rescued unregistered brain mirror: ${file} → sessionId=${sessionId}`);
        } catch (e) {
            console.warn(`[TaskViewerProvider] Failed to rescue brain mirror ${file}:`, e);
        }
    }

    if (rescued > 0) {
        console.log(`[TaskViewerProvider] Brain mirror rescue complete: ${rescued} plan(s) restored to registry`);
    }
}
```

- **Edge Cases Handled:**
  - Hash validation (`/^[a-f0-9]{64}$/i`) prevents non-hash `brain_*.md` files (e.g. `brain_dump.md`) from being processed.
  - `existingRow.status === 'active'` check is idempotent — already-registered plans are untouched.
  - Topic is truncated at 120 chars to prevent DB column overflow.
  - `brainSourcePath` is left empty — `_resolveBrainSourcePathForMirrorHash` is the authoritative resolver and will fill it in on first watcher event.

### 3. Fix: Orphan Salvage Before Reconciler Sweep

#### [MODIFY] `src/services/TaskViewerProvider.ts` — `_reconcileAntigravityPlanMirrors`

- **Context:** The reconciler (line 6304) builds `activeMirrorNames` from the DB, then at lines 6450-6460 quarantines any `brain_*.md` in staging that is not in `activeMirrorNames`. If a mirror was previously orphaned, it sits in `orphan_plans/` and is not seen. The salvage must run before `activeMirrorNames` is built and must update it so the final sweep does not re-quarantine freshly rescued files.

- **Logic:**
  1. At the top of `_reconcileAntigravityPlanMirrors`, call `await this._salvageOrphanBrainPlans(workspaceRoot, db, wsId)` (which moves qualifying files from `orphan_plans/` and `orphan_sessions/` back to staging while registering them).
  2. `_salvageOrphanBrainPlans` returns the set of mirror filenames it rescued.
  3. This rescued set is merged into `activeMirrorNames` before the final staging sweep.

- **Implementation — new method `_salvageOrphanBrainPlans`:**

```typescript
/**
 * Scans archive/orphan_plans/ for brain mirrors that can now be claimed
 * by this workspace and moves them back to staging. Also rescues matching
 * orphan runsheets from archive/orphan_sessions/.
 *
 * Returns the set of mirror filenames that were rescued (to be merged into
 * activeMirrorNames so the reconciler does not immediately re-quarantine them).
 */
private async _salvageOrphanBrainPlans(
    workspaceRoot: string,
    db: KanbanDatabase | null,
    wsId: string
): Promise<Set<string>> {
    const switchboardDir = path.join(workspaceRoot, '.switchboard');
    const stagingDir = path.join(switchboardDir, 'plans');
    const orphanPlansDir = path.join(switchboardDir, 'archive', 'orphan_plans');
    const orphanSessionsDir = path.join(switchboardDir, 'archive', 'orphan_sessions');

    const rescued = new Set<string>();

    if (!fs.existsSync(orphanPlansDir)) return rescued;

    const orphanFiles = await fs.promises.readdir(orphanPlansDir).catch(() => [] as string[]);
    const now = new Date().toISOString();

    for (const file of orphanFiles) {
        if (!file.startsWith('brain_') || !file.endsWith('.md')) continue;

        const hash = file.replace(/^brain_/, '').replace(/\.md$/, '');
        if (!/^[a-f0-9]{64}$/i.test(hash)) continue;

        const sessionId = `antigravity_${hash}`;

        // Only salvage if not already owned by another workspace
        let existingRow: KanbanPlanRecord | null = null;
        if (db) {
            existingRow = await db.getPlanBySessionId(sessionId).catch(() => null);
        }

        // Skip if another workspace owns it and it is active
        if (existingRow && existingRow.status === 'active' && existingRow.workspaceId !== wsId) {
            continue;
        }

        const orphanMirrorPath = path.join(orphanPlansDir, file);
        const targetMirrorPath = path.join(stagingDir, file);

        try {
            // Move mirror from orphan to staging (skip if staging already has the file)
            if (!fs.existsSync(targetMirrorPath)) {
                await fs.promises.rename(orphanMirrorPath, targetMirrorPath);
            } else {
                // Staging already has it — just remove the stale orphan copy
                await fs.promises.unlink(orphanMirrorPath).catch(() => {});
            }

            // Suppress watcher loop: mark staging path as a recent write
            const existing = this._recentMirrorWrites.get(targetMirrorPath);
            if (existing) clearTimeout(existing);
            this._recentMirrorWrites.set(
                targetMirrorPath,
                setTimeout(() => this._recentMirrorWrites.delete(targetMirrorPath), 3000)
            );

            // Rescue matching orphan runsheet if present
            const orphanSessionPath = path.join(orphanSessionsDir, `${sessionId}.json`);
            const activeSessionPath = path.join(switchboardDir, 'sessions', `${sessionId}.json`);
            if (fs.existsSync(orphanSessionPath) && !fs.existsSync(activeSessionPath)) {
                const sessionsDir = path.join(switchboardDir, 'sessions');
                if (!fs.existsSync(sessionsDir)) fs.mkdirSync(sessionsDir, { recursive: true });
                await fs.promises.rename(orphanSessionPath, activeSessionPath).catch((e) =>
                    console.warn(`[TaskViewerProvider] Failed to rescue orphan session ${sessionId}:`, e)
                );
            }

            // Register in DB if not already active
            if (db && (!existingRow || existingRow.status !== 'active')) {
                let topic = '(rescued orphan brain plan)';
                try {
                    const content = await fs.promises.readFile(targetMirrorPath, 'utf8');
                    const match = content.match(/^#\s+(.+)$/m);
                    if (match?.[1]) topic = match[1].trim().slice(0, 120);
                } catch { /* leave default */ }

                const mirrorRelPath = path.join('.switchboard', 'plans', file).replace(/\\/g, '/');
                await db.upsertPlans([{
                    planId: hash,
                    sessionId,
                    topic,
                    planFile: mirrorRelPath,
                    kanbanColumn: 'CREATED',
                    status: 'active',
                    complexity: existingRow?.complexity || 'Unknown',
                    tags: existingRow?.tags || '',
                    dependencies: existingRow?.dependencies || '',
                    workspaceId: wsId,
                    createdAt: existingRow?.createdAt || now,
                    updatedAt: now,
                    lastAction: existingRow?.lastAction || '',
                    sourceType: 'brain',
                    brainSourcePath: existingRow?.brainSourcePath || '',
                    mirrorPath: file,
                    routedTo: existingRow?.routedTo || '',
                    dispatchedAgent: existingRow?.dispatchedAgent || '',
                    dispatchedIde: existingRow?.dispatchedIde || '',
                }]).catch((e) =>
                    console.warn(`[TaskViewerProvider] Failed to upsert rescued orphan ${sessionId}:`, e)
                );

                // Also update in-memory registry
                this._planRegistry.entries[hash] = {
                    planId: hash,
                    ownerWorkspaceId: wsId,
                    sourceType: 'brain',
                    mirrorPath: file,
                    localPlanPath: mirrorRelPath,
                    topic,
                    createdAt: existingRow?.createdAt || now,
                    updatedAt: now,
                    status: 'active',
                };
            }

            rescued.add(file);
            console.log(`[TaskViewerProvider] Salvaged orphan brain plan: ${file}`);
        } catch (e) {
            console.warn(`[TaskViewerProvider] Failed to salvage orphan brain plan ${file}:`, e);
        }
    }

    return rescued;
}
```

- **Edge Cases Handled:**
