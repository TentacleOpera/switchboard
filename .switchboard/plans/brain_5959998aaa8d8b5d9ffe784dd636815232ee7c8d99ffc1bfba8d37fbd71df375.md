# Fix Antigravity Brain Plan Mirror Scoping and Debounce Issues

## Metadata
**Complexity:** 4
**Tags:** backend, reliability, bugfix, testing

## Goal

Resolve the issue where the Switchboard system fails to detect and mirror newly created Antigravity brain plans from the shared `~/.gemini/antigravity-ide/brain` directory.

### Core Problems & Root Cause Analysis

1. **Scoping Cutoff and Time Windows (`TaskViewerProvider.ts`)**:
   - A plan is currently only auto-claimed if it is less than 15 seconds old (`isFreshUnregisteredCandidate`) or falls within a dynamic scan `cutoff` window. If the watcher misses the creation event, or if the rescan runs after the file has aged past 15 seconds, the plan is permanently ignored and never mirrored.
   - The system also lacks a proper content-based workspace-scoping check for unregistered plans, resulting in either eager cross-workspace contamination or overly restrictive timing gates.

2. **Debounce Timer Clears Auto-Claim Flag**:
   - On macOS/Linux, creating and writing a plan file triggers a `rename` event (creation) followed by a `change` event (writing content) within milliseconds.
   - The file watcher debounce logic handles the `rename` event first (`allowAutoClaim = true`), setting a 300ms timeout.
   - Immediately after, the `change` event fires (`allowAutoClaim = false`), which clears the existing timeout and schedules a new one with `allowAutoClaim = false`.
   - As a result, the debounce callback always executes with `allowAutoClaim = false`, preventing the plan from ever being auto-claimed at creation.

3. **Stale/Rigid Test Assertions**:
   - Multiple assertions in `workspace-scope-regression.test.js`, `brain-delete-tombstone-regression.test.js`, and `brain-registry-rescue-regression.test.js` are failing because they assert exact regexes of legacy code structures that were altered during the SQLite/database-first refactoring.

## Proposed Changes

### [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)

#### 1. Add Debounce Claims Accumulation Set
Add a new private field to preserve the `allowAutoClaim` state during file event debouncing:
```typescript
private _brainDebounceClaims = new Set<string>(); // track whether any event in the debounce window requested auto-claim
```

#### 2. Fix the Watcher Debounce Callback (`handleBrainEvent`)
Update the VS Code file watcher and native fallback watcher to add the stable path to `_brainDebounceClaims` when `allowAutoClaim` is true. Read and clear it within the debounced timeout:
```typescript
const handleBrainEvent = (uri: vscode.Uri, allowAutoClaim: boolean) => {
    // ...
    const stablePath = this._getStablePath(fullPath);
    if (allowAutoClaim) {
        this._brainDebounceClaims.add(stablePath);
    }
    // ...
    this._brainDebounceTimers.set(stablePath, setTimeout(async () => {
        try {
            this._brainDebounceTimers.delete(stablePath);
            const finalAllowAutoClaim = this._brainDebounceClaims.has(stablePath);
            this._brainDebounceClaims.delete(stablePath);
            
            if (this._recentBrainWrites.has(stablePath)) return;
            if (fs.existsSync(fullPath)) {
                await this._ensureTombstonesLoaded(workspaceRoot);
                await this._mirrorBrainPlan(fullPath, finalAllowAutoClaim, workspaceRoot);
            }
        } // ...
```

Do the same for the native `fs.watch` fallback:

const rawAutoClaim = _eventType === 'rename';

#### 3. Update Scoping Check in `_isPlanEligibleForWorkspace`
Enhance `_isPlanEligibleForWorkspace` to inspect the plan's markdown contents or the session's `transcript.jsonl` log for the workspace root path. This ensures that any unregistered brain plan is accurately scoped to its corresponding workspace:
```typescript
private _isPlanEligibleForWorkspace(stableBrainPath: string, workspaceRoot: string): { eligible: boolean; reason: string } {
    const planId = this._getPlanIdFromStableBrainPath(stableBrainPath);
    if (this._isPlanInRegistry(planId)) {
        return { eligible: true, reason: 'in_plan_registry' };
    }

    // Scoping check for unregistered plans: check if the plan contains paths belonging to this workspace
    try {
        const rawPath = this._getBaseBrainPath(stableBrainPath);
        if (fs.existsSync(rawPath)) {
            const content = fs.readFileSync(rawPath, 'utf8');
            const normalizedRoot = this._getStablePath(workspaceRoot);

            if (content.toLowerCase().includes(normalizedRoot.toLowerCase())) {
                return { eligible: true, reason: 'workspace_path_match' };
            }

            // Fallback: check session transcript
            const sessionDir = path.dirname(rawPath);
            const transcriptPath = path.join(sessionDir, '.system_generated', 'logs', 'transcript.jsonl');
            if (fs.existsSync(transcriptPath)) {
                const transcript = fs.readFileSync(transcriptPath, 'utf8');
                if (transcript.toLowerCase().includes(normalizedRoot.toLowerCase())) {
                    return { eligible: true, reason: 'workspace_transcript_match' };
                }
            }
        }
    } catch (e) {
        console.warn('[TaskViewerProvider] Failed to verify plan workspace eligibility:', e);
    }

    return { eligible: false, reason: 'not_in_plan_registry' };
}
```
