## 📄 Full Description

### Epic Goal
As a new user, I want to complete onboarding quickly through a guided step-by-step flow so that I can access VARA faster and the app can personalise my experience.

### Background
The current onboarding flow is entirely AI conversation-driven, causing drop-off. Users must wait on AI responses at each step before progressing.

This redesign replaces it with a structured funnel that collects the same personalisation data faster, using pre-built screens and branching logic. The result is quicker access to VARA, richer user profiles, and better email marketing data.

### Completion Deadline: 25 May
This redesign needs to be coordinated with marketing. Therefore, we need to complete this work by 25 May. Other work can be deprioritised to ensure this happens.

**Risk of slipping past deadline is high.** Possible mitigation is an AI/human dev work breakdown in the comments.

### Long-Term Product Goal
This onboarding should gradually become a behavioural intelligence layer for Autism360.
The goal is not only conversion tracking, but understanding:
- what parents are struggling with,
- what stage they are in emotionally,
- and how those patterns differ across demographics and child age groups.

This data should later help:
- personalise onboarding,
- personalise VARA,
- improve resources,
- improve funnels,
- and guide future product decisions.

### High Level Flow
**User pre-purchase:**
- Lands from ad
- Enters short onboarding flow
- Answers guided mapping questions
- Receives adaptive insight/strategy/value screens
- Provides email
- Receives automated EDM within minutes
- Views additional value/goals
- Hits payment CTA
- Redirected to App Store / Play Store (or landing page conversion event)

**In-App:**
When the app opens, the user is presented with a login screen (email/password or Google Sign-In). The backend checks if this email completed the web funnel:
- **Match found:** Funnel selections are restored. VARA starts with contextual familiarity.
- **No match found:** Existing onboarding flow is used.

If email is recognised:
- User is linked back to their funnel path
- User creates password
- Existing funnel selections are remembered
- VARA starts with contextual familiarity
