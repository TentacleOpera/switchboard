# Refactor Custom Agent Settings Form in Kanban Tab

This implementation plan covers the removal of redundant settings from the custom agent configuration form (specifically drag & drop mode and column visibility), converting the "Design Doc Link" text field into a standard boolean checkbox that resolves to the global design doc, and simplifying the Prompt Add-ons section by removing the accordion toggle. It also updates the Kanban Column Setup menu to allow custom agents to be selected as assigned agents.

## Proposed Changes

### Frontend Changes (Webview)

#### [MODIFY] [kanban.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/kanban.html)

1. **Remove Drag & Drop Mode Select**:
   - Delete the `<label>` and `<select>` for `agents-tab-custom-agent-dragdrop`.
   - Update `agentsTabShowInlineForm` to remove setting the value of `agents-tab-custom-agent-dragdrop`.
   - Update `agentsTabSaveCustomAgent` to hardcode `dragDropMode` as `'cli'` by default.

2. **Remove Show as Kanban Column Checkbox**:
   - Delete the checkbox input and wrapper label for `agents-tab-custom-agent-kanban`.
   - Update `agentsTabShowInlineForm` to remove setting the checked state of `agents-tab-custom-agent-kanban`.
   - Update `agentsTabSaveCustomAgent` to hardcode `includeInKanban` as `false`.

3. **Convert Design Doc Link to Checkbox Add-on**:
   - Replace the Design Doc Link text input with a standard checkbox: `<label class="checkbox-label"><input type="checkbox" id="ca-addon-design-doc" style="width:auto;margin:0;"> Design Doc Reference</label>`.
   - Update `agentsTabShowInlineForm` to check this box if `addons.designDoc === true` or if `addons.designDocLink` is set (for backward compatibility).
   - Update `agentsTabSaveCustomAgent` to read the checked state and save it under `designDoc`.

4. **Expose Custom Agents in Kanban Column assigned agent selector**:
   - Update `renderKanbanAssignedAgentOptions` to append an `<optgroup label="Custom Agents">` containing all custom agents (from `agentsTabCustomAgents`) to the options dropdown, allowing them to be assigned to columns.

5. **Always Expand Prompt Add-ons Section**:
   - Replace `<details id="agents-tab-custom-agent-addons">` and its `<summary>` tag with a standard `div` titled `Prompt Add-ons`.
   - Remove `document.getElementById('agents-tab-custom-agent-addons').removeAttribute('open')` from `agentsTabShowInlineForm`.

6. **Filter Autoban Roles**:
   - Update the Multi-Column configuration tab's `autobanRoles` to include all custom agents without filtering on `includeInKanban === true`.

### Backend changes

#### [MODIFY] [agentConfig.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/agentConfig.ts)

- Update `CustomAgentAddons` interface to include `designDoc?: boolean;`.
- Update `parseCustomAgentAddons` to parse `designDoc` (boolean) and set it if `designDocLink` (string) is populated, ensuring backward compatibility.
- Update `buildKanbanColumns` to remove the dynamic `customColumns` building step, as custom agents are now represented via custom user columns configured in the Setup tab.

#### [MODIFY] [KanbanProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)

- Update `generateUnifiedPrompt` for custom agents (role starts with `custom_agent_`):
  - If `addons.designDoc` is `true`, resolve the workspace's global `designDocLink` and pre-fetch the Notion cache (using `NotionFetchService` if applicable) for `designDocContent`, just like the built-in planner.
  - Pass the resolved `designDocLink` and `designDocContent` into `buildCustomAgentPrompt`.

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)

- In `_reconcileAutobanPoolState` and `_createAutobanTerminal`, query `customAgents.map(a => a.role)` instead of filtering by `includeInKanban` to ensure custom agent roles are registered in terminal pool checks.
- In `handleRestoreKanbanDefaults`, remove setting `includeInKanban: false`.

## Verification Plan

### Automated Tests
- Run `npm test` or the appropriate test suite command to check all mocha unit tests.

### Manual Verification
1. Open Switchboard Kanban view (`kanban.html`).
2. Go to the **Agents** tab.
3. Add a new custom agent. Verify that:
   - "Drag & Drop Mode" and "Show as Kanban Column" options are gone.
   - "Prompt Add-ons" section is always expanded.
   - "Design Doc Reference" is now a checkbox.
4. Save the custom agent.
5. Go to the **Setup** tab and find "Kanban Structure".
6. Click **ADD COLUMN**. Verify that the newly created custom agent is selectable in the "Assigned agent" dropdown.
7. Save the column and drag it to order it. Verify that the custom column displays on the board, and dragging a card into it triggers the custom agent.
