# IDE Isolation via Terminal Suffixing

## Goal
Allow power users to safely operate multiple Switchboard agents across different IDE clients simultaneously (e.g., Cursor, Windsurf, VS Code) without their shared `.switchboard/state.json` heartbeats and registries colliding, by appending the `ideName` (from `vscode.env.appName`) to the internal state key for each terminal (e.g., `Lead Coder-Cursor`) while keeping the VS Code terminal tab UI unchanged.

## Metadata
**Tags:** backend, infrastructure
**Complexity:** 7

## User Review Required
> [!NOTE]
> - **No visual changes.** Terminal tabs continue to display "Lead Coder", "Coder", etc. The IDE suffix is applied only to `state.json` keys and the in-memory `registeredTerminals` map.
> - **State.json key migration.** Existing `state.json` entries keyed by bare names (e.g., `"Lead Coder"`) will become stale on first run after this change. They will be pruned by the existing heartbeat-expiry logic within 60–120 seconds. No manual migration required.
> - **MCP tool callers must use suffixed names.** Any external MCP client sending `sendToTerminal` or `createTerminal` messages with bare names (e.g., `"Lead Coder"`) will fail to match the registry. The `resolveTerminalByName()` fallback must be updated to strip suffixes during lookup.
> - **Path A only.** This plan solves cross-IDE collisions. Two identical agents in the *same* IDE window still collide. Path B (PID suffix) is deferred.

## User Requirements Captured

1. **Hidden Suffixes**: The UI MUST NOT display the `-Cursor` or `-Visual Studio Code` suffix anywhere. The interface will look exactly the same as it does today (e.g., "Lead Coder", "Claude").

## Clarification on Identical Terminals in the Same IDE

> **To answer your question:** Yes, if you open two "Claude" terminals in the *exact same IDE window*, they will both be named `Claude-Cursor` behind the scenes, which means they will still collide in `state.json` and overwrite each other.

If running two identical agents *inside the very same IDE window* is an important workflow for you, we cannot just use the IDE name. We would instead need to append the **Terminal Process ID (PID)** or a unique **Random ID** to the backend name (e.g., `Claude-12345` and `Claude-98765`).

### Path A: IDE Suffix Only (e.g., `Claude-Cursor`)
- **Fixes**: Prevents collisions between Cursor and Windsurf on the same project.
- **Fails**: If you manually open two Claudes inside Cursor, they still collide.

### Path B: PID Suffix (e.g., `Claude-48291`)
- **Fixes**: Prevents ALL collisions. You can run 5 Claudes in Cursor and 5 in Windsurf, and they all have distinct state entries.
- **Complexity**: Slightly higher, as the routing logic needs to know *which* of the 5 Claudes it's sending a message to if a workflow targets "claude" abstractly.

### 1. Suffix Utility Functions
#### [MODIFY] `src/extension.ts`
- **Context:** A centralized helper is needed to append/strip IDE suffixes consistently across all code paths. Must live near `normalizeAgentKey()` (line 297) since they're used together.
- **Logic:**
  1. `suffixedName(baseName: string): string` — Appends `-${vscode.env.appName}` if not already present. Idempotent.
  2. `stripIdeSuffix(name: string): string` — Removes the `-${vscode.env.appName}` trailer if present. Used for display and role inference.
- **Implementation:**
```typescript
// Add after normalizeAgentKey() at line 303
function suffixedName(baseName: string): string {
    const suffix = `-${vscode.env.appName}`;
    return baseName.endsWith(suffix) ? baseName : `${baseName}${suffix}`;
}

function stripIdeSuffix(name: string): string {
    const suffix = `-${vscode.env.appName}`;
    return name.endsWith(suffix) ? name.slice(0, -suffix.length) : name;
}
```
- **Edge Cases Handled:** Idempotent — calling `suffixedName()` on an already-suffixed name is a no-op. `stripIdeSuffix()` only strips the current IDE's suffix, so `"Lead Coder-Windsurf"` viewed from Cursor is left unchanged (correct — it belongs to a different IDE).

### 3. `resolveTerminalByName()` Suffix-Aware Lookup
#### [MODIFY] `src/extension.ts` (line 409–429)
- **Context:** This is the central dispatch function. MCP `sendToTerminal` and other callers pass bare names. Must resolve against suffixed registry keys.
- **Logic:**
  1. Try exact match (existing).
  2. **NEW:** Try `suffixedName(terminalName)` exact match.
  3. Fall through to normalized fuzzy match (existing), but normalize AFTER stripping suffix.
  4. Fall through to live VS Code terminal match (existing), comparing against both UI name and stripped name.
- **Implementation:**
```typescript
function resolveTerminalByName(terminalName: string): vscode.Terminal | undefined {
    // 1. Exact match
    const exact = registeredTerminals.get(terminalName);
    if (exact && exact.exitStatus === undefined) {
        return exact;
    }

    // 2. Try suffixed name (bare name → suffixed key)
    const suffixed = suffixedName(terminalName);
    const bySuffix = registeredTerminals.get(suffixed);
    if (bySuffix && bySuffix.exitStatus === undefined) {
        return bySuffix;
    }

    // 3. Normalized fuzzy match (strip suffix before normalizing)
    const normalizedTarget = normalizeAgentKey(stripIdeSuffix(terminalName));
    for (const [name, terminal] of registeredTerminals.entries()) {
        if (terminal.exitStatus !== undefined) continue;
        if (normalizeAgentKey(stripIdeSuffix(name)) === normalizedTarget) {
            return terminal;
        }
    }

    // 4. Live VS Code terminal fallback
    return (vscode.window.terminals || []).find((terminal) => {
        if (terminal.exitStatus !== undefined) return false;
        const liveName = normalizeAgentKey(terminal.name);
        const creationName = normalizeAgentKey(
            (terminal.creationOptions as vscode.TerminalOptions | undefined)?.name || ''
        );
        return liveName === normalizedTarget || creationName === normalizedTarget;
    });
}
```
- **Edge Cases Handled:** Bare names from MCP resolve via step 2. Already-suffixed names resolve via step 1. Cross-IDE names (e.g., `"Lead Coder-Windsurf"` from Cursor) will NOT resolve — correct behavior, as you shouldn't route to another IDE's terminal.
