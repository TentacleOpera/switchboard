# Fix Switchboard Kanban Missing/Hidden Antigravity Brain Plans

This plan addresses two critical bugs that prevent Antigravity brain plans from showing up on the Switchboard Kanban board, while they are visible in the Research tab (which directly reads the filesystem).

## Goal

Resolve the discrepancy where plans are mirrored but hidden from columns due to a database sentinel mismatch, or skipped entirely due to a strict 15-second autoclaim window.

### Root Cause Analysis

1. **Sentinel Project Filter Mismatch**:
   - By default, `KanbanProvider.ts` sets the project filter to `__unassigned__`.
   - When a new brain plan is mirrored in `TaskViewerProvider.ts`, it retrieves the active project filter (`__unassigned__`) and registers the plan with this project name.
   - This value is persisted as `__unassigned__` in the `project` column of the `plans` table in `kanban.db`.
   - However, during Kanban board retrieval, `KanbanDatabase.ts` translates the `__unassigned__` filter to an empty string (`''`) in SQL queries (`AND project = ''`).
   - Consequently, the query does not match rows with `project = '__unassigned__'`, making them invisible on the board.

2. **15-Second Autoclaim Cutoff**:
   - `TaskViewerProvider._mirrorBrainPlan` restricts auto-claiming unregistered plans to a strict 15-second window from file creation.
   - If a plan is created when VS Code is closed, inactive, or not focused, subsequent rescans process it outside this window. Because `isRecent` is `false` and the window has expired, the plan is permanently skipped and never registered or mirrored.

## Proposed Changes

### [Component: Switchboard Services]

#### [MODIFY] [KanbanDatabase.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanDatabase.ts)
- Add database migration version 28 to update existing plans where `project = '__unassigned__'` to `project = ''`.
- Update `assignPlansToProject` to translate `'__unassigned__'` to `''` before executing SQL.

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)
- Normalize `project` values loaded in `_loadPlanRegistry` to translate `'__unassigned__'` to `''`.
- Normalize `project` values saved in `_savePlanRegistry` and `_registerPlan` to translate `'__unassigned__'` to `''`.
- Normalize the `activeProject` fetched in `_mirrorBrainPlan` to translate `'__unassigned__'` to `''` before registration.
- Relax the `isFreshUnregisteredCandidate` age check: allow auto-claiming a plan of any age if no claim marker file exists on disk, or if the claim marker is owned by the current workspace.
