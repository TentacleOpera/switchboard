# Prune Stale Selected Cards in Kanban View

Prune selected card IDs from the `selectedCards` state map when the board is updated and they are no longer present in the board data (e.g., because they were deleted, or filtered out due to workspace or project filter change).

## Goal

Remove stale entries from the `selectedCards` Map inside `renderBoard()` so that the ASSIGN button count and merge-button state stay accurate when cards disappear from the board.

## Proposed Changes

### [MODIFY] [kanban.html](src/webview/kanban.html)

#### Context
The `selectedCards` Map (line 3322) accumulates card IDs as the user clicks cards. When `renderBoard(cards)` is called (line 4452) after a board update — triggered by card deletion, workspace switch, or project filter change — the incoming `cards` array may no longer contain previously selected cards. Today, those stale entries persist in the Map, causing the ASSIGN button to show an inflated count and merge-selected buttons to reference non-existent cards.

#### Logic
Build a `Set` of all card IDs from the raw `cards` array (not the backlog-filtered `displayCards`), then delete any `selectedCards` keys not in that set. Using the raw array ensures that BACKLOG cards hidden by the backlog toggle remain selected (they still exist in the data), while cards removed by deletion or workspace/project filter change are correctly pruned.

#### Implementation
Insert the pruning block inside `renderBoard(cards)`, immediately after the line `currentCards = cards;` (line 4455) and before the backlog filtering at line 4462.

```javascript
// Inside renderBoard(cards), after currentCards = cards; (line 4455):
// Prune selectedCards to only keep cards that are present in the current board data
const cardIds = new Set(cards.map(c => c.sessionId || c.planId).filter(Boolean));
for (const key of selectedCards.keys()) {
    if (!cardIds.has(key)) {
        selectedCards.delete(key);
    }
}
```

No additional call to `updateReassignButtonVisibility()` is needed — it is already invoked at the end of `renderBoard` (line 4711), which will pick up the pruned `selectedCards.size`.

#### Edge Cases
- **Empty `cards` array:** The `Set` will be empty, so all `selectedCards` entries are pruned. This is correct — if the board has no cards, nothing should be selected.
- **BACKLOG toggle off:** Cards in BACKLOG column are in `cards` but not in `displayCards`. Since pruning uses `cards`, these selections are preserved. If the user toggles backlog back on, the cards reappear and remain selected.
- **Cards with neither `sessionId` nor `planId`:** `filter(Boolean)` removes them from the `Set`. Such cards cannot be selected anyway (click handler returns early at line 4539 when `!pid`), so this is harmless.
