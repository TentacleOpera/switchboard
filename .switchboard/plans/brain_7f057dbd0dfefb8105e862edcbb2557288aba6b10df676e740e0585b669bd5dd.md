# Sidebar Exclusive Accordion Navigation

## Goal
Implement an "exclusive accordion" behaviour for the Switchboard sidebar so that expanding one collapsible panel automatically collapses all others, reducing vertical clutter. Additionally, clicking "OPEN AGENT TERMINALS" in the Terminal Operations accordion must automatically collapse that accordion immediately upon click.

## Metadata
**Tags:** frontend, UI
**Complexity:** Low

## User Review Required
> [!NOTE]
> This is a purely client-side webview change. No backend, extension host, or persistence layer is touched.  
> The only observable side-effect is that the "SETUP" and "Database Operations" panels will now close when "Terminal Operations" opens (and vice-versa). This changes existing UX behaviour intentionally.

## Complexity Audit

### Routine
- Add a centralised `collapseAllAccordions()` helper that finds all accordion pairs by a shared `data-accordion` attribute and removes the `open` class from both `fields` and `chevron` elements.
- Wire the helper into the three existing toggle click handlers (`terminal-operations-toggle`, `startup-toggle`, `db-sync-toggle-btn`) so it is called before the target panel is expanded.
- Attach a call to `collapseAllAccordions()` (or just the Terminal Operations collapse) inside the `#createAgentGrid` click handler immediately after the button's loading state is set.

### Complex / Risky
- **Guard: startup auto-expand must not trigger mutual collapse.** The onboarding guard at line ~3224 programmatically opens `terminal-operations-fields` without going through the toggle click handler. The exclusive-accordion call must be inserted there too, so that when Terminal Ops auto-opens on startup (0 green agents), it collapses SETUP and Database Operations. Care must be taken that this doesn't fight with `hasManuallyCollapsedThisSession` — the flag only prevents *re-opening* Terminal Ops; collapsing others during an auto-open is always safe.
- **Guard: live-feed toggle is NOT part of the exclusive group.** The `live-feed-toggle` / `panel-activity` panel is styled differently (`panel-fields` class) and logically orthogonal. It must be excluded from the accordion group.

## Edge-Case & Dependency Audit
- **Race Conditions:** None. All logic is synchronous DOM manipulation in the same JS event loop tick.
- **Security:** No impact. No data sent to extension host.
- **Side Effects:** When SETUP or Database Operations is open and the user clicks "OPEN AGENT TERMINALS", both panels will collapse. This is expected per spec. No settings are lost because the accordion body still exists in the DOM — inputs retain their values.
- **Dependencies & Conflicts:**

  - No other in-flight plans touch `implementation.html` accordion JS.

### Balanced Response
All of Grumpy's concerns are valid. The implementation steps below address them precisely:

1. **HTML annotation**: Add `data-accordion="true"` to all three toggle `div`s and all three field `div`s. The live-feed toggle gets no attribute — exclusion is structural.
2. **CSS class heterogeneity**: The `collapseAllAccordions(exceptId)` helper selects by `data-accordion` on the **field** divs (not by class), then removes `open` from both the field and its sibling chevron. No CSS class name assumption needed.
3. **Startup auto-expand**: Call `collapseAllAccordions('terminal-operations-fields')` immediately before the `toFields.classList.add('open')` block at line 3228. This collapses SETUP and DB Ops while opening Terminal Ops — safe and additive.
4. **`hasManuallyCollapsedThisSession` after grid launch**: The programmatic collapse inside the `createAgentGrid` handler should NOT set `hasManuallyCollapsedThisSession = true`. The Terminal Ops panel was not collapsed by the user — it was collapsed as a UX courtesy after the user clicked the launch button. The onboarding guard should not count this as user intent. When agents come up within seconds, the guard won't try to re-open (because `noAgentsDetectionTime` resets once active terminals are detected). If agents fail to register, the guard may expand Terminal Ops again — which is actually correct behaviour (showing the user the panel to retry). This is acceptable.

**Step 2: Helper function** (add immediately before `const termOpsToggle = ...` at line ~2220):

```javascript
/**
 * Collapses all accordion panels that carry data-accordion="true",
 * except the one whose id matches `exceptFieldsId`.
 * Also removes 'open' from the sibling chevron element.
 *
 * The chevron element is located by walking to the parent toggle div
 * (previousElementSibling of the fields div) and finding its .chevron child.
 *
 * @param {string|null} exceptFieldsId - ID of the fields div to leave untouched.
 */
function collapseAllAccordions(exceptFieldsId) {
    document.querySelectorAll('[data-accordion="true"]').forEach(fieldsEl => {
        if (fieldsEl.id === exceptFieldsId) return; // skip the target panel
        if (!fieldsEl.classList.contains('open')) return; // already closed
        fieldsEl.classList.remove('open');
        // Find the chevron in the sibling toggle div (always the immediately preceding sibling)
        const toggleEl = fieldsEl.previousElementSibling;
        if (toggleEl) {
            const chevron = toggleEl.querySelector('.chevron');
            if (chevron) chevron.classList.remove('open');
        }
        // For panel-fields that expose aria-hidden, update it
        if (fieldsEl.hasAttribute('aria-hidden')) {
            fieldsEl.setAttribute('aria-hidden', 'true');
        }
    });
}
```
