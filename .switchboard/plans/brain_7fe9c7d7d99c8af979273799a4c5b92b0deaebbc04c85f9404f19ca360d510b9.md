# Sync Planner Metadata Updates to Kanban Database

Ensure that when a planner agent writes metadata updates (complexity, tags, dependencies, and topic) to a mirror plan file (e.g., `.switchboard/plans/brain_*.md` or `ingested_*.md`), the updates are immediately parsed and synced to the Kanban database, and the Kanban board UI is refreshed.

Also, refine metadata regex patterns to handle case/formatting variations (e.g. `**Complexity**: 5` vs `**Complexity:** 5`).

## Proposed Changes

### Staging Watcher Coordination

Modify the staging directory watcher to parse mirror plan metadata and update the SQLite database records whenever mirror files are updated.

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)

- Update the staging watcher callback (`this._stagingWatcher = fs.watch(...)`):
  - In both `isBrainMirror` and the ingested branch, after syncing the mirror content to the respective source file, query `KanbanDatabase` for the corresponding plan record by the relative mirror path (`planFile`).
  - Read the mirror file content, parse it with `parsePlanMetadata`, and update the record (topic, complexity, tags, dependencies, `updatedAt`) via `db.upsertPlans`.
  - Trigger a UI refresh by calling `this._kanbanProvider?.refreshIfShowing(workspaceRoot)`.

---

### Metadata Extraction Robustness

Refine metadata parsing regexes in `planMetadataUtils.ts` to support both `**Header**: <value>` and `**Header:** <value>` formats.

#### [MODIFY] [planMetadataUtils.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/planMetadataUtils.ts)

- Update the matching regexes in `parsePlanMetadata`:
  - `Manual Complexity Override`: Match `\*\*Manual Complexity Override(?:\*\*:\s*|:\*\*)\s*`
  - `Complexity`: Match `\*\*Complexity(?:\*\*:\s*|:\*\*)\s*`
  - `Tags`: Match `\*\*Tags(?:\*\*:\s*|:\*\*)\s*`
  - `Project`: Match `\*\*Project(?:\*\*:\s*|:\*\*)\s*`

---

### UI Refresh Guard Case-Insensitivity

Make the workspace path comparison case-insensitive in the `refreshIfShowing` guard to prevent refresh failures due to path casing variations on macOS/Windows.

#### [MODIFY] [KanbanProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)

- Update `refreshIfShowing`:
  - Compare lowercase resolved paths:
    ```typescript
    const resolved = path.resolve(workspaceRoot).toLowerCase();
    if (this._currentWorkspaceRoot && path.resolve(this._currentWorkspaceRoot).toLowerCase() === resolved) {
        this._scheduleBoardRefresh(this._currentWorkspaceRoot);
    }
    ```
