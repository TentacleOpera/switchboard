# Fix Database & Sync Panel

The 'Database & Sync' panel in the sidebar is currently non-functional and aesthetically inconsistent. This plan addresses the CSS/HTML bugs preventing the UI from working correctly and provides a modernization pass to ensure a premium experience.

## User Review Required

> [!IMPORTANT]
> The "Quick Actions" will be rearranged into a 3-column grid to maximize space efficiency in the sidebar.

> [!NOTE]
> Database path updates will now trigger an immediate board refresh to ensure the change is visually confirmed.

## Proposed Changes

### Webview UI

#### [MODIFY] [implementation.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/implementation.html)
- Define missing CSS variables: `--bg-dim`, `--border-dim`.
- Fix typo: Change class `db-action-grid` to `db-quick-actions` (or vice versa) to enable the grid layout.
- Modernize the styling:
    - Update `.db-sync-fields` to use a glassmorphic background.
    - Update buttons to use the standard `action-btn` or `secondary-btn` styles.
    - Improve the "Quick Actions" layout with better icons and spacing.
- Ensure the toggle chevron rotates correctly and the animation is smooth.

### Extension Backend

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)
- Improve `testDbConnection` handler to provide more detailed error messages if the connection fails (e.g., "Permission denied", "Invalid path").
- Add a refresh signal to the webview after `editDbPath` or `setPresetDbPath` completes.
- Ensure `KanbanDatabase.invalidateWorkspace` is called whenever the path setting changes globally.

#### [MODIFY] [KanbanDatabase.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanDatabase.ts)
- Add a static method or logic to validate a path *before* attempting to initialize the DB, providing better feedback for the `testDbConnection` action.
