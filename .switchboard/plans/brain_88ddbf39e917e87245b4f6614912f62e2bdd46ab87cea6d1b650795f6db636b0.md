Bug Analysis: Terminal Startup Commands Not Saved During Bootstrapping
Root Cause
The bug is a timing/initialization issue in the onboarding flow. Here's what happens:

During Onboarding (lines 4182-4196 in implementation.html):
User enters CLI commands in onboarding inputs (onboard-cli-lead, onboard-cli-coder, etc.)
Clicks "SAVE & FINISH" → sends saveStartupCommands with commands
Backend saves to state.json ✓
handleOnboardingProgress receives step: 'cli_saved' and calls finishOnboarding
After Onboarding Completes (lines 4143-4147):
javascript
_setupComplete = true;
onboardingContainer.classList.add('hidden');
mainContainer.classList.remove('hidden');
BUG: The main UI is shown, but getStartupCommands is never requested!
Terminal Operations Section (lines 1623-1659):
The input fields with data-role attributes are empty (not pre-populated)
lastStartupCommands is still {} (initialized at line 2395)
When User Clicks "OPEN AGENT TERMINALS" (lines 2143-2199):
javascript
const commands = { ...lastStartupCommands };  // Empty!
document.querySelectorAll('input[type="text"][data-role]').forEach(input => {
    const role = input.dataset.role;
    commands[role] = input.value.trim();  // Overwrites with empty values!
});
vscode.postMessage({ type: 'saveStartupCommands', commands, ... });  // Saves empty!
createAgentGrid in extension.ts (lines 2185-2206):
Calls taskViewerProvider.getStartupCommands()
But the commands were just overwritten with empty strings!
The Fix
When onboarding completes, the code needs to request the startup commands so that:

lastStartupCommands is populated with saved values
Terminal Operations inputs are filled with the saved commands
Location: implementation.html line ~4143, in the toggleOnboarding function when needsSetup becomes false.

The fix is to add:

javascript
vscode.postMessage({ type: 'getStartupCommands' });
vscode.postMessage({ type: 'getVisibleAgents' });
This ensures the saved settings are loaded into the UI before the user can interact with the Terminal Operations section.
