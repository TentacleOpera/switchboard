# Fix Workspace Mapping Overrides

The user reported that workspace database mappings defined in the `setup.html` menu are being overridden unpredictably. They requested that the mappings defined in the workspace file be the definitive source of truth, removing any fragile folder-level overrides. 

## Background Context
Currently, the extension retrieves the `workspaceDatabaseMappings` setting using `vscode.workspace.getConfiguration('switchboard').get('workspaceDatabaseMappings')`. 
In VS Code's configuration cascade, folder-scoped `.vscode/settings.json` files override workspace-scoped (`.code-workspace`) settings. When a child repository (e.g., the `switchboard` repo itself) contains its own `.vscode/settings.json` with `workspaceDatabaseMappings` disabled or empty, it can override the master mapping configuration depending on the active context. This leads to the "bouncing" dropdown behavior and inconsistent state.

Additionally, an earlier agent identified a secondary bug in `_getWorkspaceItems`: `(m.workspaceFolders && m.workspaceFolders[0])`. In JavaScript, an empty array `[]` is truthy, so when a mapping has no child folders, it evaluates `[][0]` resulting in `undefined`, which fails silently and hides the workspace from the dropdown.

## Proposed Changes

We will introduce a central utility to retrieve the `workspaceDatabaseMappings` strictly from the Workspace or Global scope, explicitly ignoring any folder-scoped values (`workspaceFolderValue`). We will then update all references to use this unified source of truth.

### 1. `src/services/WorkspaceIdentityService.ts`
Add a new exported utility function that safely reads the configuration while bypassing folder overrides.

#### [MODIFY] [WorkspaceIdentityService.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/WorkspaceIdentityService.ts)
```typescript
export function getWorkspaceDatabaseMappingsConfig(): { enabled?: boolean; mappings?: WorkspaceDatabaseMapping[] } {
    const inspection = vscode.workspace.getConfiguration('switchboard').inspect<any>('workspaceDatabaseMappings');
    if (!inspection) {
        return { enabled: false, mappings: [] };
    }
    // Strictly use workspace value (from .code-workspace) or global value (User settings).
    // Explicitly ignore inspection.workspaceFolderValue to prevent child repo overrides.
    const val = inspection.workspaceValue !== undefined ? inspection.workspaceValue : inspection.globalValue;
    return val || { enabled: false, mappings: [] };
}
```
