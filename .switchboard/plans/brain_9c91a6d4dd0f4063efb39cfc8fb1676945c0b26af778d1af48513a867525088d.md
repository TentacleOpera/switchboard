# Kanban Board Column Header Buttons Improvements

Improve the visual layout, sizing, alignment, and background colors of the buttons in the "NEW" (Created) column header in the Switchboard Kanban webview (`kanban.html`).

## Goal

Unify the Created column header icon buttons under a consistent 24px square dark-theme style and move the backlog toggle to the left side of the header to prevent collision with the right-side action buttons.

## Metadata

- **Tags:** [frontend, UI, UX]
- **Complexity:** 3

## User Review Required

> [!NOTE]
> We are restructuring the header layout of the "NEW" column. The "BACKLOG" toggle button is being moved from the right side controls to the left side (positioned right next to the "NEW" / "BACKLOG" column title in a flex container). This ensures they never overlap or collide, regardless of the column width.
>
> All header icon buttons (Chat, Add Plan, and Import) are being unified under a new `24px` square button class `.column-header-btn` with a dark theme background, consistent borders, and hover effects, removing the mismatched sizing and glow background.

## Open Questions

None. The user's visual feedback is clear and the proposed changes directly address each issue:
- **Collision**: Addressed by grouping title and backlog toggle on the left using flex gap.
- **Size difference & Spacing**: Addressed by unifying all header actions to `24px` square `.column-header-btn` with a standard container gap.
- **Teal glow background**: Addressed by replacing the `.btn-add-plan` style with `.column-header-btn` (which has a dark theme background).

## Complexity Audit

### Routine
- Adding a new CSS class `.column-header-btn` modeled on existing `.mode-toggle` / `.complexity-routing-btn` patterns
- Replacing class names on 3 buttons in the Created column `rightSide` template literal
- Moving `backlogToggleBtn` from `rightSide` to the left-side title container
- Updating the event handler selector to include the new class
- Removing an inline style override on the import button icon

### Complex / Risky
- None

## Edge-Case & Dependency Audit

- **Race Conditions**: None. All changes are static HTML/CSS template modifications rendered on column rebuild.
- **Security**: No security implications. Purely visual changes.
- **Side Effects**: The `.btn-add-plan` CSS class must NOT be deleted — it is still used by `btn-add-project` (line 2128) in the settings strip. Only the column header buttons stop using it.
- **Dependencies & Conflicts**: The generic column icon button event handler at line 4299 uses `document.querySelectorAll('.column-icon-btn')`. The chat button (currently `.column-icon-btn`) is being changed to `.column-header-btn`. The selector must be updated to `'.column-icon-btn, .column-header-btn'` or the `chatCopyPrompt` action will silently stop firing.

## Dependencies

None.

## Adversarial Synthesis

Key risks: Event handler breakage if the `.column-icon-btn` selector isn't updated to include `.column-header-btn`; left-side DOM structure must explicitly wrap the title and toggle in a row-direction flex container (not just append to the existing column-direction container); `.btn-add-plan` CSS must be preserved for the settings strip `btn-add-project` button. Mitigations: Update the querySelectorAll selector, specify the DOM structure precisely, and keep `.btn-add-plan` CSS intact.
