# HTML-to-Markdown Clipboard Import and Template Removal

This plan implements a robust clipboard import feature that reads rich HTML formatting from the clipboard in the Kanban webview, converts it to Markdown (preserving headings, lists, bold/italic, code blocks, and links), and completely removes the default plan template auto-appending behavior from the backend.

## Proposed Changes

### Kanban Frontend (Webview)

#### [MODIFY] [kanban.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/kanban.html)
* Update the clipboard import button click event listener to:
  * Attempt to read clipboard items via `navigator.clipboard.read()`.
  * If successful and `text/html` is available, parse the HTML using a custom, lightweight DOM-to-Markdown converter (`convertHtmlToMarkdown`).
  * If modern clipboard access fails or HTML is not present, fall back to posting the request to the extension to read standard plain text.
  * Send the parsed Markdown text to the backend.

### Kanban Backend (Extension Host)

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)
* Update `importPlanFromClipboard` to receive the optionally converted markdown text from the webview. If not sent, fallback to standard `vscode.env.clipboard.readText()`.
* In `_createInitiatedPlan`, remove the `isFullPlan` check and the fallback behavior that appends `## Goal`, `## Proposed Changes`, etc. stub sections to the bottom of the imported plans. Always save the imported text exactly as-is.
