## Proposed Changes

### [MODIFY] planning.js (`src/webview/planning.js`)

Three surgical insertions to add `'online'` alongside existing `'local'`/`'research'` calls.

#### 1. Initialization — line 74 (after `applySidebarState('research', ...)`)

Add the `online` tab to the initialization block so sidebar state is correct on page load.

```js
// BEFORE (lines 72-75):
applySidebarState('local', state.docsListCollapsed);
applySidebarState('research', state.docsListCollapsed);
applySidebarState('html-preview', state.htmlPreviewCollapsed);

// AFTER:
applySidebarState('local', state.docsListCollapsed);
applySidebarState('research', state.docsListCollapsed);
applySidebarState('online', state.docsListCollapsed);
applySidebarState('html-preview', state.htmlPreviewCollapsed);
```

#### 2. Toggle handler — line 60 (inside `toggleSidebarCollapsed`, after `applySidebarState('research', ...)`)

Add the `online` tab so clicking the sidebar toggle button on the Online Docs tab actually collapses/expands the sidebar.

```js
// BEFORE (lines 58-60):
state.docsListCollapsed = !state.docsListCollapsed;
applySidebarState('local', state.docsListCollapsed);
applySidebarState('research', state.docsListCollapsed);

// AFTER:
state.docsListCollapsed = !state.docsListCollapsed;
applySidebarState('local', state.docsListCollapsed);
applySidebarState('research', state.docsListCollapsed);
applySidebarState('online', state.docsListCollapsed);
```

#### 3. Tab switch handler — line 124 (inside the tab click listener)

Add `'online'` to the condition that applies sidebar state when switching tabs.

```js
// BEFORE (line 124):
} else if (tabName === 'local' || tabName === 'research') {

// AFTER:
} else if (tabName === 'local' || tabName === 'research' || tabName === 'online') {
```

No other changes needed. The `getActiveTabName()` function already returns `'online'` correctly (it reads `data-tab` from the active button). The `online` tab falls into the else branch of `toggleSidebarCollapsed()` (since `'online' !== 'html-preview'`), so it correctly uses `docsListCollapsed`. Persistence logic is unchanged — `online` shares the same `docsListCollapsed` state key.
