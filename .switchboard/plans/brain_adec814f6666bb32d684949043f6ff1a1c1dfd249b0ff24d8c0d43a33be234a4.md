# Local Docs Sidebar Layout Refactoring

## Metadata
**Complexity:** 2
**Tags:** frontend, ui, ux, bugfix

## Goal

Currently in the Switchboard planning webview (`planning.html`), the "Manage Folders" button is positioned absolutely on top of the collapse/toggle sidebar button. This is visually confusing, creates overlapping click zones, and looks unpolished.

This plan details the changes to redesign the sidebar toolbar into a clean, flexbox-based layout containing a text-only button labeled "Manage Folders" next to the sidebar collapse button, positioning the folder and document lists directly underneath.

## User Review Required

> [!NOTE]
> The "Manage Folders" text-only button will be automatically hidden when the sidebar is collapsed to maintain the minimal 40px width of the collapsed sidebar. The toggle collapse button (`«` / `»`) will remain visible and centered when collapsed.

## Proposed Changes

### Webview

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)

- Update CSS styles for `#tree-pane`, `#tree-pane-online`, and `#tree-pane-html` to use symmetrical padding (e.g., `12px` instead of `12px 36px 12px 12px`) since we no longer need to dodge absolute positioned floating buttons.
- Update CSS styles for `.sidebar-toggle-row` to use flex layout (`display: flex`, `justify-content: flex-end`, `align-items: center`, `gap: 8px`) and normal flow layout (removing `position: absolute`).
- Update CSS styles for `.sidebar-folders-btn` to style it as a text button (with `margin-right: auto` to push the toggle collapse button to the right).
- Ensure `.sidebar-folders-btn` is hidden in collapsed view: add it to `.content-row.collapsed .sidebar-folders-btn { display: none !important; }`.

#### [MODIFY] [planning.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.js)

- Update the DOM creation logic inside `renderLocalDocs` to use a text-only label "Manage Folders" in the `foldersBtn` button, removing any SVG icons.
- Adjust `docList.style.paddingTop` from `'10px'` to `'0px'` since the sidebar header row is now in-flow and naturally pushes the list down.
