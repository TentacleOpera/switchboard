# Disable Eager Codebase Research in Consultation and Planning Mode

This change introduces strict instructions to prevent AI agents from performing eager codebase research (such as running searches, directory listings, or viewing files) during the onboarding phase of a consultation session. It also emphasizes the importance of a snappy first-turn greeting to minimize initial latency and token overhead.

## User Review Required

> [!IMPORTANT]
> The change affects the startup instructions of the "Consultation & Planning Mode" (the default chat view). Because this involves editing workflow/persona files, explicit user approval is required to proceed with modification of these files.

## Proposed Changes

### Configuration and Services

---

#### [MODIFY] [switchboard-chat.md](file:///Users/patrickvuleta/Documents/GitHub/switchboard/.agent/workflows/switchboard-chat.md)

Add **NO EAGER RESEARCH** under Critical Constraints and Rule 9 under Rules.

##### Under `Critical Constraints` (after `NO EAGER CONTEXT` / line 12):
```markdown
- **NO EAGER RESEARCH**: Do not run any codebase search, file-view, directory-listing, or other exploration tools during the initial onboarding step or until the user specifies a specific codebase issue/file. When starting a general consultation (e.g., 'PLANS TO DISCUSS: None'), your primary goal is a snappy response; simply greet the user and wait for their input without performing unnecessary reasoning or research.
```

##### Under `Rules` (after `8. No Eager Context Adoption` / line 21):
```markdown
    9. **No Eager Research**: Do not run any search, file-view, directory-listing, or other codebase exploration tools when starting a general consultation session or until the user explicitly mentions a file path or specifies the problem details. Respond to the user immediately with a quick greeting rather than planning or researching on the first turn.
```
