# Prompts Tab Workflow File Path Add-on Integration

This plan outlines the changes required to add a 'Workflow File' add-on to all agents (built-in and custom) in the Prompts tab, allowing them to follow a workflow file step-by-step. The Planner agent UI will also be updated to match this checkbox + text input format, enabled by default with `.agent/workflows/improve-plan.md` pre-populated.

## User Review Required

> [!IMPORTANT]
> **Custom Agent Compatibility**:
> Custom agents will now display the "Workflow File" add-on in the Prompts tab. Settings updated here are saved under `roleConfig_custom_agent_<id>` and merged with their base configurations dynamically at prompt generation time.

## Proposed Changes

---

### UI Components & Configuration Defaults

#### [MODIFY] [sharedDefaults.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/sharedDefaults.js)
- Add a general default config for the new `workflowFilePathEnabled` and `workflowFilePath` settings for all roles in `DEFAULT_ROLE_CONFIG`.
- For `planner`, add `workflowFilePathEnabled: true` to its `addons` list.
- Add `{ id: 'workflowFilePath', label: 'Workflow File', tooltip: 'Read a workflow file and follow it step-by-step', type: 'file', default: false }` to the `ROLE_ADDONS` metadata list for all non-planner roles (e.g. `lead`, `coder`, `reviewer`, `tester`, `intern`, `analyst`, `ticket_updater`, `researcher`, `splitter`, `code_researcher`, `gatherer`).

#### [MODIFY] [kanban.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/kanban.html)
- **HTML Update**: Update the Planner agent's Workflow File subsection (lines 2462-2480) to include a checkbox `plannerWorkflowEnabled` and wrap the path input inside a display container `plannerWorkflowFilePathGroup`.
- **Add-on Rendering**: Update `renderRoleAddons` to support `addon.type === 'file'`, rendering a checkbox and a text input below it. Toggle display of the text input based on the checkbox state, and handle input change events to save configuration values.
- **Dynamic Add-ons for Custom Agents**: Update `renderRoleAddons` to return the new `workflowFilePath` addon if `role.startsWith('custom_agent_')` is true.
- **Role Change Handler**: Update `handleRoleChange` to:
  - Configure the `plannerWorkflowEnabled` checkbox and the container visibility for the planner role.
  - Enable display of prompt customization (`promptCustomization.style.display = 'block'`) for researcher and code_researcher roles, allowing them to configure addons too.
- **Event Listeners**: Register change listener for `plannerWorkflowEnabled` to toggle container display and save `workflowFilePathEnabled` in planner's addons.
- **Custom Agent Option Value Bugfix**: Update `updateCustomAgentsDropdown` to set `opt.value = agent.role || 'custom_agent_' + agent.id` instead of `agent.id`, ensuring consistency with the backend's custom agent detection.
- **Dynamic Config Retrieval**: Update `roleSelect` change listener and `settingResult` case for `selectedRole` to request and retrieve the custom agent configs if they aren't already loaded in the webview.
- **Crash Prevention**: Fix potential JSON parsing error in `settingResult` case for `roleConfig_` if `DEFAULT_CONFIG[role]` is undefined.

### Backend Logic & Prompt Generation

#### [MODIFY] [agentPromptBuilder.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/agentPromptBuilder.ts)
- Add `workflowFilePath?: string` and `workflowFilePathEnabled?: boolean` to `PromptBuilderOptions`.
- In `buildKanbanBatchPrompt`:
  - For the `planner` role: only prepend `Read ${workflowPath} and follow it step-by-step.\n\n` if `options?.workflowFilePathEnabled !== false`.
  - For all other roles: if `options?.workflowFilePathEnabled` is true and `options?.workflowFilePath` is non-empty, prepend the `Read ${options.workflowFilePath} and follow it step-by-step.\n\n` instruction to the final composed prompt string.
- In `buildCustomAgentPrompt`:
  - Support prepending `Read ${addons.workflowFilePath} and follow it step-by-step.\n\n` if `addons?.workflowFilePathEnabled` is true and `addons?.workflowFilePath` is non-empty.

#### [MODIFY] [KanbanProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)
- In `_getPromptsConfig`:
  - Resolve `workflowFilePathEnabledByRole` and `workflowFilePathByRole` maps for all roles from their role configs and VS Code workspace settings.
- In `generateUnifiedPrompt`:
  - Pass the resolved `workflowFilePathEnabled` and `workflowFilePath` values into `resolvedOptions` for all roles.
  - For custom agents: retrieve and merge their base `agentConfig.addons` with the prompts-specific `roleConfig_custom_agent_...` addons.
