# Prompts Tab Workflow File Path Add-on Integration

This plan outlines the changes required to add a 'Workflow File' add-on to all agents (built-in and custom) in the Prompts tab, allowing them to follow a workflow file step-by-step. The Planner agent UI will also be updated to match this checkbox + text input format, enabled by default with `.agent/workflows/improve-plan.md` pre-populated. The existing "Custom Workflow Path" field in the Agents tab custom agent form is removed — the Prompts tab becomes the single source of truth for workflow file paths across all agents.

> **Note**: A future plan will unify all custom agent addons (the `ca-addon-*` fields in the Agents tab) into the same `roleConfig` / `renderRoleAddons` system used by the Prompts tab, eliminating `CustomAgentAddons` and `parseCustomAgentAddons` entirely. This plan only addresses the workflow file path field.

## Goal

Add a "Workflow File" checkbox + text-input add-on to every built-in and custom agent in the Prompts tab, add an enable/disable checkbox to the Planner's existing workflow file section, and remove the redundant "Custom Workflow Path" field from the Agents tab custom agent form.

## Metadata

- **Tags:** [frontend, backend, UI, workflow]

## User Review Required

> [!IMPORTANT]
> **Custom Agent Workflow Path Migration**:
> The "Custom Workflow Path" field in the Agents tab custom agent form (`ca-addon-workflow-path`) is being removed. Workflow file paths for custom agents are now configured exclusively via the "Workflow File" add-on in the Prompts tab, stored under `roleConfig_custom_agent_<id>`. This consolidates workflow path configuration into a single UI surface. The existing `customWorkflowPath` field in `CustomAgentAddons` and its handling in `buildCustomAgentPrompt` are removed.

## Complexity Audit

### Routine
- Adding `workflowFilePathEnabled` and `workflowFilePath` fields to `DEFAULT_ROLE_CONFIG` and `PromptBuilderOptions` (follows existing addon pattern)
- Adding `type: 'file'` addon entries to `ROLE_ADDONS` for non-planner roles (follows existing `type: 'radio'` pattern)
- Adding `workflowFilePathEnabledByRole` and `workflowFilePathByRole` maps to `_getPromptsConfig` (follows existing `cavemanOutputByRole` pattern)
- Passing resolved values through `generateUnifiedPrompt` into `resolvedOptions` (follows existing pattern)
- Removing `ca-addon-workflow-path` HTML element and its read/write in the Agents tab form (straightforward deletion)
- Removing `customWorkflowPath` from `CustomAgentAddons` interface and `parseCustomAgentAddons`
- Bugfix: `updateCustomAgentsDropdown` option value (one-line change)
- Bugfix: `DEFAULT_CONFIG[role]` undefined guard in `settingResult` handler (one-line change)

### Complex / Risky
- New `type: 'file'` rendering in `renderRoleAddons` — checkbox toggles text input visibility, input change saves config; must not break existing `type: 'radio'` and default checkbox rendering paths
- Custom agent addon injection — `ROLE_ADDONS[role]` is undefined for custom agents; must synthesize the `workflowFilePath` addon dynamically before rendering
- Planner backward compatibility — currently workflow is always prepended; new `workflowFilePathEnabled` field must default to enabled (true) for existing saved configs that lack this field

## Edge-Case & Dependency Audit

- **Race Conditions**: None identified. All UI state changes are synchronous and saved immediately via `saveRoleConfig`.
- **Security**: Workflow file paths are user-supplied strings injected into prompts. No filesystem access is performed based on these paths at prompt-generation time (the agent reads the file itself). The existing "Validate" button for the planner checks file existence; the new `type: 'file'` addon does not include validation (deferred — agent will report if file is missing).
- **Side Effects**: Enabling `workflowFilePath` for a role that previously had no workflow instruction will change the prompt structure. The prepend instruction `Read <path> and follow it step-by-step.\n\n` is added before all other prompt content. Removing `customWorkflowPath` from custom agents will cause any agent that relied on it to lose its workflow instruction until the user re-configures it via the Prompts tab addon.
- **Dependencies & Conflicts**: None. The old `customWorkflowPath` and the new `workflowFilePath` are not coexisting — the old field is fully removed.

## Dependencies

None

## Adversarial Synthesis

Key risks: (1) users with existing custom agents configured via `customWorkflowPath` in the Agents tab will lose their workflow setting after this change — the Prompts tab addon defaults to disabled, (2) backward compatibility for planner workflow which is currently always-on, (3) `ROLE_ADDONS[role]` being undefined for custom agents requires dynamic addon synthesis. Mitigations: the `customWorkflowPath` removal is a clean break since the field was behind a collapsed `<details>` and likely rarely used; use `!== false` check for planner default; synthesize addon array in `renderRoleAddons` for custom agent roles.
