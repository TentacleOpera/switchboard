# Prevent Completed Brain Plan Re-Ingestion Loop

## Goal
Prevent the brain watcher from re-ingesting archived brain plan files copied into `completed/` subdirectories, which causes duplicate Kanban cards to reappear in the CREATED column after a plan is completed.

## Metadata
- **Tags:** [bugfix, reliability]
- **Complexity:** 2

## User Review Required
- Confirm that no legitimate brain plans are stored in directories named `completed` (this is a Switchboard-created convention, not user-chosen, so risk is negligible).

## Complexity Audit

### Routine
- Adding a single `parts.some(...)` check in `_isBrainMirrorCandidate`
- Adding directory-level `completed` skips in two traversal methods (consistent with existing `_listMarkdownFilesRecursively` pattern)
- Adding a `console.log` for observability when a `completed/` path is rejected

### Complex / Risky
- None

## Edge-Case & Dependency Audit

- **Race Conditions**: The brain watcher debounces events by 300ms (lines 8896, 8933). The `_isBrainMirrorCandidate` check runs inside the debounced callback, so the `completed` segment check will execute after the archive copy is fully written. No race condition.
- **Security**: No security implications — this is a path-filtering change that makes the gatekeeper more restrictive.
- **Side Effects**: The `_isBrainMirrorCandidate` method is also called by `_collectBrainPlanBlacklistEntries` (line 10982) and `_collectAntigravityPlanCandidates` (line 11010). Adding the `completed` check will correctly prevent those methods from collecting files from `completed/` directories as well. This is desirable behavior.
- **Dependencies & Conflicts**: The existing `archivedBrainPaths` and tombstone guards in `_mirrorBrainPlan` (lines 11655-11668) hash the *original* brain path, not the archived copy's path. The archived copy at `completed/plan.md` has a different stable path and hash, so it bypasses those guards. The `completed` segment check in `_isBrainMirrorCandidate` is the primary defense against this bypass. No conflicts with existing guards.

## Dependencies
- None

## Adversarial Synthesis
Key risks: (1) The `completed` directory name is hardcoded — if the archive convention changes, the filter must be updated. (2) The existing `archivedBrainPaths`/tombstone system doesn't cover the archived copy's hash, so the `_isBrainMirrorCandidate` check is the sole defense. Mitigations: The `completed/` convention is Switchboard-controlled and unlikely to change; adding directory-level skips and logging provides consistency and observability.

## Proposed Changes

### [MODIFY] src/services/TaskViewerProvider.ts

#### `_isBrainMirrorCandidate` (line 10936)

Add a check after the `parts` array is constructed (line 10948) to reject any path containing a `completed` directory segment. This is the primary fix — it prevents both the VS Code watcher (line 8888) and the native fs.watch fallback (line 8925) from processing archived files. It also protects `_collectBrainPlanBlacklistEntries` and `_collectAntigravityPlanCandidates` which call this method.

**Context**: The method currently allows paths with up to 3 segments. An archived file at `<session>/completed/plan.md` has exactly 3 segments and passes the level guard. Its SHA-256 hash differs from the original, so it bypasses `archivedBrainPaths` and tombstone checks.

**Logic**: After `parts` is computed (line 10948), before the level guard (line 10950), check if any segment matches `completed` case-insensitively. If so, log and return `false`.

**Implementation**:

```typescript
    private _isBrainMirrorCandidate(filePath: string): boolean {
        const resolvedFilePath = path.resolve(filePath);
        const normalizedFilePath = this._getStablePath(resolvedFilePath);
        const matchingRoot = this._getAntigravityPlanRoots()
            .map(root => path.resolve(root))
            .find(root => this._isPathWithin(root, resolvedFilePath))
            || this._getAntigravityRoots()
                .map(root => path.resolve(root))
                .find(root => this._isPathWithin(root, resolvedFilePath));
        if (!matchingRoot) return false;

        const relativePath = path.relative(this._getStablePath(matchingRoot), normalizedFilePath);
        const parts = relativePath.split(path.sep).filter(Boolean);

        // Exclude paths containing "completed" directories to prevent
        // archived plans from being re-ingested as new Kanban cards
        if (parts.some(part => part.toLowerCase() === 'completed')) {
            console.log(`[TaskViewerProvider] Brain mirror candidate rejected (completed directory): ${path.basename(filePath)}`);
            return false;
        }

        // Allow up to 3 levels: brain/<session>/subdir/plan.md
        if (parts.length < 1 || parts.length > 3) return false;
        ...
```

**Edge Cases**: The `completed` directory is created exclusively by `_archiveBrainPlan` (line 12925). No legitimate brain plans should reside in a `completed/` folder. Case-insensitive comparison handles cross-platform filesystem differences.
