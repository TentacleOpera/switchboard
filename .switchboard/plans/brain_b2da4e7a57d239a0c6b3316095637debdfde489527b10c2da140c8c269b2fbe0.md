# Remove Read Receipts + Add Tiered Interval Warnings

The Autoban engine currently asks the agent to write a completion signal ("read receipt") at the end of every batch. This is fragile — if the agent is interrupted, runs out of context, or simply forgets, the pipeline silently stalls. At the same time, there is no feedback when a user sets an interval that is likely too short for their batch size.

This plan removes the read receipt instruction and adds a **tiered warning system** to the UI. Floors are reference thresholds only — users are never blocked or autocorrected. A user running rapid-fire low-complexity plans (e.g. "change button colour") can legitimately use a 1-minute interval; the UI will inform them of the risk without overriding their intent. The backend enforces only a 1-minute absolute minimum to prevent obviously degenerate configs.

Warning evaluation function (called from `onChange` AND from `batchSelect.addEventListener('change')`):
```js
function evaluateIntervalWarning(column, val, batchSize, floorPerPlan, minInput, warnLabel) {
    const floor = batchSize * floorPerPlan;
    const ratio = val / floor;
    if (ratio < 0.5) {
        minInput.style.borderColor = 'var(--accent-red)';
        warnLabel.textContent = `⚠ ${floor}m recommended for batch ${batchSize} — plans may not finish in time`;
        warnLabel.style.color = 'var(--accent-red)';
    } else if (ratio < 1.0) {
        minInput.style.borderColor = 'var(--accent-orange)';
        warnLabel.textContent = `Low-complexity plans only — recommended ${floor}m for batch ${batchSize}`;
        warnLabel.style.color = 'var(--accent-orange)';
    } else {
        minInput.style.borderColor = '';
        warnLabel.textContent = '';
    }
}

// onChange:
const val = Math.max(1, parseInt(minInput.value, 10) || defaultMin);
autobanState.rules[column].intervalMinutes = val;
minInput.value = String(val);
evaluateIntervalWarning(column, val, autobanState.batchSize, floorPerPlan, minInput, warnLabel);
```

**Batch size change re-evaluation** — add to `batchSelect.addEventListener('change')`:
```js
batchSelect.addEventListener('change', () => {
    autobanState.batchSize = parseInt(batchSelect.value, 10) || 3;
    // Re-evaluate all interval warnings with new batch size
    columnTransitions.forEach(({ column, floorPerPlan }) => {
        const input = document.getElementById(`interval-input-${column}`);
        const label = document.getElementById(`warn-label-${column}`);
        if (input && label) {
            evaluateIntervalWarning(column, autobanState.rules[column]?.intervalMinutes || 0,
                autobanState.batchSize, floorPerPlan, input, label);
        }
    });
    emitAutobanState();
});
```
*(Requires adding `id` attributes to `minInput` and `warnLabel` elements using the column name as key.)*

**Warning tiers:**

| Input vs recommended floor | Border | Message |
|---|---|---|
| ≥ floor | none | (silent) |
| 50–99% of floor | orange | "Low-complexity plans only — recommended Xm" |
| < 50% of floor | red | "Plans may not finish in time" |
