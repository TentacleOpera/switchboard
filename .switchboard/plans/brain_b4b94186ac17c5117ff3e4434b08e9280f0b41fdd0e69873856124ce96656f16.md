# Implementation Plan - Fix Workspace Mappings Configuration Scope Mismatch

We will fix the configuration scope mismatch in `SetupPanelProvider` handlers that causes workspace mappings to be wiped when the enabled state is toggled or mappings are saved. We will also update the corresponding tests to verify this target scope.

## Proposed Changes

### Switchboard Extension Service

#### [MODIFY] [SetupPanelProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/SetupPanelProvider.ts)

We will modify three config update handlers to check if `folderUri` is defined and use `vscode.ConfigurationTarget.WorkspaceFolder`, falling back to `vscode.ConfigurationTarget.Workspace` if not.

- **Change 1:** In case `'setWorkspaceMappingEnabled'`, change:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      { ...current, enabled },
      vscode.ConfigurationTarget.Workspace
  );
  ```
  to:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      { ...current, enabled },
      folderUri ? vscode.ConfigurationTarget.WorkspaceFolder : vscode.ConfigurationTarget.Workspace
  );
  ```

- **Change 2:** In case `'saveWorkspaceMappings'`, change:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      incoming,
      vscode.ConfigurationTarget.Workspace
  );
  ```
  to:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      incoming,
      folderUri ? vscode.ConfigurationTarget.WorkspaceFolder : vscode.ConfigurationTarget.Workspace
  );
  ```

- **Change 3:** In case `'initializeWorkspaceDatabase'`, change:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      { ...current, enabled: true, mappings: updatedMappings },
      vscode.ConfigurationTarget.Workspace
  );
  ```
  to:
  ```typescript
  await config.update(
      'workspaceDatabaseMappings',
      { ...current, enabled: true, mappings: updatedMappings },
      folderUri ? vscode.ConfigurationTarget.WorkspaceFolder : vscode.ConfigurationTarget.Workspace
  );
  ```

#### [MODIFY] [workspace-mappings-settings-sync.test.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/test/workspace-mappings-settings-sync.test.ts)

We will update the test suite to assert the target configuration targets `WorkspaceFolder` when `folderUri` is mocked and resolved.

- **Change 1:** In test `'2. SetupPanelProvider - setWorkspaceMappingEnabled handler writes to config using folderUri'`, change:
  ```typescript
  assert.strictEqual(configUpdatedData[0].target, vscode.ConfigurationTarget.Workspace);
  ```
  to:
  ```typescript
  assert.strictEqual(configUpdatedData[0].target, vscode.ConfigurationTarget.WorkspaceFolder);
  ```

- **Change 2:** In test `'3. SetupPanelProvider - saveWorkspaceMappings handler writes to config using folderUri'`, change:
  ```typescript
  assert.strictEqual(configUpdatedData[0].target, vscode.ConfigurationTarget.Workspace);
  ```
  to:
  ```typescript
  assert.strictEqual(configUpdatedData[0].target, vscode.ConfigurationTarget.WorkspaceFolder);
  ```
