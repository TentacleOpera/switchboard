# Fix Setup Panel Organization and Implement Modern Autosave

## Goal
The Switchboard setup and implementation panels currently suffer from several UI inconsistencies, misleading labels, and a frustrating saving mechanism where global saves are trapped inside local accordions. This plan resolves these issues by reorganizing settings logically and shifting the entire Setup page to a modern, frictionless "Autosave" architecture.

Success means the central setup panel becomes the sole owner of the Jules auto-sync setting, the affected configuration controls no longer depend on the misleading global save/apply buttons, and autosave persists the same state without clobbering sidebar-only flows or already-landed setup-panel behavior.

## Edge-Case & Dependency Audit
- **Race Conditions:** The main risk is duplicate or stale writes. `src/webview/setup.html` currently posts `saveStartupCommands` only on a button click, while git-ignore has its own apply button and the sidebar still reads `#jules-auto-sync-toggle` during terminal launch. Autosave must therefore (1) dedupe identical payloads, (2) cancel pending debounce timers before blur-triggered saves, (3) suppress outbound autosave while inbound state messages are hydrating DOM fields, and (4) stop the sidebar from persisting a stale Jules value when `OPEN AGENT TERMINALS` is clicked.
- **Security:** No new external capability is introduced, but the autosave path must keep validation strict. Only `localExclude`, `gitignore`, and `none` are valid git-ignore strategies; ignore rules must remain a trimmed string array; and autosave must not start posting integration tokens or database paths simply because those inputs also live in the setup panel.
- **Side Effects:** Autosave removes the user's explicit "I saved this" step. The implementation should therefore replace the two removed buttons with a passive status line in `src/webview/setup.html` (for example `SAVING...`, `SAVED`, `FAILED`) rather than silently discarding feedback. Also note that plan-ingestion-folder validation currently shows warnings from `handleSaveStartupCommands`; autosave should avoid repeating the same warning for the same unchanged invalid value.
- **Dependencies & Conflicts:**
  - `switchboard-get_kanban_state` returned **no New plans** and only this plan in **Planned**, so there is no active New/Planned dependency or active parallel owner on the live Kanban board.
  - A scan of `.switchboard/plans/` found adjacent same-file work in `add_git_ignore_ui_to_setup_menu.md`, `move_open_setup_button_in_terminal_operations.md`, `remove_central_setup_panel_header.md`, `feature_plan_20260411_014806_move_configuration_components_to_central_setup_panel.md`, and `setup_view_improvements.md`. **Clarification:** these are reviewed/adjacent merge hotspots rather than active Kanban blockers, but the implementer must diff current source before editing `src/webview/setup.html`, `src/webview/implementation.html`, and `src/test/setup-panel-migration.test.js` so already-landed changes are not unintentionally reverted.

## Adversarial Synthesis
### Grumpy Critique
> Marvelous. The draft says "just make it autosave" in a setup panel that currently saves half its state with one blunt button, hides the Jules toggle in the wrong webview, persists git-ignore through a completely separate apply path, and keeps custom-agent mutations in memory until somebody finally remembers to press SAVE CONFIGURATION. Rip out the buttons without re-plumbing those dependencies and you'll ship the software equivalent of a fake elevator door: it looks modern, it feels sleek, and it does absolutely nothing when it matters.
>
> And no, this is not a "small UI cleanup." `src/webview/setup.html` is now the control surface for agent commands, visibility, git-ignore behavior, Team Lead orchestration settings, plan-ingestion config, and custom-agent state. If you slap a generic `querySelectorAll('input, textarea, select')` autosave on top of that mess, you will accidentally start persisting integration tokens, database-path drafts, and every transient modal field the moment somebody pauses to think. Congratulations, you have reinvented data corruption with nicer spacing.
>
> The only defensible implementation is painfully explicit: define exactly which controls are autosaved, guard hydration so inbound messages do not echo back out, autosave custom-agent mutations now that the global save button is gone, and collapse git-ignore persistence behind the same backend save path so blur/debounce does not duel with an old apply-button code path. Anything less is theater.

### Balanced Response
The critique is correct about the hidden coupling, so the implementation below narrows autosave to the specific configuration controls already implied by the draft: prompt-control toggles, built-in/custom agent visibility and commands, Team Lead orchestration settings, plan-ingestion folder, git-ignore config, and the relocated Jules auto-sync toggle. It explicitly excludes onboarding inputs, integration-token handshakes, database-location actions, and prompt-override modal submission.

To keep the behavior safe, the plan adds a dedicated autosave collector in `src/webview/setup.html`, uses explicit selector lists (or equivalent field markers) instead of blanket listeners, dedupes payloads across `input` + `blur`, autosaves custom-agent mutations when `lastCustomAgents` changes, and moves git-ignore persistence behind a shared helper in `src/services/TaskViewerProvider.ts`. That preserves the current product scope while removing the draft's misleading manual save bottlenecks.

```js
// setup.html :: autosave helpers
let isHydratingSetupState = false;
let pendingSetupAutosaveTimer = null;
let lastSentSetupSignature = '';
let lastAckedSetupSignature = '';

function setSetupSaveStatus(label, isError = false) {
    const node = document.getElementById('setup-save-status');
    if (!node) return;
    node.textContent = label;
    node.style.color = isError ? 'var(--accent-red)' : 'var(--text-secondary)';
}

function collectSetupSavePayload() {
    const commands = {
        ...lastStartupCommands,
        'team-lead': teamLeadCommandInput?.value.trim() || ''
    };
    document.querySelectorAll('#agents-fields input[type="text"][data-role]').forEach(input => {
        const role = input.dataset.role;
        if (role) commands[role] = input.value.trim();
    });

    const visibleAgents = {
        ...getCustomVisibleAgentsPatch(),
        'team-lead': !!teamLeadVisibleToggle?.checked
    };
    document.querySelectorAll('#agents-fields .agent-visible-toggle').forEach(toggle => {
        const role = toggle.dataset.role;
        if (role) visibleAgents[role] = toggle.checked;
    });

    return {
        type: 'saveStartupCommands',
        accurateCodingEnabled: !!document.getElementById('accurate-coding-toggle')?.checked,
        advancedReviewerEnabled: !!document.getElementById('advanced-reviewer-toggle')?.checked,
        leadChallengeEnabled: !!document.getElementById('lead-challenge-toggle')?.checked,
        aggressivePairProgramming: !!document.getElementById('aggressive-pair-toggle')?.checked,
        designDocEnabled: !!document.getElementById('design-doc-toggle')?.checked,
        designDocLink: lastDesignDocLink,
        planIngestionFolder: document.getElementById('plan-ingestion-folder-input')?.value.trim() || '',
        customAgents: lastCustomAgents,
        commands,
        visibleAgents,
        teamLeadComplexityCutoff: Number(teamLeadComplexityCutoffInput?.value || 0),
        teamLeadKanbanOrder: Number(teamLeadKanbanOrderInput?.value || 170),
        julesAutoSyncEnabled: !!document.getElementById('jules-auto-sync-toggle')?.checked,
        gitIgnoreStrategy: document.getElementById('git-ignore-strategy')?.value || 'localExclude',
        gitIgnoreRules: sanitizeGitIgnoreRules(document.getElementById('git-ignore-rules')?.value || '')
    };
}

function flushSetupAutosave(reason) {
    if (isHydratingSetupState) return;
    if (pendingSetupAutosaveTimer) {
        clearTimeout(pendingSetupAutosaveTimer);
        pendingSetupAutosaveTimer = null;
    }
    const payload = collectSetupSavePayload();
    const signature = JSON.stringify(payload);
    if (signature === lastSentSetupSignature || signature === lastAckedSetupSignature) return;
    lastStartupCommands = payload.commands;
    lastVisibleAgents = { ...lastVisibleAgents, ...payload.visibleAgents };
    lastSentSetupSignature = signature;
    setSetupSaveStatus(`SAVING (${reason})...`);
    vscode.postMessage(payload);
}

function queueSetupAutosave(reason, immediate = false) {
    if (isHydratingSetupState) return;
    if (immediate) {
        flushSetupAutosave(reason);
        return;
    }
    if (pendingSetupAutosaveTimer) clearTimeout(pendingSetupAutosaveTimer);
    pendingSetupAutosaveTimer = setTimeout(() => flushSetupAutosave(reason), 750);
}

});
```

```ts
private _normalizeGitIgnoreConfig(
    rawStrategy: unknown,
    rawRules: unknown
): { strategy: 'localExclude' | 'gitignore' | 'none'; rules: string[] } {
    const strategy = rawStrategy === 'gitignore' || rawStrategy === 'none' ? rawStrategy : 'localExclude';
    const rules = Array.isArray(rawRules)
        ? Array.from(new Set(rawRules.map(rule => String(rule).trim()).filter(Boolean)))
        : [];
    return { strategy, rules };
}

private async _persistGitIgnoreConfig(
    rawStrategy: unknown,
    rawRules: unknown,
    options: { emitApplyResult: boolean }
): Promise<void> {
    const { strategy, rules } = this._normalizeGitIgnoreConfig(rawStrategy, rawRules);
    const config = vscode.workspace.getConfiguration('switchboard.workspace');
    await config.update('ignoreStrategy', strategy, vscode.ConfigurationTarget.Workspace);
    await config.update('ignoreRules', rules, vscode.ConfigurationTarget.Workspace);
    this._postSharedWebviewMessage({ type: 'gitIgnoreConfig', strategy, rules });
    if (options.emitApplyResult) {
        this._postSharedWebviewMessage({ type: 'saveGitIgnoreConfigResult', success: true });
    }
}

public async handleSaveGitIgnoreConfig(data: any): Promise<void> {
    try {
        await this._persistGitIgnoreConfig(data?.strategy, data?.rules, { emitApplyResult: true });
    } catch (error) {
        console.error('[Switchboard] Failed to save git ignore config:', error);
        this._postSharedWebviewMessage({ type: 'saveGitIgnoreConfigResult', success: false });
        throw error;
    }
}
```
