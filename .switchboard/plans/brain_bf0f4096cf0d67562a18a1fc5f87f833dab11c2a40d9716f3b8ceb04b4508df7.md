# Kanban Toggle Opacity and Click Animation Enhancements

## Metadata
**Complexity:** 2
**Tags:** frontend, ui, ux, refactor

## Goal
The toggles on the Kanban view (coder collapsed columns, CLI triggers, and complexity routing) currently use a prominent, permanent glowing box-shadow (`--glow-teal`) to indicate they are active. This is visually overwhelming and inconsistent with other inactive states (e.g., the delete project button, which uses reduced opacity when inactive). 

Additionally, only the "Copy chat prompt" button exhibits a pop/flash scaling animation when clicked, creating an inconsistent and unpolished user experience across the board.

### Background and Root Cause Analysis
- The CSS styles for `.strip-icon-btn.is-active`, `.kanban-sub-bar .strip-icon-btn.is-active`, and `.complexity-routing-btn.is-active` permanently apply `box-shadow: var(--glow-teal)` and background highlights when active, rather than restricting them to the hover state.
- Some buttons do not toggle an `.is-off` class (only `.is-active`), resulting in no distinct style class being applied when they are in the off/inactive state.
- Button click flash animations are applied via ad-hoc calls to `flashIconBtn(btn)` inside specific individual click handlers, rather than using a global delegated event listener on the document level.

To fix this, we will:
1. Update active styling to have normal opacity (1.0) and remove permanent glow/background highlights.
2. Update inactive/off styling to have reduced opacity (0.5) and grayscale filters.
3. Apply glow effects on hover for all buttons.
4. Implement a global `click` event listener that applies the flash animation to all clickable buttons and button-like divs on the board.

## User Review Required
No breaking changes or risky migrations are required. The changes are purely cosmetic and enhance usability.

> [!NOTE]
> All button elements (and the complexity routing div) will now scale slightly and flash when clicked, matching the behavior of the copy chat prompt button.

### Kanban Webview

#### [MODIFY] [kanban.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/kanban.html)
- **CSS Styles**:
  - Update `.strip-icon-btn.is-active` and `.kanban-sub-bar .strip-icon-btn.is-active` to have transparent background and no shadow, and define `:hover` states to apply the background highlight and `var(--glow-teal)` shadow.
  - Update `.complexity-routing-btn.is-active` to have transparent background and no shadow, and define `:hover` state to apply the background and `var(--glow-teal)` shadow.
  - Set `opacity: 0.5` on `.strip-icon-btn.is-off` and `.complexity-routing-btn.is-off`.
  - Add `box-shadow: var(--glow-teal)` to `.strip-icon-btn:hover:not(:disabled)` and `.complexity-routing-btn:hover` to apply glow effects on hover.
- **HTML / Templates**:
  - Update `complexity-routing-toggle` rendering logic in the column header generator to use `'is-active'` or `'is-off'` class instead of just `'is-active'` or `''`.
- **Javascript Logic**:
  - Update `updateComplexityRoutingToggleUi()` to toggle `.is-off` class based on `!dynamicComplexityRoutingEnabled`.
  - Update the `btn-collapse-coders` event listener and restoration block to toggle `.is-off` class based on `!collapseCodersEnabled`.
  - Remove manual `flashIconBtn(btn)` calls from specific event listeners (the copy chat prompt handler and column icon button handlers) to avoid duplicate execution.
  - Add a global `click` event listener in `DOMContentLoaded` that listens for clicks on any `button` or `.complexity-routing-btn` element and triggers `flashIconBtn(btn)`.
