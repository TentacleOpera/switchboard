# Reorganize Kanban Webview Control Strip Icon Positions and Assets

Reorganize the controls within the first function bar (`.controls-strip`) in `src/webview/kanban.html`. Update the icons used for the CLI triggers and collapse coders actions, and move both to the left section of the bar, directly to the left of the delete project button. The pair programming mode select dropdown will remain on the right.

Additionally, adjust the sizes of the `.strip-icon-btn` class and its child images to scale them down so they mesh seamlessly with the existing `+` (`.btn-add-plan`) and `🗑` (`.strip-btn`) project controls.

## Goal

Reorganize the kanban controls strip by moving the CLI Triggers and Collapse Coders icon buttons to the left group (alongside workspace/project selectors), swapping their icons to more semantically appropriate assets, and scaling all `.strip-icon-btn` elements down to match the visual height of adjacent text-based strip controls.

The general `.strip-icon-btn` change will not visually affect sub-bar buttons because the sub-bar override takes CSS specificity precedence for container properties.
- **Dependencies & Conflicts:** The `{{ICON_CLI}}` template is already defined in `KanbanProvider.ts` (line 6641) and already used in `kanban.html` JS (line 3355). No server-side changes are needed. The `{{ICON_28}}` template is already defined (line 6635). All icon templates are pre-existing.

## Adversarial Synthesis

Key risks: (1) The `.kanban-sub-bar .strip-icon-btn img` CSS override at 16px will conflict with the planned 14px inline style on `#btn-autoban` if not updated — the inline style wins on specificity but creates a maintenance inconsistency. Mitigations: Update the sub-bar CSS image size to 14px explicitly; accept the 2px difference with the `+` button as expected for text-vs-icon mixing.
