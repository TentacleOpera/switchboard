# Document Card Layout and Antigravity Header Cleanup

## Goal
Make sidebar document cards larger and vertically stacked (matching Kanban plans cards) across all planning tabs (Local Docs, Online Docs, Design System, HTML Previews), and remove the unwanted Antigravity session codename teal headers while displaying session times directly on the artifact cards.

**Core Problem**: Document cards in the sidebar currently use horizontal flex layout (`align-items: center`) which crams the title, subtitle, and action buttons into a single row. This makes titles truncate aggressively and feels cramped compared to the Kanban plan cards which use a more spacious vertical layout. Additionally, Antigravity sessions display a redundant teal "Session: <codename>..." header row per session that adds visual noise without useful information — the session timestamp is the actually useful metadata and should appear directly on each artifact card as a subtitle.

## Metadata
- **Tags:** frontend, ui, ux, bugfix
- **Complexity:** 3

## User Review Required
- Confirm that removing the session codename header entirely (vs. keeping it as a smaller/less prominent element) is the desired behavior.
- Confirm that `Session at <time>` subtitle format on artifact cards is acceptable (vs. just the time, or a different label).

## Complexity Audit

### Routine
- Adding a CSS rule for `.tree-node:has(.card-text)` to switch to vertical column layout
- Removing the `sessionRow` element creation in `renderAntigravitySessions`
- Passing session timestamp as subtitle to `renderDocCard` for antigravity artifacts
- Adding `.card-text`, `.card-actions`, `.card-subtitle` to collapsed sidebar hide list
- Removing dead `.antigravity-session-ts` CSS class
- Adjusting `.antigravity-artifact-node` margin-left from 24px to 12px

### Complex / Risky
- None

## Edge-Case & Dependency Audit

- **Race Conditions**: None — all changes are synchronous DOM/CSS mutations during render.
- **Security**: No security implications — purely visual CSS and DOM changes.
- **Side Effects**: The `:has(.card-text)` selector will affect ALL `.tree-node` elements that contain `.card-text`, which includes local-folder, design-folder, html-folder, online source, and antigravity artifact cards. This is the intended behavior (uniform card layout across all tabs). Folder subheaders and source headers do NOT contain `.card-text`, so they are unaffected.
- **Dependencies & Conflicts**: The collapsed sidebar currently does not hide `.card-text` or `.card-actions` (only hides `.label`, `.file-name`, etc.). This is a pre-existing gap that becomes more visible with vertical card layout. Must add these to the collapsed hide list to prevent broken rendering at 40px sidebar width.

## Dependencies
- None

## Adversarial Synthesis
Key risks: (1) `:has(.card-text)` selector could unintentionally affect future card types if `.card-text` is added to non-document nodes — mitigated by `.card-text` being exclusively created inside `renderDocCard`. (2) Collapsed sidebar at 40px width will break vertical cards unless `.card-text`/`.card-actions` are hidden — mitigated by adding them to the collapsed hide list. (3) Removing session codename headers loses the ability to distinguish same-time sessions — mitigated by session name being rarely useful and date+time being sufficient for identification.

## Proposed Changes

### [MODIFY] planning.html (`src/webview/planning.html`)

#### Context
The `.tree-node` base style (line 779) uses `display: flex; align-items: center;` — a horizontal layout. Document cards rendered via `renderDocCard` contain `.card-text` and `.card-actions` children that should stack vertically. The Kanban plan items (`.kanban-plan-item`, line 1653) already use a more spacious vertical layout with `padding: 10px 12px; gap: 10px;`.

#### Logic

1. **Add vertical layout rule for document cards** (insert after line ~810, after `.tree-node.selected` block):
   ```css
   /* Document cards: vertical column layout matching Kanban plan cards */
   .tree-node:has(.card-text) {
       flex-direction: column;
       align-items: stretch;
       padding: 10px 12px;
       gap: 8px;
   }
   ```
   This uses `:has()` (supported in Chromium/VS Code webview) to target only `.tree-node` elements that contain `.card-text`, which are exclusively created by `renderDocCard`. Folder subheaders, source headers, and other non-card tree nodes do not contain `.card-text` and are unaffected.

2. **Add collapsed sidebar hide rules** (append to the collapsed hide list at line ~282-294):
   ```css
   .content-row.collapsed .card-text,
   .content-row.collapsed .card-actions,
   .content-row.collapsed .card-subtitle {
       display: none !important;
   }
   ```
   Pre-existing gap: collapsed sidebar hides `.label` and `.file-name` but not `.card-text`/`.card-actions`. Without this, vertical cards at 40px width are broken.

3. **Remove dead `.antigravity-session-ts` CSS** (lines 1535-1541):
   Delete the entire `.antigravity-session-ts` rule block. The `sessionRow` element (the only consumer) is being removed from `renderAntigravitySessions`.

4. **Adjust `.antigravity-artifact-node` margin** (line 1543-1545):
   Change `margin-left: 24px` to `margin-left: 12px`. The 24px indent was designed to nest under the session header row. With the session header removed, artifacts sit directly under the date subheader — a lighter indent is more appropriate.

#### Edge Cases
- **Cyber theme overrides**: `.cyber-theme-enabled .tree-node:hover` and `.cyber-theme-enabled .tree-node.selected` (lines 1824-1830) apply box-shadow overrides. These work identically in column layout — no changes needed.
- **Theme overrides (Claude Terracotta, Slightly Darker Black)**: These override CSS custom properties only, not layout properties — unaffected.
- **`:has()` browser support**: Chromium has supported `:has()` since v105 (2022). VS Code webviews use Chromium — safe to use.

### [MODIFY] planning.js (`src/webview/planning.js`)

#### Context
`renderAntigravitySessions` (line 1520) creates a `sessionRow` element (line 1562-1575) displaying "Session: <codename>..." in a teal header. The plan removes this row and instead surfaces the session timestamp as a subtitle on each artifact card.

#### Logic

1. **Remove `sessionRow` creation** (delete lines 1562-1575):
   Remove the entire block that creates the `sessionRow` div, `sessionLabel` span, and `sessionTs` span, and appends them to the section. The date subheader (line 1556-1559) remains as the grouping header.

2. **Add session timestamp as subtitle on artifact cards** (modify the `renderDocCard` call at line 1579):
   Before the `renderDocCard` call, compute the formatted session time:
   ```js
   const sessionTime = new Date(session.timestamp).toLocaleTimeString(undefined, { hour: '2-digit', minute: '2-digit' });
   ```
   Then pass it as the `subtitle` parameter:
   ```js
   const artifactCard = renderDocCard({
       title: artifact.name,
       subtitle: 'Session at ' + sessionTime,
       sourceId: 'antigravity',
       nodeId: artifact.id,
       nodeMetadata: { absolutePath: artifact.id },
       actions: ['Set Context', 'Link Doc', 'Copy Path'],
       isSelected: state.activeSource === 'antigravity' && state.activeDocId === artifact.id,
       clickHandler: (cardWrapper, e) => {
           document.querySelectorAll('.tree-node.selected').forEach(n => n.classList.remove('selected'));
           cardWrapper.classList.add('selected');
           state.activeSource = 'antigravity';
           state.activeDocId = artifact.id;
           state.activeDocName = artifact.name;
           vscode.postMessage({
               type: 'fetchAntigravityArtifact',
               artifactPath: artifact.id,
               requestId: ++state.previewRequestId
           });
       },
       extraClass: 'antigravity-artifact-node'
   });
   ```
   This reuses the same `toLocaleTimeString` call that was previously on the session row timestamp, maintaining locale-appropriate formatting.

#### Edge Cases
- **Multiple artifacts in same session**: All artifacts under the same session will show the same "Session at <time>" subtitle. This is correct — the time identifies the session, not the individual artifact.
- **`renderDocCard` subtitle filtering**: `renderDocCard` (line 551) only renders subtitle if `subtitle && subtitle !== title`. Since the subtitle is "Session at 08:20" and the title is the artifact name (e.g., "task", "walkthrough"), these will never match — subtitle will always render.
