# Fix Custom Agent Integration Issues

The custom agent feature is currently failing in two ways:
1. Custom agents do not appear on the Kanban board or break the pipeline transition.
2. Custom agent terminals may not open as expected due to potential configuration mismatches.

## Proposed Changes

### [agentConfig.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/agentConfig.ts)
- Standardize the prefix for custom agent workflows.
- Ensure that `buildKanbanColumns` and related utilities are used consistently with the full set of custom agents.

### [TaskViewerProvider.ts](file:///c:/Users/patvu/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)
- Update the `saveStartupCommands` message handler to correctly persist `data.customAgents` to the extension state.
- Update `_targetColumnForRole` and `_roleForKanbanColumn` to use the standardized prefix if necessary, or ensure they match the sanitized roles.
- Fix the `workflowName` construction for custom agents to be consistent with the derivation logic.
