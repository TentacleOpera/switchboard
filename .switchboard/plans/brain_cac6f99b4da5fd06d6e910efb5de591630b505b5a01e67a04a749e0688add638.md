# Sync Prompt Builder Settings Across IDEs and Align Default Options

To resolve prompt configurations resetting when switching IDEs and prevent coder roles from losing critical default restrictions, this plan modifies Switchboard to store role configurations inside the workspace-level `.switchboard/state.json` file and aligns the prompt builder's fallback values with the correct defaults.

## User Review Required

> [!NOTE]
> Moving role configuration storage from the IDE's state database to the project's `.switchboard/state.json` means prompt settings will be persisted in the filesystem. This allows settings to transfer seamlessly across any IDE (e.g., Windsurf, Antigravity, VS Code) and be shared across multi-repo layouts that resolve to the same effective parent root.

> [!WARNING]
> Existing role configurations saved in your previous IDE state database will not be automatically imported. However, new settings and any adjustments you make will now write to `.switchboard/state.json` and persist permanently across all IDE instances opening this workspace.

## Proposed Changes

### Configuration Core

#### [MODIFY] [sharedDefaults.js](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/sharedDefaults.js)
- Add a prominent warning comment at the top of the file and specifically above the `DEFAULT_ROLE_CONFIG` structure stating: `// CRITICAL: DO NOT CHANGE DEFAULTS UNLESS SPECIFICALLY ASKED`.

#### [MODIFY] [KanbanProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/KanbanProvider.ts)
- Update default fallback values inside `_getPromptsConfig` to match the correct defaults from `sharedDefaults.js`:
  - `skipCompilationByRole` for coder roles (`lead`, `coder`, `intern`) should fall back to `true` (instead of `false`).
  - `skipTestsByRole` for coder roles should fall back to `true` (instead of `false`).
  - `cavemanOutputByRole` for coder roles should fall back to `true` (instead of `false`).
- Update `saveSetting` message handler:
  - When saving `roleConfig_`, write it to the workspace's `.switchboard/state.json` via `this._taskViewerProvider.updateState()` instead of using `_taskViewerProvider.saveRoleConfig()`.
- Update `_getSetting` wrapper or `_getPromptsConfig` load logic:
  - Read `roleConfig_[role]` from `.switchboard/state.json` first. Fall back to standard settings storage (to support migration) or `DEFAULT_ROLE_CONFIG` if not present.

#### [MODIFY] [TaskViewerProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/TaskViewerProvider.ts)
- Update `saveRoleConfig` to persist configuration to `.switchboard/state.json` using `this.updateState(...)`.
- Update `getRoleConfig` to check `.switchboard/state.json` first, keeping the VS Code setting storage as a fallback.
