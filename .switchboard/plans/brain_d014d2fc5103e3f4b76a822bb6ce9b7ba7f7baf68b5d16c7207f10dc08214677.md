# Deprecating Legacy File-Based IPC, IDE-Specific Scaffolding, and Housekeeping Configuration

This implementation plan details the deprecation and cleanup of legacy components in the Switchboard extension. With `AGENTS.md` established as the single, clean source of truth for agent system prompts, and with task-tracking fully migrated to `kanban.db` (SQLite), several older file-based directories and editor-specific templates are now redundant and can be removed.

## Goal

Remove legacy file-based IPC (inbox/handoff), IDE-specific template scaffolding, and housekeeping configuration from the Switchboard extension, consolidating all agent communication to direct terminal push and all agent configuration to `AGENTS.md`.

## Metadata

- **Tags:** [reliability, workflow]
- **Complexity:** 5

## User Review Required

> [!WARNING]
> This plan deprecates cross-editor file-based messaging (`.switchboard/inbox/`) and the legacy external terminal-bridge (`terminal-bridge.js`). All agent execution will run within VS Code terminals or via direct process injection. If external editors (e.g. Windsurf, Cursor) are used on the same codebase, they must rely on the root `AGENTS.md` instructions and will not receive file-based commands.
>
> Additionally, the pair-programming Coder prompt backup file (`.switchboard/handoff/coder_prompt_*.md`) is removed. If a user dismisses the "Copy Coder Prompt" notification, they must re-trigger pair-programming from the Kanban board to regenerate the prompt — there is no longer a file-based recovery path.

## Edge-Case & Dependency Audit

- **Race Conditions:** The current code moves the Kanban card to the target column *before* dispatching (TaskViewerProvider.ts line 14775). When `_attemptDirectTerminalPush` fails, the inbox fallback writes a file that nobody reads (no InboxWatcher exists) — the card is already silently stranded today. The fix is to add a rollback on dispatch failure so the card returns to its source column, plus a warning notification so the user knows what happened.
- **Security:** Removing `_executeRemote` eliminates the inbox-writing attack surface (F-04 agent name validation, F-08 session token injection). No new security concerns are introduced.
- **Side Effects:** Removing `housekeeping.policy.json` creation — no runtime housekeeping runner was found in the current codebase; the file is vestigial. Safe to remove.
- **Dependencies & Conflicts:** The `check_inbox` and `send_message` agent-facing tools reference inbox/outbox semantics that are being removed. These tools must be stripped from all documentation and configuration files (`AGENTS.md`, `.github/copilot-instructions.md`, `.github/agents/switchboard.agent.md`, `.codeium/windsurf-instructions.md`, `docs/TECHNICAL_DOC.md`, `docs/SECURITY-AUDIT.md`, `docs/DELEGATION_WORKFLOWS_README.md`). The `InboxWatcher` described in TECHNICAL_DOC.md section 11 was not found in current source code (likely already removed or documentation drift) — remove the section from the doc.

## Dependencies

## Adversarial Synthesis

Key risks: (1) Cards are already silently stranded today when direct push fails (inbox writes go unread) — this plan adds rollback + notification to fix the existing bug. (2) Lost Coder prompt recovery path when user dismisses pair-programming notification — acceptable trade-off since re-triggering from the board regenerates the prompt.
