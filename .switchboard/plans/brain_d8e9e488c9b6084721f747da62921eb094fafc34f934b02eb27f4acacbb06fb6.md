# Visual Synchronization of Artifacts Panel Theme

This plan outlines the visual enhancements, bug fixes, and the new optional **Cyber Panel Theme** to synchronize the **Artifacts Preview** and **Planning Panel** with the cyber-neon aesthetic of the **Kanban Board**.

## Goal

Synchronize the Artifacts/Planning panel's visual styling (typography, cards, badges, code highlights, blockquotes, and optional CRT scanlines) with the Kanban Board's cyber-neon theme, and fix the broken blockquote rendering bug.

## Problem & Root Cause Analysis

Based on the visual audit and code inspection, several areas of the Artifacts/Planning panel do not match the polished styling of the Kanban Board:
1. **Broken Blockquotes**: The markdown renderer escapes HTML characters first (converting `>` to `&gt;`). The blockquote regex `replace(/^> (.+)$/gm, ...)` runs after escaping, which fails to match `&gt;` and outputs blockquotes as plain text with raw `>` characters.
2. **Typography Disparity**: The tab bar (`.research-tab-bar` and `.research-tab-btn`) uses a basic sans-serif font instead of the geometric monospace font (`var(--font-mono)`) with rounded corners and glowing hover states found in the Kanban view.
3. **Card Design Inconsistency**: The list of plans (`.kanban-plan-item`) in the planning panel uses simple dividers and borders instead of matching the separate, glassmorphic card design (`.kanban-card`) of the Kanban board.
4. **Subdued Badges**: Column badges (`.kanban-column-badge`) have a generic grey background instead of vibrant colors matching their respective columns.
5. **Code Block/Inline Code Contrast**: Inline code elements use a bright orange color (`#d29922`) with yellow-tinted backgrounds, which clashes with the cyan/teal/dark-blue cyber-neon theme of the rest of the application.
6. **Optional Cyber Aesthetics**: The extension looks highly customized, and having a game-like CRT look (scanlines/glow) is desired, but it must be purely optional and toggleable from the Setup screen.

### Complex / Risky
- Fixing blockquote rendering in `renderMarkdown` (planning.js lines 384–452): requires matching `&gt;` after HTML escaping and grouping consecutive blockquote lines into a single `<blockquote>` element — this is stateful line-by-line processing that modifies the rendering pipeline's core logic
- Markdown alert detection (`[!NOTE]`, `[!TIP]`, etc.) inside grouped blockquotes: requires the blockquote grouping to work first, then parsing the first line of each group for alert tags and transforming them into styled alert blocks
- Cyber scanline overlay HTML wrapping: requires restructuring preview pane containers (`#preview-pane`, `#preview-pane-online`, `#kanban-preview-pane`) inside relative-positioned wrappers with overlay divs — must not break existing layout or block pointer events

## Edge-Case & Dependency Audit

- **Race Conditions**: The `cyberThemeSetting` message from `PlanningPanelProvider` could arrive before the webview DOM is ready. Mitigate by queuing the setting and applying it on DOM ready, or by checking `document.body` exists before toggling the class.
- **Security**: The `data-column` attribute value is produced by `escapeHtml(plan.column)` which already sanitizes the value. No XSS risk. The scanline overlay has no user-controlled content.
- **Side Effects**: Changing inline code color from `#d29922` to `var(--accent-teal)` will affect ALL markdown preview content across all three preview panes (`#markdown-preview`, `#markdown-preview-online`, `#kanban-preview-pane`). This is intentional but should be verified that teal code doesn't clash with teal links (links already use `var(--vscode-textLink-foreground, var(--accent-teal))` at line 911).
- **Dependencies & Conflicts**: The blockquote fix must be implemented before the markdown alert detection, since alerts are a special case of blockquotes. The `data-column` attribute must be added to the JS before the CSS attribute selectors will have any effect. The cyber theme toggle requires all 4 backend files to be updated before the frontend toggle will function.

## Dependencies

- None (this plan is self-contained)

## Adversarial Synthesis

Key risks: Blockquote fix is coupled to HTML-escape order (fragile if escape logic changes); custom kanban column IDs may not match the 5 hardcoded CSS color rules; scanline overlay could block mouse interaction if `pointer-events` is not set. Mitigations: Add code comment documenting escape-order coupling; include a CSS fallback rule for unmatched column badges; add `pointer-events: none` to `.cyber-scanlines` overlay.
