# Fix HTML Preview Blank Screen in Planning Panel

Fix the HTML preview panel showing a blank screen by correcting CSS layout properties and allowing VS Code webviews to load files from configured HTML/workspace directories.

## User Review Required

> [!NOTE]
> The fix consists of two main parts:
> 1. Adding CSS flexbox and height rules for `#html-preview-content` so that the split-pane content layout does not collapse to `0` height.
> 2. Expanding the webview's `localResourceRoots` to include all workspace folders and configured HTML/local folders, resolving VS Code security blocks when rendering files via `asWebviewUri`.

## Proposed Changes

---

### Planning Panel Webview

#### [MODIFY] [planning.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/planning.html)

- Add `#html-preview-content` to the styling blocks that set `display: flex`, `flex-direction: column`, `height: calc(100vh - 40px)`, and `background: var(--panel-bg)`.
- Add `#html-preview-content.active` to the style block that applies `display: flex`.

---

### Extension Services

#### [MODIFY] [PlanningPanelProvider.ts](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/services/PlanningPanelProvider.ts)

- Update the `localResourceRoots` configuration in `vscode.window.createWebviewPanel` options within `open()` to include all open workspace folder URIs and any configured HTML and local docs directories.
