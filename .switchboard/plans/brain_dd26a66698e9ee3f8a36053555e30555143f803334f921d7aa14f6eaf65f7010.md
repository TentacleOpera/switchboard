## Step 1: Understand the Goal
Install the Airbridge React Native SDK and configure it natively for iOS and Android. This allows for precise measurement of SKAdNetwork attribution, App Opens, and deep links. Crucially, the configuration must map the unified user identity (`_id`) matching RevenueCat.

## Step 2: Complexity, Edge-Case & Dependency Audit
*   **Complexity:** High. Requires touching native Android (`MainApplication.java`) and iOS (`AppDelegate.swift`) entry points, along with generating a strictly formatted `airbridge.json` configuration.
*   **Edge Cases:** App Tracking Transparency (ATT) on iOS 14.5+ forces us to delay the first event until consent is determined. We must set `autoStartTrackingEnabled: false` in the configuration so we can explicitly inject the User ID before firing the initial Install event.
*   **Dependencies & Conflicts:** Relies on `Auth.tsx` to dispatch the user ID concurrently with Amplitude.

## Step 3: Improve Plan (/improve-plan)
*   **Grumpy Persona:** Modifying native code often breaks the build if React Native headers aren't imported correctly. Also, if Airbridge fires its "Install" event before `Auth.tsx` sets the user ID, Airbridge will generate a random device ID, creating ghost users and destroying ROAS matching.
*   **Balanced Persona:** We will provide the exact native imports needed. By setting `autoStartTrackingEnabled: false` in `airbridge.json`, we prevent the SDK from flushing prematurely. We will manually call `Airbridge.startTracking()` in `TrackerService.ts` only *after* ATT permissions are settled, and update `Auth.tsx` to set the identity safely.

## 5. Exhaustive Implementation Spec

### Adversarial Synthesis
#### Grumpy Critique
If you use `airbridge.json` for configuration, it won't apply to different build environments correctly unless you set the `AIRBRIDGE_ENVIRONMENT` variables.
#### Balanced Response
For React Native SDK v4.5.1+, `airbridge.json` serves as the default root configuration. As long as it sits in the project root, it will successfully dictate default behavior (like the 30-second ATT delay).
