# Setup Webview Theme Tab and Docs Button Relocation

## Metadata
**Complexity:** 3
**Tags:** frontend, ui, ux, refactor

## Goal Description
The `setup.html` layout is being optimized to group user settings and visual preferences more logically. 
Additionally, the animation toggle UI is being changed from negative wording ("Disable Cyber Theme Animation") to positive wording ("Enable Artifacts panel animation"). Since the animation defaults to ON for new users, the checkbox will default to checked.

## Proposed Changes

### Switchboard Webview

#### [MODIFY] [setup.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/setup.html)
- Relocate `<button id="btn-open-docs">` from the Docs tab to the "Switchboard guide" section inside the Setup tab (`#startup-fields`).
- Wrap the "Copy Tutorial Prompt" and "Open Docs" buttons in a flex container so each button occupies 50% width.
- Rename the tab header `<button class="tab-btn" data-tab="docs">` to `<button class="tab-btn" data-tab="theme">` and change the display text from "Docs" to "Theme".
- Rename the tab content container `#docs-fields` (with `data-tab-content="docs"`) to `#theme-fields` (with `data-tab-content="theme"`).
- Remove the "Visual Theme" section from `#artifacts-panel-fields` and place it in the new `#theme-fields` tab.
- Add the requested explanatory text at the top of the `#theme-fields` tab:
  > 'Change the visual look of Switchboard. More themes coming soon for people who don't want to look like they're playing video games at work. More animations coming soon for people who do want to look like they're playing video games at work'
- Add a radio button selection for available themes, with a default checked option for "Afterburner".
- Change the animation toggle checkbox to positive wording:
  - Label: **Enable Artifacts panel animation**
  - Description: "Show the animated rolling CRT sweep beam in the planning panel preview."
- Invert the UI checkbox state logic in JavaScript to map to the underlying negative backend configuration (`theme.disableCyberAnimation`):
  - On hydration/load: `checkbox.checked = !disabledSetting`
  - On change/save: send `enabled = !checkbox.checked` as the `disableCyberAnimation` value.
- Implement JavaScript logic to conditionally show the animation settings based on whether the "Afterburner" theme is selected (defaulting to visible).
- Register the `'theme'` tab in `tabLoadCallbacks` to query the cyber animation state setting when selected.
