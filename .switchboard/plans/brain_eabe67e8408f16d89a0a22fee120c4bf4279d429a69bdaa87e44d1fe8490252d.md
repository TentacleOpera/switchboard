# Kanban Control Button Relocation and Visual Polish

This plan addresses a series of layout, icon, and styling improvements to the Kanban board webview in order to improve visual consistency, accessibility, and button placement.

## Goal

Relocate the "Copy Chat Prompt" button from the CREATED column header into the global kanban sub-bar (next to the Collapse Coders button), update icon assignments for the collapse coders and automation buttons, and apply visual polish to column header and icon buttons.

## Metadata

- **Tags:** UI, UX, frontend
- **Complexity:** 3

## User Review Required

- Confirm whether the copy chat prompt action in the sub-bar should act on **all** selected cards globally (current plan uses `selectedCards.keys()`) versus only cards in the CREATED column. The sub-bar is board-global so global selection is more appropriate, but this is a behavioral change from the old column-header button.
- Confirm the tooltip for `#btn-chat-copy-prompt` wording is acceptable.

## Complexity Audit

### Routine

- CSS property updates to existing, well-named selectors
- Adding a new entry to the icon map in `KanbanProvider.ts` (existing pattern, same block)
- Changing an icon filename for one existing token (`{{ICON_22}}`)
- Moving a button from one HTML location to another
- Adding a JS event listener using the existing `flashIconBtn` / `postKanbanMessage` / `selectedCards` pattern
- Updating one JS constant string (`ICON_COLLAPSE_CODERS`)

### Complex / Risky

- **Selection-scope change**: The old column-header button used `getSelectedInColumn(column)` (scoped to CREATED). The new sub-bar button uses `Array.from(selectedCards.keys())` (board-global). This is intentional and an improvement, but is a behavioral change that must be tested manually.

## Edge-Case & Dependency Audit

### Race Conditions

- None. All changes are DOM-event-driven in a single-threaded webview JS context.

### Security

- No new postMessage types introduced. Existing `chatCopyPrompt` message type is already handled by `KanbanProvider.ts`.

### Side Effects

- The `case 'chatCopyPrompt'` branch in the column-icon-btn event delegation switch (line 4465 in `kanban.html`) will remain orphaned after the button is removed from the CREATED column. It is safe to leave in place — it will simply never be triggered. A code comment should be added if desired.
- `ICON_JULES` (line 3458) and `ICON_SPLITTER` (line 3459) continue to reference `'{{ICON_28}}'` — this is intentional and unchanged.

### Dependencies & Conflicts

- Icon assets `25-1-100 Sci-Fi Flat icons-66 copy.png` and `25-1-100 Sci-Fi Flat icons-78.png` are confirmed present in `/icons/`.
- The new `{{ICON_COLLAPSE_CODERS}}` token must be added to `KanbanProvider.ts` before the webview HTML token-replacement loop runs; the two files must be deployed together.

## Dependencies

- None (no cross-plan session dependencies)

## Adversarial Synthesis

Key risks: (1) The selection scope changes from column-scoped to board-global when the button moves to the sub-bar — this is an intentional improvement but must be manually verified. (2) `updateReassignButtonVisibility()` must be in scope at the new listener's call site (it is, confirmed by existing usage patterns). (3) The `#btn-add-plan` id-targeted CSS correctly overrides `column-header-btn` base styles, but is semantically distinct from the `.btn-add-plan` class used on `#btn-add-project`. Mitigations: Test selection behavior with multi-column selections before and after; confirm the plus button renders with correct teal color and bold weight.

// Chat copy prompt button (sub-bar global action)
document.getElementById('btn-chat-copy-prompt')?.addEventListener('click', () => {
    const btn = document.getElementById('btn-chat-copy-prompt');
    if (btn) flashIconBtn(btn);
    const sessionIds = Array.from(selectedCards.keys());
    postKanbanMessage({ type: 'chatCopyPrompt', sessionIds, workspaceRoot: getActiveWorkspaceRoot() });
    selectedCards.clear();
    document.querySelectorAll('.kanban-card.selected').forEach(el => el.classList.remove('selected'));
