# Immediate Workspace Switching and Plan Selection Persistence in Kanban View

This plan details the changes required to allow the workspace to switch immediately upon selection in the `workspace-project-select` dropdown in the Kanban board, even when plan cards are currently selected. In addition, it ensures that plan selections persist across workspace transitions so that they can be successfully reassigned to the new workspace.

## User Review Required

> [!IMPORTANT]
> The plan selection state (selected cards) will be maintained in memory in the webview. If plans are selected in Workspace A, switching to Workspace B will render Workspace B's cards, and the visual selection states of Workspace A's cards will be "hidden" (since those cards are not on the board). However, if the user then clicks **ASSIGN**, any selected cards from Workspace A will be reassigned to Workspace B. If they switch back to Workspace A, the cards will visually show as selected again.

## Proposed Changes

### Webview Interface

#### [MODIFY] [kanban.html](file:///Users/patrickvuleta/Documents/GitHub/switchboard/src/webview/kanban.html)

1. **Remove Workspace Switch Block**:
   In the `workspace-project-select` dropdown `change` event listener, remove the check that prevents switching when `selectedCards.size > 0`. This will allow the workspace or project filter to change immediately when the user interacts with the dropdown.

2. **Refine Selected Card Pruning**:
   In `renderBoard()`, adjust the selected card pruning logic. Currently, any selected card that is not in the newly rendered list is deleted from the `selectedCards` map. Instead, only prune cards if they belong to the current workspace root (`currentWorkspaceRoot`) and are missing from the board. Selected cards belonging to other workspaces will be preserved.

3. **Improve Reassignment Handler**:
   Update the `btn-assign-workspace-project` click handler. Instead of assuming all selected cards are from a single source workspace and using `isSameWorkspace` as a binary flag, group the selected cards by their source workspace root.
   - For plans whose source workspace root differs from the target workspace root, send `reassignPlansWorkspace` messages.
   - For plans whose source workspace root matches the target workspace root, send `assignSelectedToProject` messages (if a project filter is selected).
   - Once all messages are sent, trigger `selectWorkspace` or `setProjectFilter` to update the board context accordingly and clear `selectedCards`.

### Manual Verification
1. Open the Switchboard Kanban view.
2. Select one or more plan cards in Workspace A.
3. Switch the workspace in the dropdown picker to Workspace B.
4. Verify that the view immediately switches to Workspace B.
5. Verify that the **ASSIGN (N)** button remains enabled in the controls strip (indicating the selected plans are persisted).
6. Click **ASSIGN**.
7. Switch the workspace back to Workspace B / verify that the plans are now moved to Workspace B.
8. Switch back to Workspace A and verify that the plans are no longer there.
