# Feature A · A2b — Per-Verb Handler Burn-Down (Remaining Kanban & Panel Verbs)

This implementation plan covers the continuation of migrating remaining message handler `case` arms out of `KanbanProvider` (and subsequently other providers: `PlanningPanelProvider`, `DesignPanelProvider`, `SetupPanelProvider`, `TaskViewerProvider`) into their corresponding host-agnostic service modules (`KanbanService`, `PlanningService`, `DesignService`, `SetupService`, `TaskViewerService`).

## Goal
Extract the remaining message handler `case` arms out of the panel providers into their corresponding host-agnostic service modules, and wire them to the generic HTTP/WS verb dispatchers so they can be driven remotely.

## Scope of This Session

We will proceed panel-by-panel in the requested order:
1. **Kanban Provider**: Migrate all remaining verbs (such as autoban config, MCP monitor config, remote control configuration, and other utilities).
2. **Planning Provider**: Migrate planning-related verbs.
3. **Design Provider**: Migrate design-related verbs.
4. **Setup Provider**: Migrate setup-related verbs.
5. **TaskViewer Provider**: Migrate task viewer-related verbs.

We will run the migrations in batches, verifying compilation (`npm run compile`) and parity check (`node scripts/check-protocol-parity.js`) along the way.
