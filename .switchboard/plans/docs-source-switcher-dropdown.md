# Docs Tab Source Switcher: Replace Toggle Buttons with Single-Select Dropdown

## Goal
Replace the four individually-toggleable source chip buttons in the DOCS tab with a single `<select>` dropdown so the user can pick one source — or "All Sources" — in a single click.

## Adversarial Synthesis
Key risks: (1) on migration, a persisted multi-source filter would leave the dropdown showing "All Sources" while the sidebar hides sources — a control that lies to the user; (2) stale `dist/` artifact if `src/` is edited without recompiling. Mitigations: normalize any non-single persisted filter to `allSources` + persist on init so the dropdown matches the sidebar; rebuild (`npm run compile`) before validating, with `src/` as source of truth. Rejected as over-engineering: defensive cloning of the shared `allSources` array (no mutation path exists) and adding an aria label (control matches the existing adjacent workspace-filter convention).

## Problem
In the DOCS tab of `planning.html`, the source filter is implemented as four individually toggleable chip buttons (`Local`, `ClickUp`, `Linear`, `Notion`). To view documents from only one source (e.g., Notion), the user must click three separate buttons to disable the other sources. This is cumbersome and unintuitive.

## Root Cause
The current UI uses independent toggle buttons (`filter-chip` elements) each with its own click handler that adds/removes the source from an array filter. There is no single-select affordance.

### `src/webview/planning.js`

**Context:** The per-chip click handler is at lines 459–480. State is initialized at line 36; `rerenderUnifiedDocs()` reads `state.docsSourceFilter` as a `Set` at line 1337.

**Logic:** Replace the four `click` listeners with one `change` listener on the `<select>`. Keep `state.docsSourceFilter` as an array so the downstream `Set` contract is unchanged. Set the dropdown's initial value from persisted state, normalizing any non-single filter to "All Sources" so the control matches the sidebar.

**Implementation — Step 3 (JS, lines 459–480):**
Replace the existing per-chip click handler:
```javascript
document.querySelectorAll('#docs-source-filter .filter-chip').forEach(chip => {
    chip.addEventListener('click', () => {
        const source = chip.dataset.source;
        let filter = state.docsSourceFilter || ['local', 'clickup', 'linear', 'notion'];
        if (chip.classList.contains('active')) {
            chip.classList.remove('active');
            filter = filter.filter(s => s !== source);
        } else {
            chip.classList.add('active');
            if (!filter.includes(source)) {
                filter.push(source);
            }
        }
        state.docsSourceFilter = filter;
        const currentPersisted = vscode.getState() || {};
        vscode.setState({
            ...currentPersisted,
            docsSourceFilter: state.docsSourceFilter
        });
        rerenderUnifiedDocs();
    });
});
```
With:
```javascript
const allSources = ['local', 'clickup', 'linear', 'notion'];
const sourceFilterSelect = document.getElementById('docs-source-filter');
if (sourceFilterSelect) {
    // Set initial dropdown value based on persisted state.
    // Clarification: if the persisted filter is not exactly one valid source
    // (e.g. a legacy multi-select like ['local','notion']), normalize it to
    // "All Sources" AND reset the underlying filter + persist, so the dropdown
    // label never contradicts what the sidebar actually shows.
    const currentFilter = state.docsSourceFilter || allSources;
    if (currentFilter.length === 1 && allSources.includes(currentFilter[0])) {
        sourceFilterSelect.value = currentFilter[0];
    } else {
        sourceFilterSelect.value = 'all';
        if (currentFilter.length !== allSources.length) {
            state.docsSourceFilter = allSources;
            const persisted = vscode.getState() || {};
            vscode.setState({ ...persisted, docsSourceFilter: state.docsSourceFilter });
        }
    }

    sourceFilterSelect.addEventListener('change', (e) => {
        const value = e.target.value;
        state.docsSourceFilter = value === 'all' ? allSources : [value];
        const currentPersisted = vscode.getState() || {};
        vscode.setState({
            ...currentPersisted,
            docsSourceFilter: state.docsSourceFilter
        });
        rerenderUnifiedDocs();
    });
}
```

This keeps `state.docsSourceFilter` as an array, so `rerenderUnifiedDocs()` (which builds a `Set` from it at line 1337) requires **no changes**.

**Edge Cases:** Do not mutate `state.docsSourceFilter` in place — the "all" branch points it at the shared `allSources` const; reassign only.

## Edge Cases & Risks
| Risk | Mitigation |
|------|------------|
| **Persisted state from old multi-select** — Users may have a persisted `docsSourceFilter` array with 2–3 sources active. On load, the dropdown wouldn't match, and showing "All Sources" while hiding sources would mislead the user. | On init, if the filter is not exactly one valid source, set the dropdown to `"all"` **and** reset `state.docsSourceFilter = allSources` (persisted) so the control matches the sidebar. A single-source persisted filter restores its matching option. |
| **Build artifact stale** — `dist/webview/planning.{html,js}` are webpack `CopyPlugin` outputs of `src/`. | `src/` is the source of truth; run `npm run compile` after editing so dist is regenerated. (Build run separately by the user this session.) |
| **Shared `allSources` reference** — assigning `state.docsSourceFilter = allSources` shares the const array. | Safe today: rerender only reads it. Never mutate `state.docsSourceFilter` in place going forward. |
| **No sources selected** — Old UI allowed zero active sources (empty sidebar). New dropdown always has a valid selection, so this impossible state is eliminated. | This is a UX improvement, not a risk. |
