# Enforce Implementation Patch Appendix in Feature Plans

## Review Status
- [x] Grumpy Critique completed
- [x] Balanced Synthesis completed
- [x] Plan Approved

## Goal
Update the Switchboard "How to Plan" template generated by the Airlock export feature. Adding a dedicated "Appendix: Implementation Patch" section forces AI agents to provide the exact code changes directly within the plan, eliminating the need to ask for it separately.

## User Review Required
> [!NOTE]
> This modifies the template generator in the backend extension code. You will need to rebuild the extension or update the compiled `.js` file to see the changes take effect.

## Complexity Audit

### Band A — Routine
- Add a few string lines to the markdown array inside the `_handleAirlockExport` method in `TaskViewerProvider.ts`.

### Band B — Complex / Risky
- None. This is a low-risk string template modification.

## Edge-Case Audit
- **Race Conditions:** None.
- **Security:** None.
- **Side Effects:** If an AI agent does not have access to the codebase (e.g., in a blind chat), it might generate a placeholder appendix. This is harmless.

## Adversarial Synthesis

### Grumpy Critique
Why are we hardcoding more bloat into the template? What if the plan is purely architectural and involves no code? Now we're forcing the AI to hallucinate a patch or leave an ugly blank section! Plus, if you don't update the compiled `TaskViewerProvider.js` alongside the TypeScript file, this won't even work until you run a full build!

### Balanced Response
Grumpy is correct that architectural plans might not need code, but the vast majority of Switchboard plans are actionable feature requests where a diff is highly valuable. If no code is needed, the AI can simply state "N/A". To address the deployment concern, the implementation patch will include instructions to update both the `.ts` and `.js` files if testing live.

## Proposed Changes

### `src/services/TaskViewerProvider.ts`
- Locate the `_handleAirlockExport()` method.
- Find the `exportContent` array where the plan template is constructed.
- Append the `## Appendix: Implementation Patch` headers immediately after the `Manual Testing` block.
