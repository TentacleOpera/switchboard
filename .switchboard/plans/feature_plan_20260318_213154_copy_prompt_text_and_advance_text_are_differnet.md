# copy prompt text and advance text are differnet

## Goal
When using the kanban buttons, the prompts generated by the "Copy Prompt" and "Advance" buttons are different, even for same types of plans. They should be identical to ensure consistency whether a user copies the prompt manually or lets the system auto-dispatch it.

## Proposed Changes
1. **Centralize Prompt Generation**: Extract `_buildKanbanBatchPrompt` from `src/services/TaskViewerProvider.ts` (around line 2310) into its own shared utility file (e.g., `src/services/agentPromptBuilder.ts`) OR expose it as a public static/instance method.
2. **Adopt Unified Prompt in KanbanProvider**: Modify `src/services/KanbanProvider.ts` to use this centralized `_buildKanbanBatchPrompt` instead of its own `_generateBatchPlannerPrompt` and `_generateBatchLowComplexityPrompt` methods.
3. **Handle Kanban Columns -> Roles**: Since `_buildKanbanBatchPrompt` requires a `role` (planner, coder, lead, reviewer), `KanbanProvider` should map the target column to the corresponding role using its existing `_columnToRole` or similar logic before generating the prompt.
4. **Preserve Dynamic Complexity (Plan 1 Integration)**: Ensure the unified prompt builder incorporates the dynamic wording for high vs. low complexity plans correctly.
