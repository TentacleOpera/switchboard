# Unbold "create a separate plan file for each issue" in Memo Intro

## Goal (Problem analysis + Root Cause with cited file:line)

The MEMO modal's intro paragraph renders the phrase "create a separate plan file for each issue" in bold. It should render as normal (unbolded) body text.

Root cause: the phrase is wrapped in literal HTML `<strong>` tags inside the memo modal's intro `<p>` block.

- `src/webview/kanban.html:3032` — the exact markup:
  ```html
  <strong>create a separate plan file for each issue</strong> in <code>.switchboard/plans/</code>.
  ```

The surrounding text is **HTML** (a static `<p>` inside the `#memo-modal` markup at `src/webview/kanban.html:3026-3036`), not markdown and not JS-built. So the correct fix is to remove the `<strong>` / `</strong>` HTML tags around this specific phrase. The `<code>.switchboard/plans/</code>` immediately after is unrelated styling and stays.

Note: the same `<p>` intentionally bolds several other phrases (`<strong>separate issue</strong>`, `<strong>Copy Prompt</strong>`, `<strong>Send to Planner</strong>`, `<strong>saved automatically</strong>`, `<strong>cleared</strong>`, `<strong>Clear</strong>`). Those are out of scope and MUST be left untouched — only the plan-file phrase changes.
