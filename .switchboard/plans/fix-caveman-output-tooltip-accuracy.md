# Fix Caveman Output Tooltip Inaccuracy in Runtime Source

## Goal
Correct the misleading "Caveman Output" addon tooltip across all role configurations. The current text claims a specific, unverified 65-75% token reduction. The fix is to replace it with the honest, user-approved description: "Compress responses to reduce output tokens".

## Root Cause
The previous fix was applied only to `src/webview/kanban.html`, which contains hardcoded fallback HTML. However, the actual runtime source of truth is `src/webview/sharedDefaults.js` — a JavaScript file injected into the webview at runtime that defines addon metadata (including tooltips) for every role. Because the webview loads and renders from `sharedDefaults.js` dynamically, any change to the static HTML is overwritten on reload/reinstall.
