# Fix Import Plans Button Style in Kanban View

## Metadata
- **Complexity:** 1
- **Tags:** ui, bugfix, frontend

## Goal
Replace the text-based `IMPORT PLANS` button in the Kanban sub-bar with an icon button that matches the surrounding `strip-icon-btn` controls, using the existing `25-1-100 Sci-Fi Flat icons-67.png` asset.

## Background & Root Cause
In `src/webview/kanban.html` at line ~2329, the `#btn-import-plans` control is rendered as a text button (`<button class="strip-btn">IMPORT PLANS</button>`). All adjacent controls in the same toolbar are icon buttons (`<button class="strip-icon-btn"><img src="..." /></button>`). This creates a visual inconsistency in the Kanban sub-bar.

Key risks: Visual regression if `.strip-icon-btn` styling differs from `.strip-btn` (e.g., width, height, padding, hover states); the sci-fi icon may not semantically communicate "import plans" to users. Mitigations: Verify visual alignment with adjacent buttons during manual testing; the tooltip text is preserved to aid discoverability.

## Proposed Changes

### `src/webview/kanban.html`
- **Context:** Kanban controls strip, line ~2329.
- **Logic:** Replace the text button markup with an icon button markup, using a new template variable `{{ICON_IMPORT_PLANS}}`.
- **Implementation:**

```html
<!-- Before -->
<button class="strip-btn" id="btn-import-plans" data-tooltip="...">IMPORT PLANS</button>

<!-- After -->
<button class="strip-icon-btn" id="btn-import-plans" data-tooltip="...">
    <img src="{{ICON_IMPORT_PLANS}}" alt="Import Plans">
</button>
```

- **Edge Cases:** The `id`, `data-tooltip`, and event listener wiring remain unchanged.

### `src/services/KanbanProvider.ts`
- **Context:** `iconMap` record inside the HTML content injection method, around line 6719.
- **Logic:** Add the new placeholder-to-URI mapping.
- **Implementation:**

```typescript
'{{ICON_IMPORT_PLANS}}': webview.asWebviewUri(vscode.Uri.joinPath(iconDir, '25-1-100 Sci-Fi Flat icons-67.png')).toString(),
```

- **Edge Cases:**
  - **Tooltip retention:** The existing tooltip text will be preserved on the new `<button>` element.
  - **Accessibility:** The `<img>` will include `alt="Import Plans"` for screen readers.
  - **No JS changes required:** The existing `document.getElementById('btn-import-plans')` listener at line ~3608 continues to work because the `id` is unchanged.
  - **Dist build:** The `dist/webview/kanban.html` will be regenerated by the build pipeline; only the source file in `src/` needs editing.
