# ClickUp/Linear Ticket Caching and Lazy Loading

Implement TTL-based caching for ClickUp tasks and Linear issues in the sidebar with background prefetch on extension startup, cache invalidation on mutations, and manual refresh capability.

## Problem Statement

The ClickUp/Linear sidebar currently fetches tasks/issues from the API on every load, causing unresponsiveness. There is no caching mechanism for tickets (only documents have caching via `PlanningPanelCacheService`). Users experience delays when opening the sidebar or searching for tickets.

## Solution Overview

Add in-memory caching with configurable TTL for ClickUp tasks and Linear issues. Cache is rebuilt on extension startup (not persisted across restarts). Implement background prefetch of last-accessed lists/projects, immediate cache invalidation on mutations (create/update/delete), and a manual refresh command.

## Design Decisions

Based on user requirements:
- **Cache persistence**: Rebuilt on each extension activation (not persisted to disk)
- **TTL duration**: 5 minutes (tickets don't change often, but mutations refresh immediately)
- **Prefetch scope**: Only last-accessed ClickUp lists and Linear projects (tracked in state.json)
- **UI indicators**: None (transparent to user)
- **Pagination**: Load all at once (can refine later if needed)
- **Manual refresh**: Command to force cache refresh

## Implementation Plan

### Phase 1: Extend PlanningPanelCacheService for Task/Issue Caching

**File**: `src/services/PlanningPanelCacheService.ts`

Add in-memory caching methods for tasks and issues:

```typescript
interface TaskCacheEntry {
  tasks: any[]; // ClickUpTask[] or LinearIssue[]
  cachedAt: number; // timestamp
  ttl: number; // milliseconds
}

private _taskCache = new Map<string, TaskCacheEntry>();

// Cache methods
public async cacheTasks(sourceId: string, key: string, tasks: any[], ttlMs: number = 300000): Promise<void>
public async getCachedTasks(sourceId: string, key: string): Promise<{ tasks: any[]; isStale: boolean } | null>
public async invalidateTaskCache(sourceId: string, key?: string): Promise<void>
public async clearAllTaskCache(): Promise<void>
```

**Rationale**: Extend existing cache service rather than create new one. In-memory only (no disk persistence per requirement). TTL-based with stale-while-revalidate pattern.

### Phase 3: Add Background Prefetch on Extension Startup

**File**: `src/services/TaskViewerProvider.ts`

Add prefetch methods:

```typescript
private async _prefetchIntegrationData(): Promise<void> {
  // Fire off all prefetches in parallel, don't await
  void Promise.allSettled([
    this._prefetchClickUpTasks(),
    this._prefetchLinearIssues()
  ]);
}

private async _prefetchClickUpTasks(): Promise<void> {
  const workspaceRoot = this._resolveWorkspaceRoot();
  if (!workspaceRoot) return;

  const clickUp = this._getClickUpService(workspaceRoot);
  const config = await clickUp.loadConfig();
  if (!config?.setupComplete) return;

  const lastAccessedLists = await this._getLastAccessedClickUpLists();
  const listsToPrefetch = lastAccessedLists.length > 0 
    ? lastAccessedLists 
    : Object.values(config.columnMappings).filter(Boolean);

  for (const listId of listsToPrefetch) {
    try {
      const tasks = await clickUp.getListTasks(listId, { includeClosed: false });
      await this._getCacheService(workspaceRoot).cacheTasks('clickup', listId, tasks, 300000);
    } catch (err) {
      console.warn(`[TaskViewer] Failed to prefetch ClickUp tasks for list ${listId}:`, err);
    }
  }
}

// Similar for Linear
```

**File**: `src/extension.ts`

Trigger prefetch after extension activation:

```typescript
// In activate() function, after TaskViewerProvider creation
setTimeout(() => {
  void taskViewerProvider._prefetchIntegrationData();
}, 500); // 500ms delay to let activation complete
```

**Rationale**: Non-blocking prefetch runs in background after activation. Uses last-accessed lists or falls back to all configured lists.

### Phase 4: Update ClickUpSyncService to Use Cache

**File**: `src/services/ClickUpSyncService.ts`

Add cache service reference and update task-fetching methods:

```typescript
private _cacheService?: PlanningPanelCacheService;

public setCacheService(service: PlanningPanelCacheService): void {
  this._cacheService = service;
}

public async getListTasks(listId: string, options: {...}): Promise<ClickUpTask[]> {
  const cacheKey = `list:${listId}`;
  
  // Try cache first
  const cached = await this._cacheService?.getCachedTasks('clickup', cacheKey);
  if (cached && !cached.isStale) {
    return cached.tasks;
  }
  
  // Fetch from API
  const tasks = await this._listTasksFromAPI(listId, options);
  
  // Cache with 5min TTL
  await this._cacheService?.cacheTasks('clickup', cacheKey, tasks, 300000);
  
  return tasks;
}

// Similar updates for searchTasks, findTask
```

Add cache invalidation on mutations:

```typescript
public async createTask(...): Promise<ClickUpTask> {
  const task = await this._createTaskAPI(...);
  await this._cacheService?.invalidateTaskCache('clickup');
  return task;
}

public async updateTask(...): Promise<void> {
  await this._updateTaskAPI(...);
  await this._cacheService?.invalidateTaskCache('clickup');
}
```

**Rationale**: Cache-first pattern with TTL fallback. Invalidate entire cache on any mutation (conservative but safe).

### Phase 5: Update LinearSyncService Similarly

**File**: `src/services/LinearSyncService.ts`

Same pattern as ClickUpSyncService:

```typescript
private _cacheService?: PlanningPanelCacheService;

public setCacheService(service: PlanningPanelCacheService): void {
  this._cacheService = service;
}

public async listIssues(projectId?: string): Promise<LinearIssue[]> {
  const cacheKey = projectId ? `project:${projectId}` : 'all';
  
  const cached = await this._cacheService?.getCachedTasks('linear', cacheKey);
  if (cached && !cached.isStale) {
    return cached.tasks;
  }
  
  const issues = await this._listIssuesFromAPI(projectId);
  await this._cacheService?.cacheTasks('linear', cacheKey, issues, 300000);
  
  return issues;
}

// Invalidate on mutations
public async updateIssueState(...): Promise<void> {
  await this._updateIssueStateAPI(...);
  await this._cacheService?.invalidateTaskCache('linear');
}
```

**Rationale**: Consistent pattern across both integrations.

### Phase 7: Add Manual Refresh Command

**File**: `src/services/TaskViewerProvider.ts`

```typescript
public async forceRefreshIntegrationCache(): Promise<void> {
  const workspaceRoot = this._resolveWorkspaceRoot();
  if (!workspaceRoot) return;

  // Clear cache
  await this._getCacheService(workspaceRoot).clearAllTaskCache();

  // Prefetch fresh data
  await this._prefetchIntegrationData();
  
  vscode.window.showInformationMessage('Integration cache refreshed');
}
```

**File**: `src/extension.ts`

```typescript

);

```

**Rationale**: User control to force refresh when needed.

## Edge Cases

- **Cache miss on startup**: Fallback to API fetch
- **Prefetch failure**: Log warning, don't block activation
- **No last-accessed lists**: Fallback to all configured lists
- **Mutation during prefetch**: Invalidate cache, re-fetch on next access

## Performance Considerations

- **Memory usage**: In-memory cache only, bounded by number of lists/projects
- **API calls**: Reduced significantly (cache hit = 0 API calls)
- **Startup time**: Unaffected (prefetch is non-blocking, 500ms delayed)
- **Sidebar responsiveness**: Instant load from cache
