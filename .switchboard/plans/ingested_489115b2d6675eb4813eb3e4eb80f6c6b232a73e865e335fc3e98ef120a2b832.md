# Relax Clipboard Import Title Extraction

Modify clipboard import to fall back to H2 or H3 headers when no H1 is found, before using the default title.

## Problem

Current clipboard import only looks for H1 headers (`# Title`). If none exists, it immediately falls back to "Imported Plan" as the default title. This is too strict for plans that may use H2 or H3 as their primary heading.

## Solution

Update the single plan import logic to try H1 → H2 → H3 before falling back to the default title. This should NOT affect multi-plan import logic.

## Changes Required

### 1. TaskViewerProvider.ts - Single Plan Import (around line 13141)

**Current code:**
```typescript
const h1Match = text.match(/^#\s+(.+)$/m);
const title = h1Match ? h1Match[1].trim() : 'Imported Plan';

if (!h1Match) {
    vscode.window.showWarningMessage('No "# Title" found in clipboard. Importing with default title.');
}
```

**New code:**
```typescript
// Try H1, then H2, then H3, then fall back to default
const h1Match = text.match(/^#\s+(.+)$/m);
const h2Match = !h1Match ? text.match(/^##\s+(.+)$/m) : null;
const h3Match = !h1Match && !h2Match ? text.match(/^###\s+(.+)$/m) : null;

let title: string;
let warningMessage: string | null = null;

if (h1Match) {
    title = h1Match[1].trim();
} else if (h2Match) {
    title = h2Match[1].trim();
    warningMessage = 'No "# Title" found. Using H2 header as title.';
} else if (h3Match) {
    title = h3Match[1].trim();
    warningMessage = 'No "# Title" or "## Title" found. Using H3 header as title.';
} else {
    title = 'Imported Plan';
    warningMessage = 'No header found in clipboard. Importing with default title.';
}

if (warningMessage) {
    vscode.window.showWarningMessage(warningMessage);
}
```
