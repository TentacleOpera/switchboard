# Add Delete Button to Local Docs in Planning Panel

Add a delete button to each document in the Local Docs tab of the planning panel that moves files to trash.

## Implementation Steps

### 1. Update `LocalFolderService.ts`
Add a `deleteFile` method that moves files to trash using VS Code's API:
- Resolve the full file path from relative path
- Validate path is within the configured folder (security)
- Use `vscode.workspace.applyEdit` with `deleteFile` and `useTrash: true`
- Return success/error result

### 2. Update `PlanningPanelProvider.ts`
Add a message handler for `deleteLocalDoc`:
- Extract `docId` (relativePath) from the message
- Call `LocalFolderService.deleteFile()`
- On success: refresh the local docs list via `_sendLocalDocsReady()`
- On error: send error message to webview for display

### 3. Update `planning.js` (`renderNode` function)
Add a delete button to document nodes (only for `local-folder` source):
- Create a small "🗑️" or "×" button element positioned to the right of the label
- Add click handler that stops propagation (to avoid triggering doc selection)
- Post `deleteLocalDoc` message with the docId
- Show visual feedback while delete is in progress

### 4. Update `planning.html` (if needed)
Add CSS for the delete button:
- Small, subtle styling (opacity until hover)
- Positioned at the right edge of the tree node
- Ensure it doesn't interfere with the existing click-to-select behavior
