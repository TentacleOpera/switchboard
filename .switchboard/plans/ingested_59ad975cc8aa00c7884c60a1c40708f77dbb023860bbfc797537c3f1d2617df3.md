# Fix Imported Docs Display Hierarchy

Restructure the local docs tab sidebar to show source in the teal header and document titles as subheaders.

## Current Behavior
- Static header "IMPORTED DOCS" at the top of the imported section
- Source name (e.g., "CLICKUP") shown as subheader below
- Individual docs listed under each source subheader

## Desired Behavior
- Teal header shows "IMPORTED FROM {source}" for each source
- Document title (e.g., "Product - User Personas") shown as subheader
- Individual docs/pages grouped under their respective document title

## Changes Required

### 1. Modify `renderLocalDocs` in `/src/webview/planning.js` (lines 479-488)
- Remove the static "IMPORTED DOCS" section HTML
- Replace with a dynamic container that `handleImportedDocsReady` will populate

### 2. Modify `handleImportedDocsReady` in `/src/webview/planning.js` (lines 984-1087)
- Change grouping logic from `docsBySource` to a two-level hierarchy
- First level: Group by `sourceId`, create section header "IMPORTED FROM {source}"
- Second level: Within each source, group by `docName`, create subheader for each unique docName
- Third level: List individual docs under their docName subheader

### 3. Update CSS if needed
- Ensure proper styling for nested hierarchy (source header → doc subheader → items)
