# Add /clear-before-prompt option to kanban setup tab

Add a toggle in the kanban's **Setup** tab that, when enabled, sends `/clear` to a terminal 2 seconds before dispatching any prompt message. This applies to all terminal dispatches (batch, autoban, single-plan, and pair programming). Off by default, persisted globally.

---

## Files changed

- `src/webview/kanban.html` — UI: new "Terminal Context" section in Setup tab with checkbox + state handlers
- `src/services/KanbanProvider.ts` — state: load/persist `terminal.clearBeforePrompt` global config, wire webview messages
- `src/services/TaskViewerProvider.ts` — dispatch: read config and send `/clear` + 2s delay before `sendRobustText` in `_attemptDirectTerminalPush`

## Detailed steps

### 1. UI (`kanban.html` Setup tab)
- Add a new `<div class="setup-section">` titled **Terminal Context** after the existing "Batch Operations" section.
- Inside it, add a toggle identical in style to the existing `unknown-complexity-toggle`:
  - `<input type="checkbox" id="clear-terminal-before-prompt-toggle">`
  - Label: **Clear terminal before prompt**
  - Tooltip: "Send /clear to the terminal before dispatching prompts"
- Add a JS variable `let clearTerminalBeforePrompt = false;` near the other toggle state vars.
- In the `postMessage` handler, add a case for `clearTerminalBeforePromptState` to set the variable and update the checkbox.
- Add a `change` event listener on the toggle that:
  1. Updates the local variable
  2. Calls `updateClearTerminalBeforePromptUi()`
  3. Posts `postKanbanMessage({ type: 'toggleClearTerminalBeforePrompt', enabled: checked })`
- Add `updateClearTerminalBeforePromptUi()` to sync checkbox state and label styling.

### 2. State (`KanbanProvider.ts`)
- Add private field `_clearTerminalBeforePrompt: boolean`.
- In the constructor, initialize it from global VS Code config:
  ```ts
  this._clearTerminalBeforePrompt = vscode.workspace.getConfiguration('switchboard').get<boolean>('terminal.clearBeforePrompt', false);
  ```
- In `_refreshBoard` and the initial state push, post:
  ```ts
  this._panel.webview.postMessage({ type: 'clearTerminalBeforePromptState', enabled: this._clearTerminalBeforePrompt });
  ```
- In `_handleWebviewMessage`, add case `toggleClearTerminalBeforePrompt`:
  - Set `_clearTerminalBeforePrompt = !!msg.enabled`
  - Persist: `await vscode.workspace.getConfiguration('switchboard').update('terminal.clearBeforePrompt', this._clearTerminalBeforePrompt, true)` (global)
  - Re-post state to webview for sync

### 3. Dispatch (`TaskViewerProvider.ts`)
- In `_attemptDirectTerminalPush`, after the terminal is resolved and before `sendRobustText` is called:
  ```ts
  const clearBeforePrompt = vscode.workspace.getConfiguration('switchboard').get<boolean>('terminal.clearBeforePrompt', false);
  if (clearBeforePrompt) {
      terminal.sendText('/clear', true);
      await new Promise(r => setTimeout(r, 2000));
  }
  ```
- Keep the existing `terminal.show()` and logging behavior untouched.

## Notes
- The setting key is `switchboard.terminal.clearBeforePrompt`, stored globally (`true` target in `config.update`).
- The `/clear` command is sent via `terminal.sendText('/clear', true)` so the terminal itself executes it; this works for CLI agents that accept slash commands.
- No changes needed to `sendRobustText`; the delay is injected upstream in `_attemptDirectTerminalPush`.
- The toggle follows the exact same pattern as `allowUnknownComplexityAutoMove` for consistency.
