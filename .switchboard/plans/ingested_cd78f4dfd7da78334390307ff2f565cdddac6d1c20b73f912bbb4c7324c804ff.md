# Fix Planning Panel Multi-Root Workspace Consistency

Unify workspace root resolution across PlanningPanelProvider to fix empty docs view in multi-root workspaces by ensuring all methods search across all roots with consistent fallback priority.

## Problem

The planning panel shows no documents in multi-root workspaces because different methods use inconsistent workspace root resolution:

- `_handleFetchImportedDocs` (lines 966-1011): Searches all roots with fallback chain
- `_sendLocalDocsReady` (lines 725-747): Uses only the passed `workspaceRoot` param
- `_handleFetchDocsFile` (lines 1097-1130): Searches all roots with fallback chain
- `_sendOnlineDocsReady` (lines 749-779): Uses `_getWorkspaceRoot()` which may return wrong root

## Root Cause Analysis

1. **Inconsistent resolution**: `_sendLocalDocsReady` doesn't search across roots like imported docs does
2. **Stale root detection**: `_getWorkspaceRoot()` captures root at panel open time; doesn't refresh
3. **Silent failures**: Adapters register once per root; if active root has no ClickUp/Linear/Notion config, those adapters never register even if other roots have valid configs

## Implementation

### Step 1: Register online adapters from ALL workspaces

Replace `_ensureAdaptersRegistered` to iterate through all roots and collect adapters from each:

```typescript
private _ensureAdaptersRegistered(): void {
    const allRoots = this._getWorkspaceRoots();
    if (allRoots.length === 0) return;
    
    // Track if any roots changed to determine if we need to re-register
    const rootsKey = allRoots.join(',');
    if (this._registeredRootsKey === rootsKey) return;

    // Clear existing adapters to avoid duplicates from previous registrations
    this._researchImportService.clearAdapters?.();

    for (const workspaceRoot of allRoots) {
        // Notion
        try {
            const notionService = this._adapterFactories.getNotionService?.(workspaceRoot);
            const notionBrowseService = this._adapterFactories.getNotionBrowseService?.(workspaceRoot);
            if (notionService && notionBrowseService) {
                this._researchImportService.registerAdapter(
                    new NotionResearchAdapter(notionService, notionBrowseService)
                );
            }
        } catch { /* ignore if config not found at this root */ }

        // Linear
        try {
            const linearAdapter = this._adapterFactories.getLinearDocsAdapter?.(workspaceRoot);
            if (linearAdapter) {
                this._researchImportService.registerAdapter(linearAdapter);
            }
        } catch { /* ignore if config not found at this root */ }

        // ClickUp
        try {
            const clickUpAdapter = this._adapterFactories.getClickUpDocsAdapter?.(workspaceRoot);
            if (clickUpAdapter) {
                this._researchImportService.registerAdapter(clickUpAdapter);
            }
        } catch { /* ignore if config not found at this root */ }
    }

    this._registeredRootsKey = rootsKey;
}
```

### Step 2: Create unified path resolution helper

```typescript
private async _resolveWorkspacePath(
    relativePath: string,
    options?: { preferActive?: boolean }
): Promise<{ path: string | null; source: string }> {
    const allRoots = this._getWorkspaceRoots();
    const activeRoot = this._getWorkspaceRoot();
    const resolvedPath = path.join(activeRoot || '', relativePath);
    
    // Try active root first
    if (activeRoot && fs.existsSync(resolvedPath)) {
        return { path: resolvedPath, source: 'active workspace' };
    }
    
    // Try first root as fallback
    if (allRoots.length > 0 && fs.existsSync(path.join(allRoots[0], relativePath))) {
        return { path: path.join(allRoots[0], relativePath), source: 'first workspace' };
    }
    
    // Search all roots
    for (const root of allRoots) {
        if (root === activeRoot || root === allRoots[0]) continue;
        const candidate = path.join(root, relativePath);
        if (fs.existsSync(candidate)) {
            return { path: candidate, source: `workspace ${path.basename(root)}` };
        }
    }
    
    return { path: null, source: 'not found' };
}
```

### Step 3: Update `_sendLocalDocsReady`

Replace single-root listing with multi-root search:

```typescript
private async _sendLocalDocsReady(): Promise<void> {
    const allFiles = [];
    const allRoots = this._getWorkspaceRoots();
    
    for (const root of allRoots) {
        const service = this._getLocalFolderService(root);
        const files = await service.listFiles();
        allFiles.push(...files.map(f => ({ ...f, _root: root })));
    }
    
    this._panel?.webview.postMessage({
        type: 'localDocsReady',
        sourceId: 'local-folder',
        nodes: this._mapLocalFilesToTreeNodes(allFiles)
    });
}
```

## Risks

| Risk | Mitigation |
|------|------------|
| Performance: scanning all roots | Cache results; only scan on panel open/refresh |
| Duplicate file names across roots | Include workspace indicator in UI |
| Breaking single-root behavior | Maintain backward-compatible fallback chain |
| Same source configured in multiple workspaces | Last adapter wins (registered in iteration order) - this is acceptable for now as configs should be workspace-specific |
| Duplicate adapter instances | Clear adapters before re-registering; use rootsKey to avoid unnecessary re-registration |
