# Fix Stuck Active Design Doc

When the user clicks "Turn off" to disable the active design doc in the planning panel, it currently only clears the `designDocEnabled` flag but leaves the `designDocLink` setting intact. This causes the UI to still display the old design doc name even though it's supposed to be disabled. The fix will clear both settings when disabling.

## Changes

**File: `src/services/PlanningPanelProvider.ts`**

Modify the `_handleDisableDesignDoc()` method (lines 535-553) to also clear the `designDocLink` setting:

```typescript
private async _handleDisableDesignDoc(): Promise<void> {
    try {
        await vscode.workspace.getConfiguration('switchboard').update(
            'planner.designDocEnabled',
            false,
            vscode.ConfigurationTarget.Workspace
        );
        // Clear the designDocLink to remove stale references
        await vscode.workspace.getConfiguration('switchboard').update(
            'planner.designDocLink',
            undefined,
            vscode.ConfigurationTarget.Workspace
        );
        // Send updated state back to panel
        await this._sendActiveDesignDocState();
    } catch (err) {
        console.error('[PlanningPanelProvider] Failed to disable design doc:', err);
        this._panel?.webview.postMessage({
            type: 'activeDesignDocUpdated',
            enabled: true,
            docName: this._getDesignDocName(),
            error: String(err)
        });
    }
}
```

The existing UI logic in `planning.js` already handles displaying "None" when the design doc is disabled, so no frontend changes are needed.
