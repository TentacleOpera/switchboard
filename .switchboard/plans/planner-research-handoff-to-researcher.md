# Hand Off Planner Research Prompts to an Active Researcher Agent

**Complexity:** 5

## Goal

When the planner's **"advise research if unsure"** add-on identifies uncertain assumptions, it
should — *if a Researcher agent is currently active* — send the ready-to-run research prompt
**directly to that agent over the LocalApiServer HTTP surface**, and instruct it to save the
findings to the **configured (or default) research-docs folder**. When no researcher is live it
must fall back to today's behavior: paste the prompt into the chat summary for the user to run
manually.

### Problem & root-cause analysis

Today `ADVISE_RESEARCH_DIRECTIVE` (`src/services/agentPromptBuilder.ts`) makes the planner end its
turn by pasting a research prompt into chat and asking the *user* to run it and feed findings back.
That stalls the pipeline on a human round-trip even when the workspace already has an idle
Researcher agent that could do the work immediately.

- **The planner is a CLI agent with a shell**, so it can reach the localhost API — but it **cannot
  read VS Code settings.** The "configured research-docs folder" is the extension setting
  `switchboard.research.localFolderPaths[0]` (fallback `.switchboard/docs/`). Any design that asks
  the planner to name the save folder itself is wrong by construction — the folder must be resolved
  **server-side** in the extension.
- **"Active" has a precise meaning.** A researcher counts as active only when a `researcher`-role
  terminal is both *registered* and *live* (`vscode.Terminal.exitStatus === undefined`).
  Registration is in-memory extension state surfaced on `GET /health.terminals`; **no file on disk
  reflects it.** Deciding "active or not" therefore also belongs server-side.
- **Existing primitives are too low-level for the planner to orchestrate reliably.** A raw hand-off
  would force the planner to read the port file → `GET /health` → parse the terminal list → guess
  which name is the researcher → `POST /taskViewer/verb/sendToTerminal` with exact webview field
  names → and separately resolve the docs folder (which it can't). That is fragile shell
  choreography with a silent-no-op failure mode (the verb rail answers `{success:true}` even when
  the arm no-ops on wrong field names).
- **Root cause:** there is no single, intent-level operation for "delegate this research to the
  researcher if one is available." The fix is to add one — a first-class endpoint that encapsulates
  the liveness check + folder resolution + dispatch — and make the directive call it and branch on
  the result.

## Grounded facts (verified in code, 2026-07-13)

- **Research add-on wiring.** `ADVISE_RESEARCH_DIRECTIVE` is defined in
  `src/services/agentPromptBuilder.ts` and appended to the planner base when
  `options.adviseResearchIfUnsure` is true (default true). The flag is populated in
  `KanbanProvider.ts` from `plannerConfig?.addons?.adviseResearch ?? true`.
- **Researcher role already exists.** `agentConfig.ts` `BuiltInAgentRole` includes `researcher`;
  the `RESEARCHER` kanban column maps to role `researcher`. The researcher prompt branch in
  `agentPromptBuilder.ts` already supports a save-to-docs instruction gated by
  `options.saveToLocalDocs` / `options.localDocsPath`.
- **Research-docs folder source.** `KanbanProvider.ts` resolves `localDocsPath` as
  `config.get<string[]>('research.localFolderPaths', [])[0]`; the researcher prompt's default when
  unset is the literal `.switchboard/docs/`. The setting is declared in `package.json`
  (`switchboard.research.localFolderPaths`, array, default `[]`). No new setting is needed.
- **HTTP surface pattern.** `src/services/LocalApiServer.ts` routes are a hand-written `if/else if`
  chain on `pathname` + method inside `_handleRequest`; command endpoints delegate to injected host
  callbacks declared on the `LocalApiServerOptions` interface and wired in `TaskViewerProvider.ts`.
  `POST /phone-a-friend` (→ `onPhoneAFriend`) is the closest precedent: localhost-only, **no auth**,
  best-effort, and its callback **must not throw on "no terminal."**
- **Terminal-send primitive.** `TaskViewerProvider.sendPromptToAgentTerminal(role, text, root)`
  resolves role → terminal name (`_getAgentNameForRole`) → live `vscode.Terminal` (registered map,
  then open-terminals fallback) and sends via `sendRobustText`. **Caveat: it spawns a terminal if
  none is found** — so it cannot itself serve as the availability gate.
- **Liveness source.** `_registeredTerminals` (a `Map<string, vscode.Terminal>`); a terminal is
  live iff `exitStatus === undefined`. This is exactly what `getRegisteredTerminals()` uses to feed
  `/health.terminals`.
- **Skill mirror.** `.agents/skills/advise_research/SKILL.md` is the source of truth;
  `.claude/skills/advise-research/SKILL.md` is generated by `generateClaudeMirror` and differs only
  by a prepended YAML frontmatter block. `npm run mirror:check` gates drift, so both bodies must
  stay byte-identical below the frontmatter.

## Design decisions

1. **Add a first-class endpoint `POST /research/dispatch`**, not directive-driven orchestration of
   `/health` + `sendToTerminal`. Justification: config resolution and liveness detection both must
   happen server-side, and a single `{ dispatched: true|false }` result is far more robust for the
   planner to branch on.
2. **Mirror the `/phone-a-friend` contract:** localhost-only, no auth, best-effort. The callback
   **never throws on "no researcher active"** — it returns `dispatched:false` so the fallback is a
   normal branch, not an error path.
3. **Never spawn a researcher.** Availability is an *observation*, not an action. If no
   `researcher`-role terminal is registered and live, dispatch nothing and report
   `dispatched:false`.
4. **Single source for the save folder** — reuse `research.localFolderPaths[0]` (fallback
   `.switchboard/docs/`), identical to `KanbanProvider`'s `localDocsPath` and the researcher
   prompt's existing default. No new setting.
5. **Multi-root safe** — `workspaceRoot` flows through every call; the planner passes its own
   absolute root; the server defaults to the primary root only if omitted.

## Implementation

### 1. `src/services/LocalApiServer.ts`
- Add an `onDispatchResearch?` callback to the `LocalApiServerOptions` interface (next to
  `onPhoneAFriend`):
  `(workspaceRoot: string, prompt: string) => Promise<{ dispatched: boolean; researcher?: string; savePath?: string; reason?: string }>`.
- Add a route arm in `_handleRequest`: `POST /research/dispatch → _handleResearchDispatch`.
- Add `_handleResearchDispatch`: 503 if the callback is unwired; parse body (`prompt` required →
  400 if missing; `workspaceRoot` defaults to `this._options.workspaceRoot`); call the callback;
  respond `200 { success:true, ...result }`; 500 only on an unexpected throw.

### 2. `src/services/TaskViewerProvider.ts`
- Wire `onDispatchResearch` in the options object to a new private method.
- Add `_dispatchResearchToResearcher(workspaceRoot, prompt)`:
  1. Resolve the workspace root; bail `dispatched:false` if unresolvable.
  2. `_getAgentNameForRole('researcher', root)`; bail `dispatched:false` if none configured.
  3. Resolve the `vscode.Terminal` the same way `sendPromptToAgentTerminal` does (registered map by
     name/suffixed key, then open-terminals fallback), **but never spawn**; bail `dispatched:false`
     if not found or `exitStatus !== undefined`.
  4. `savePath = getConfiguration('switchboard', root).get('research.localFolderPaths', [])[0] || '.switchboard/docs/'`.
  5. Append a save-to-docs instruction to the prompt and call
     `sendPromptToAgentTerminal('researcher', fullPrompt, root)` (terminal already live → no spawn).
  6. Return `{ dispatched:true, researcher, savePath }`.

### 3. `src/services/agentPromptBuilder.ts` — `ADVISE_RESEARCH_DIRECTIVE`
- Extend with a **"researcher hand-off"** step: build the research prompt, read the port from
  `.switchboard/api-server-port.txt`, `POST` it to `/research/dispatch` with `{ workspaceRoot,
  prompt }` (build JSON safely with `jq -Rs` / `python3 json.dumps`, never hand-escape newlines).
  On `dispatched:true` → tell the user it was handed to the Researcher agent and where findings
  save; **do not** paste the prompt. On `dispatched:false` / missing port file / request failure →
  fall back to the current end-of-summary prompt. The `## Uncertain Assumptions` plan-file section
  is unchanged.

### 4. Skill docs
- Add a **"Researcher Hand-Off"** section to `.agents/skills/advise_research/SKILL.md` (source of
  truth) and reproduce the identical body change in `.claude/skills/advise-research/SKILL.md`.

## Verification

- `tsc -p tsconfig.test.json` — the three changed source files must compile with zero new errors.
- `npm run mirror:check` — the `advise_research` → `advise-research` mirror must be consistent (no
  drift introduced by the doc edit).
- Runtime smoke (needs the extension running): with a live Researcher terminal,
  `curl -s -X POST $BASE/research/dispatch -d '{"workspaceRoot":"…","prompt":"…"}'` →
  `{success:true,dispatched:true,researcher,savePath}` and the prompt lands in the researcher
  terminal carrying the save instruction; with no researcher live →
  `{success:true,dispatched:false,reason}`; and the planner, run with the add-on on, chooses
  hand-off vs. chat-paste accordingly.

## Out of scope / follow-ups

- **Protocol catalog / verb-allowlist regeneration** (`protocol-catalog.json`,
  `src/generated/verbAllowlist.ts`) is already badly stale at branch HEAD (a no-edit regen still
  produces a ~3,900-line diff), so regenerating it belongs in a separate maintenance pass rather
  than this change. `GET /catalog` will list `/research/dispatch` once the catalog is next
  regenerated (routes are auto-discovered from the if-else chain).
- **No dedicated test** — consistent with the analogous `phone-a-friend` callback endpoint, which
  has none; a contract test would need VS Code terminal-registry mocks.
- **Possible refinement** — return `dispatched:false` (rather than a 500) if the terminal send
  itself throws after liveness was confirmed, for an even cleaner caller branch.
