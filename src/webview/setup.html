GIT IGNORE STRATEGY
Target: .gitignore (targeted)
Preset strategies are read-only. Switchboard updates only its fenced managed block and preserves unrelated rules. Cloud coders (e.g., Jules) require .switchboard/plans/ and .switchboard/epics/ to be in the repository, so avoid blanket .switchboard/* rules unless you intentionally want to hide plans from git.
AGENT PROTOCOL TARGET
Antigravity reads AGENTS.md; Claude Code reads CLAUDE.md and discovers skills in .claude/skills/. Selecting Claude Code generates native /memo, /clickup-api (etc.) slash commands and a permission allow-list so Bash proxy skills don't prompt.

Note: in Claude Code, skills hot-reload mid-session, but CLAUDE.md and the permission settings load only at session start — restart Claude Code for those to take effect. If the "Accept, do not ask again" prompt narrows your allow-list, re-run Switchboard setup to restore it.

File watchers pick up new plans instantly. Switchboard also scans once at startup to import any plans created while the extension was not running — this startup scan always happens. The Scan Speed below controls how often that scan repeats as a fallback for missed events — both external IDE folders (Antigravity / Cursor / Claude Code / Windsurf-Devin) and the internal .switchboard/plans directory. Set to Off to disable the recurring scan while keeping the startup pass.
Scan Speed
How often Switchboard sweeps for plans as a fallback when file watchers miss an event. Faster = plans appear sooner; slower = less background work. Also controls the internal .switchboard/plans fallback sweep.
IDE Presets
Built-in plan-folder locations per tool. Detected presets are scanned; missing folders are skipped at no cost.
Chat Plan Destination(s)
Where the chat / consultation agent is told to write its plans. Leave empty to use the workspace's .switchboard/plans folder. Add more than one to let the agent choose between them. Each destination is also scanned for ingestion, so plans written here always reach the board. Use ~ for your home directory and <repo> for each workspace root.
Custom Sources
Point the scanner at any other tool's plan folder. Use ~ for your home directory and <repo> for each workspace root. Example: ~/.mytool/plans/*.md
